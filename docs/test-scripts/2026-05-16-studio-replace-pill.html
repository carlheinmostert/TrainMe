2026-05-16 · gesture-discoverability fix · Replace pill

Device QA — Studio long-press reorders; Replace pill on Demo surface

Regression. Long-pressing a Studio card was firing the image-picker swap instead of activating drag-to-reorder. Two hidden gestures fought for the same input; the labelled affordance lost. Carl flagged the regression yesterday.

Fix shape. Drop onLongPress: onReplaceMedia from the card's InkWell so the gesture bubbles up to the parent's ReorderableDelayedDragStartListener (drag-to-reorder). Surface the Replace action as a labelled coral pill on the Demo tab's bottom-left chrome cluster — same visual pattern as the existing Rotate pill. The cluster gains a third pill above Rotate (or, on photos, a single Replace pill in place of the video-only stack).

R-11 / R-01. Gestures should never carry destructive or irreversible affordances. Replace media swaps the captured footage — that needs a labelled, deliberate target.

Files touched: app/lib/widgets/studio_exercise_card.dart (long-press unwired + docstring + missing-media chip text), app/lib/screens/studio_mode_screen.dart (new _ReplacePill, wired into _buildBottomLeftChromeCluster for video and photo paths, callback added to MediaViewerBody), app/lib/widgets/exercise_editor_sheet.dart (host forwards onReplaceMedia through to the viewer).
