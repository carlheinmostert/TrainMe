2026-05-14 · Hero resolver refactor (audit follow-up)

Device QA — Hero resolver: derives treatment internally + no silent fallbacks

Purpose: verify the resolver-API refactor that Carl ratified today. The hero pic on every surface (Studio card, filmstrip cell, lobby row, prep overlay, deck poster, snapshot) must reflect the per-exercise preferred_treatment. When a treatment's variant isn't available, the caller renders an explicit "<TREATMENT> not available" coral placeholder — no silent fallback to a B&W or line-art image of a different treatment.

Before (broken): every caller passed (treatment, bodyFocus) into resolveExerciseHero. Some callers (notably the prep overlay and one filmstrip path) hard-coded grayscale, producing the audit's B6/D13/F17/F21 symptoms — B&W hero on a Line-treatment exercise, etc.

After (this PR): resolver signature simplifies to resolveExerciseHero({exercise, surface}) (Flutter) and HomefitHero.resolve(exercise, {surface}) (web). The resolver maps exercise.preferred_treatment via treatmentFromWire and reads exercise.body_focus internally. When the required variant isn't on disk / not available, return null (Flutter) or a skeleton tag (web), and callers render .hero-not-available instead of substituting.

In scope: Studio card hero, filmstrip cells, lobby rows, prep overlay, deck poster, embedded preview — both mobile (Flutter) and web (player + embedded WKWebView). Drift sync between web-player/* and app/assets/web-player/*.

Out of scope: Studio card 1:1 image-left layout, media-type icon top-right (stacked in a follow-up small PR). Treatment-switching control on the player (unchanged). MediaViewer playback semantics (unchanged).

Refactor narrative: every caller (lobby, prep, deck, snapshot, Studio card, filmstrip, MediaViewer, peek) previously computed treatment + bodyFocus and handed them to resolveExerciseHero. Drift accumulated — the prep overlay and one filmstrip path hard-coded grayscale, masking the audit's B6 / D13 / F17 / F21 symptoms behind a uniform B&W coat of paint. The resolver now derives both from the exercise itself (preferred_treatment + body_focus). Callers only specify surface. Fallback chains across treatments are gone — missing variant = explicit .hero-not-available placeholder.
