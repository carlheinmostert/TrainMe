Device QA — Count glyph auto-fits via FittedBox (no manual tier)

Scope: the leading exercise-count glyph on session cards (the big coral number on the left of every client session card) used a manual font-size tier — 42pt for single-digit counts, 34pt for multi-digit. Two-digit counts were still visually clipping inside the fixed 60×60 slot on Carl's iPhone.

Fix: drop the manual tier entirely. Wrap the Text in FittedBox(fit: BoxFit.scaleDown) with a single fontSize: 42. FittedBox auto-scales the glyph down for 2- and 3-char labels so any count fits the 60×60 footprint without clipping.

What did NOT change: the 60×60 footprint, the coral colour, Inter w800, letter-spacing, tabular figures, drop shadow, the empty-slot behaviour at zero count. Card height, filmstrip background, and analytics row are untouched.

  • PR for branch worktree-agent-a87c218d281746262 merged + ./install-device.sh from repo root.
  • Have at least one client with sessions covering a range of exercise counts — ideally one with 1–9 exercises (single-digit), one with 10–99 (two-digit), and (if you have one) a session with 100+ exercises (three-char "99+"). An empty draft session helps verify the zero-count path.
