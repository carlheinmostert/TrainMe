2026-04-22 · Wave 18.8 — polish round 5

Device QA — Wave 18.8

Purpose: five items from Wave 18.7 live QA. (1) [+] padding tightened 12pt → 10pt; pill now ~38pt wide (was ~42pt) — leaner match to the text chips. (2) Section header label AND summary both bumped to 16pt. Even at 14pt the coral label perceptually read smaller than the 13pt grey inner labels; coral on dark loses the contrast battle. 16pt vs 13pt inner tier gives a real 3pt absolute gap so the section header finally dominates as designed. Label/summary differentiation now carried purely by weight + case + colour. (3) Expand gap between _GroupHeader and first inner content row tightened 8pt → 2pt — header + body feel like one connected unit. Intra-content spacing unchanged. (4) NOTES collapsed summary: replaces all whitespace (including newlines) with single spaces, forces single-line with ellipsis. PACING keeps its up-to-3-line wrap. (5) Inline numeric editor (PREP + DURATION PER REP) now uses Expanded on the text field and a call-site Expanded wrapper — the < Cancel [_N_] Done > row fits entirely inside the card's content area, no more overflow past the right edge.

In scope: mobile-only. preset_chip_row.dart ([+] padding) + studio_exercise_card.dart (section header font bump, expand gap tightening, NOTES single-line collapse, inline editor width fix).

Out of scope: anything already covered by Wave 18 / 18.1 / 18.2 / 18.3 / 18.4 / 18.5 / 18.6 / 18.7. Web player. Portal. Storage semantics of customDurationSeconds unchanged. Chip long-press remove/undo, [+] long-press no-op, inline (not modal) input, R-01 "no popups" — all unchanged.

Prerequisites

  • Wave 18.7 merged (SHA 1a97d15). Wave 18.8 branch fix/wave-18.8-polish installed via ./install-device.sh.
  • At least one session with: (a) a video exercise WITH probed duration; (b) a photo exercise. One exercise with notes containing newlines (multi-paragraph) for Item 9.
  • iPhone 17 Pro or similar 393pt-wide device.

1 · [+] padding 12pt → 10pt

  • 1. Open any exercise card, expand DOSE. The [+] tail renders as a ~38pt-wide dashed pill (was ~42pt in Wave 18.7). Still visibly padded — there's room around the centred + icon — but leaner, with the visual weight more closely matching the text chips [15], [12], [10]. Long-press [+] → no-op. Tap [+] → inline Cancel / numeric field / Done opens (NOT a modal).

2 · Section header hierarchy (16pt label + 16pt summary)

  • 2. Expand an exercise card with DOSE values set (e.g. reps=10, sets=3, hold=5s). Section header DOSE (coral w700 uppercase) reads visibly LARGER than the 13pt inner labels REPS / SETS / HOLD. Hierarchy restored after Wave 18.7's 14pt label perceptually tied with grey inner labels. The coral header dominates the row.
  • 3. Section header summary text (after the · — e.g. DOSE · 10 reps · 3 sets · 5s hold) renders at the same 16pt size as the label. Differentiation is purely weight + case + colour: label = w700 uppercase coral; summary = w500 mixed-case grey. The row reads as one unified header.
  • 4. Inner labels REPS, SETS, HOLD, PREP, DURATION PER REP all still render at 13pt Inter w600 uppercase grey (letter-spacing 0.5). No regression in inner-label size.
  • 5. Inner values (the 10, 3, 5s, 45, 30s) all still render at 13pt JetBrainsMono w700 white. No regression in value size. 3pt absolute gap between 16pt section header and 13pt inner tier reads clearly.

3 · Tight expand gap (8pt → 2pt)

  • 6. Expand DOSE. The gap between the DOSE header row and the first inner content (REPS row) is visually tight (~2pt), not the previous 8pt. Header + body feel like one connected unit, not two separated blocks.
  • 7. Same tight header → first-content gap when expanding PLAYBACK (first row: treatment tiles), PACING (first row: PREP), and NOTES (first row: the text field). All four expanded bodies feel attached to their header now.
  • 8. INTRA-content spacing inside an expanded body is unchanged. Inside DOSE: still ~8pt breathing room between REPS → SETS → HOLD rows. Inside PACING: still breathing room between PREP row and DURATION PER REP row. Only the very first gap (header → first row) was tightened.

4 · NOTES collapsed summary — single-line with whitespace collapse

  • 9. Expand NOTES. Type a multi-line note with actual newlines: Keep back straight [return] Breathe out on exertion [return] Cadence 2-1-2. Collapse the NOTES group (tap the NOTES header). The collapsed summary row reads on ONE visual line, with newlines replaced by single spaces: something like ▸ NOTES · "Keep back straight Breathe ou…" (truncated at ~28 chars + ellipsis). NO multi-line wrap in the collapsed summary, regardless of how many paragraphs the note has.
  • 10. PACING collapsed summary with multiple non-defaults (e.g. PREP override + Manual duration, producing something like PACING · 8s prep, 5s per rep) still wraps to multiple lines if it runs long (up to 3 lines). Only NOTES forces single-line — PACING keeps its multi-line wrap when summary grows.

5 · Inline numeric editor fits within card bounds

  • 11. Expand PACING. Tap the 5s value in the PREP row → inline editor opens. The < Cancel   [_5_]   Done > row stays entirely within the card's right edge — no overflow onto or off the screen edge. The text field fills the space between Cancel and Done, regardless of phone width. On iPhone 17 Pro (393pt wide, ~353pt card content), everything fits cleanly.
  • 12. Same test on DURATION PER REP (PACING expanded on a video-with-probed-length exercise). Tap the value → inline editor opens, row stays within the card's right edge. Cancel + Done remain tappable. Commit a new value → row closes, value updates, toggle auto-flips to Manual as before.
