2026-04-21 · Wave 8 — Sticky per-client defaults

Device QA — Wave 8

Purpose: verify per-client exercise defaults propagate forward-only (never backwards) across 7 fields so a second-session capture pre-fills with the last-edited values. UX should be invisible — practitioner just notices they're not re-typing rep counts. Bundled with landscape capture and the long-press-to-record video hint.

In scope: 7-field sticky propagation (reps, sets, hold, muted, preferred treatment, prep seconds, custom duration), JSONB client_exercise_defaults persistence, SQLite mirror, landscape capture, long-press-to-record hint.

Out of scope: retrospective update of past sessions, cross-client propagation, practice-scope defaults.

Prerequisites

  • Wave 8 PR merged; SQLite schema bumped; Supabase migration for JSONB client_exercise_defaults on clients applied.
  • Test client with at least 2 prior sessions for the "forward-only" boundary checks.

1 · Forward-only propagation across sessions

Edit reps/sets/hold/etc. on exercise X in session A → those values become defaults for the next new exercise captured for that client in session B. Session A's earlier exercises are never retroactively changed.

  • 1. Pick a client with an existing session. Edit exercise X: reps = 12, sets = 4, hold = 3, muted = on, preferred treatment = B&W, prep seconds = 10, custom duration = 45s. Save.
  • 2. Create a new session for the same client. Capture a new video exercise. All 7 fields pre-fill with the values from step 1 — no zeros, no placeholders.
  • 3. Return to the ORIGINAL session (the one from step 1). Other exercises in that session retain their original values — no retroactive update.
  • 4. Create a session for a DIFFERENT client. New exercise does NOT inherit client #1's defaults — values revert to system defaults (or that client's own stored defaults if any exist).

2 · Invisible UX — no affordances added

The propagation must ship without any new toggles, dialogs, or visible hints. Practitioner only notices that fields are pre-filled.

  • 5. No "use previous defaults" toggle, button, banner, or tooltip appears anywhere in Studio. Settings has no new section.
  • 6. Pre-filled fields look identical to manually-entered fields — same colours, no ghost-text distinction, no asterisk/tag.
  • 7. Overriding a pre-filled field on the new exercise updates the sticky default again — next new exercise for this client inherits the latest value, not the one from two edits ago.

3 · Landscape capture

Capture mode now supports landscape orientation. Rotated footage records correctly and plays back without letterbox/squish.

  • 8. Rotate iPhone to landscape in Capture mode — UI adapts (shutter stays thumb-accessible, peek box remains visible, no broken layout). Recorded video orientation matches the device orientation at record time.
  • 9. Play a landscape-captured exercise back in preview — video fills horizontally, no sideways playback, aspect preserved across Line / B&W / Original.

4 · Long-press-to-record video hint

New capture-mode tooltip or label clarifies that a long-press on the shutter records video (short press = photo). Dismissed after first video capture.

  • 10. Fresh install + first capture mode open — "Hold for video" hint appears near the shutter button. Styled subtly (coral on dark, small text).
  • 11. Long-press the shutter to record a first video — hint fades out and does not reappear on subsequent capture-mode opens.
