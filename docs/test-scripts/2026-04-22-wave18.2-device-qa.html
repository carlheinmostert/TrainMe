Device QA — Wave 18.2 patch

Purpose: seven locked items from live device feedback on Wave 18.1. Delta-only — this script validates ONLY the items that changed in 18.2. Wave 18.1's script remains the canonical baseline for everything else (accordion single-open, chip unification, rest row swipe, publish RPC). Items covered here: toolbar gap symmetry; exercise-card summary mixed-case; accordion restructure (PLAYBACK folded in; all-collapsed default); non-default coral-dot position; multi-line collapsed summaries; chip label vertical centring; rest chip row wraps instead of silently hiding chips.

In scope: mobile-only (no R-10 mirror). Studio screen (studio_mode_screen.dart) + exercise card (studio_exercise_card.dart) + preset chip row (preset_chip_row.dart).

Out of scope: everything covered by Wave 18.1's 20 items. Web player. Portal.

Prerequisites

  • Wave 18.1 merged to main. Wave 18.2 branch fix/wave-18.2-qa-patch installed via ./install-device.sh.
  • At least one published session with 3+ exercises, at least one exercise with a non-default PACING value (e.g. custom duration on a video or a prep-seconds override), and at least one exercise with a long NOTES entry (30+ chars).

1 · Studio toolbar gap symmetry

  • 1. Open any session in Studio. Eyeball the gap above the toolbar (session title → toolbar top) and the gap below it (toolbar bottom → first exercise card top). The two gaps read visually equal. If you put a finger on the screen to measure, they're within ~2pt of each other.
  • 2. The overall header region is visibly tighter than Wave 18.1. The AppBar shrank from the default 56pt to 48pt, and both gaps now read as a quiet breath — not a generous margin.

2 · Exercise card header summary

  • 3. Exercise card header summary line (below the exercise name) reads mixed-case: e.g. 3 × 10 · 5s hold · ~45s. NOT 3 × 10 · 5S HOLD · ~45S (Wave 18 / 18.1 rendered it uppercase; Wave 18.2 reverts to mixed-case so it sits naturally next to the exercise name above).

3 · Accordion restructure (4-way single-open)

  • 4. Tap any exercise card to expand it. ALL FOUR groups (PLAYBACK / DOSE / PACING / NOTES) render as collapsed header rows on initial open. PLAYBACK is NO LONGER auto-expanded — the card reads lean + quiet with only chevron + label + summary per row.
  • 5. Tap PLAYBACK header. PLAYBACK expands (treatment tiles + Muted toggle visible). Tap DOSE. PLAYBACK collapses automatically; DOSE expands. Single-open spans all four groups now — PLAYBACK is no longer an independent toggle.
  • 6. From item 5's state (DOSE open), tap DOSE again. DOSE collapses, all four groups now closed. Tap NOTES. NOTES expands; everything else stays closed. Tap NOTES again → closes. All four can end up collapsed; only one can be open at a time. Chevron rotation + selectionClick haptic on every tap.

4 · Non-default coral dot (left of chevron)

  • 7. Find (or create) an exercise with a non-default PACING value — e.g. prep-seconds override to 8s, or flip "Use video length as 1 rep" on a video. Collapse all groups. The PACING row now reads [•] ▸ PACING — 8s prep: the 5pt coral dot is to the LEFT of the chevron, with a 6pt gap between dot and chevron. NOT between chevron and label as Wave 18.1 rendered it.
  • 8. A group at all-default has no ghost space for the coral dot — the chevron sits flush left, label follows immediately after. Compare a default DOSE header (no dot) to a non-default one (dot present) — the chevron position shifts right by ~11pt when the dot appears. No leftover blank slot.

5 · Multi-line collapsed summaries

  • 9. Find (or create) an exercise with a PACING that produces a long summary — e.g. prep override 8s + "use video length as rep" + custom 24s per rep → summary reads 8s prep, video-length on, custom 24s per rep. Collapse all groups. The PACING row wraps to 2 lines cleanly — text flows past the row edge onto a second line. Chevron + coral dot stay top-aligned (at the top of the row, not vertically centred).
  • 10. Create a NOTES entry long enough to wrap — e.g. "Keep back straight, slow on the way down, breathe out on exertion, count cadence". Collapse the card. The NOTES row shows up to 3 lines; anything past line 3 truncates with an ellipsis. Row height grows to fit 1 / 2 / 3 lines as needed.

6 · Chip text vertical alignment

  • 11. Expand DOSE on any exercise. Look at the Reps chip row — e.g. [5][8][10][12][15][Custom]. Text inside every chip sits dead centre vertically inside the 32pt pill. No visible drift towards the bottom of the pill (previously Inter's default line-height left a ~1-2pt low-drift). Compare selected chip (coral fill, white text) to unselected chips (raised fill, light text) — both sit on the same baseline.
  • 12. The [Custom] tail label sits vertically centred too, matching the numeric chips' baseline.

7 · Rest chip row wraps

  • 13. Add a rest period to a session. On the rest bar's chip row, long-press to REMOVE existing chips until you have fewer than 4 chips, then tap [Custom] and add several custom values — e.g. 20, 45, 75, 120, 180. The chip row wraps onto a second row when it runs out of horizontal space. All chips stay reachable — none clip past the right edge. If you add even more customs (filling MRU cap 8), the row wraps to a 3rd line as needed.
  • 14. With a multi-row rest chip layout from item 13, swipe left on the rest row. The full row (all lines of chips included) slides off as a single Dismissible. The red background + Delete label + trash icon slide in as before. No gesture is eaten by the chip row.
  • 15. DOSE chip rows (reps / sets / hold) do NOT wrap — they still scroll horizontally as in Wave 18.1. Only the rest bar's non-scrollable chip row wraps. Verify by adding lots of custom Reps values until the ListView scrolls; the row never grows vertically.
