2026-05-15 - Circuit animation (EIGHTH attempt)

Device QA - Circuit lane tracer + frame geometry

Eighth attempt at a recurring bug. PR #342 (seventh attempt) was confirmed inert by Carl. If this attempt fails too, STOP and escalate - do not ship a ninth attempt without re-architecting the painter.

Why all seven prior attempts failed. Each one fixed WHEN to paint (the trigger: rAF retry, await image load, ResizeObserver re-paint, eager image loading, etc.). None fixed HOW we measure. renderCircuitLanesFor() in web-player/lobby.js reads frame.offsetHeight at the top of paintLanesAndTracer(). The frame's height equals the SUM of its child rows' heights. Rows contain <img class="lobby-hero-media"> which, even with aspect-ratio: 1/1 on the parent .lobby-hero, can resolve to 0 on the first synchronous read before the layout pass settles. Result: frameH = 0, SVG sized to nothing, tracer drawn along an invisible path, animation invisible.

The eighth-attempt fix:
- Stop reading frame.offsetHeight. Sum row heights via getBoundingClientRect() directly on each .lobby-row, plus the header height, plus computed body padding + gap. Walking rows avoids the parent-collapse problem.
- Add aspect-ratio: 1/1 directly on .lobby-hero-media in web-player/styles.css. The parent .lobby-hero already has it; adding it on the <img> itself reserves height even before flex resolution finishes.
- Defence-in-depth: keep the ResizeObserver + MutationObserver from PR #342 (catch any subsequent layout shift). Add document.fonts.ready.then(() => renderCircuitLanesFor(...)) to cover web-font-load shifts.
- Soft fallback: if the row-sum measurement is still 0 (degenerate case), fall back to frame.offsetHeight - same as before. Belt-and-braces, never a regression.

R-10 parity: both web-player/lobby.js + web-player/styles.css AND app/assets/web-player/lobby.js + app/assets/web-player/styles.css are byte-identical (MD5-confirmed). Sensitive zone: PR body flags this as Carl-review-before-merge.

In scope: circuit lane tracer + static frame geometry, in embedded preview and live staging surfaces. The headline test is the same as #342: open preview WITHOUT scrolling, see the tracer animating within 2 seconds.

Out of scope: any non-circuit lobby behaviour, prep hero treatment, PDF export modal, filmstrip sharpness.
