Lobby hotfix round 2 — strip coral backdrops, tree-branch rail, drop numerals

Three follow-ups Carl surfaced after PR 234's lobby device-QA round landed. Same shared web-player/ bundle — mobile workflow Preview WebView gets the fixes for free.

Fix 1. Pill matrix had a coral backdrop on circuit blocks. Stripped — the matrix sits on the dark surface, only the pills carry coral.

Fix 2. Circuit-group chrome rebuilt around a coral tree-branch rail (canonical mockup at docs/design/mockups/lobby-circuit-header.html). Container is transparent (no backdrop / no border); rail is 3 px wide at left:8px, colour rgba(255,107,53,0.85), composed from .circuit-header::before/::after and .ex-row.in-circuit::before/::after. Last in-circuit row gets .last — rail clamps to height:50% to form a clean └ corner. Cycles chip reads ×3 only. Duplicate-header bug fixed: a rest slide whose circuit_id matches the open circuitGroup is now appended to the group's rows instead of closing it (close-and-reopen was emitting two .lobby-circuit-group <li>s with the same id).

Fix 3. Row index numerals dropped entirely. Studio's auto-naming gives default-named exercises labels like Exercise 1, Exercise 2 — so the number is already part of the name when the practitioner hasn't renamed. Renamed exercises (Goblet squat) show only the custom name. numberFor(), .lobby-info-num, and .lobby-info-titlebar are gone.

SW cache bumped: homefit-player-v36-lobby-fixes → homefit-player-v37-lobby-hotfix-round2. Build-marker chip on the lobby will read v37-lobby-hotfix-round2 · sw v37-lobby-hotfix-round2 once the new SW takes over.

Prerequisites

  • Pull / hard-refresh session.homefit.studio in Safari (Cmd-Shift-R or kill + relaunch) so the new SW activates.
  • Confirm the build-marker chip in the lobby's metadata strip reads v37-lobby-hotfix-round2 · sw v37-lobby-hotfix-round2. If the SW label still shows the old v36-lobby-fixes, swipe Safari closed and reopen.
  • Open a published session that has at least one multi-exercise circuit (e.g. 3 exercises × 3 rounds) so the rail geometry exercises every state. If you also have a session with a rest inside a circuit, even better — that's the duplicate-header repro path.

1 · Build marker + version sanity

  • 1. Build chip reads v37 on both halves. Open a session, scroll to the metadata strip at the top of the lobby. Expected: the dim mono chip below the date stamp reads v37-lobby-hotfix-round2 · sw v37-lobby-hotfix-round2 — both sides match. Divergence (e.g. sw v36-lobby-fixes) means the new SW hasn't activated yet; hard-refresh.

2 · Fix 1 — pill matrix has no coral backdrop

  • 2. Circuit pills sit on the dark surface, no coral wash. In the lobby, look at the sticky pill matrix at the top. Expected: the pills themselves carry coral (active/full pills); the background behind any circuit grouping is the same dark #0F1117 as the rest of the matrix — no faint coral fill block around the circuit pills. The circuit pills should still be visually grouped via spacing/columns; they just shouldn't have a tinted backdrop.
  • 3. Pill tap accuracy unchanged. Tap a circuit pill to jump to that row. Expected: works as before (PR 234 floor of 24 px per column still applies); no size regressions, no ghost taps. Long plans still scroll the matrix horizontally rather than squashing pills.

3 · Fix 2(a) — circuit container is transparent

  • 4. No coral backdrop around the circuit group. Scroll to a circuit in the lobby's hero list. Expected: the area surrounding the circuit's exercise cards has NO coral tint. Each card sits on the same dark #1A1D27 surface as standalone exercise cards; no faint coral wash, no coral border ring around the whole group.

4 · Fix 2(b) — tree-branch rail geometry

  • 5. Header rail piece + horizontal stub. Look at the circuit's header row (the uppercase circuit name + cycles chip). Expected: a thin coral rail starts at the vertical centre of the header and runs straight down to the bottom edge of that header. A short coral stub (~14 px) extends rightward from that rail to just before the circuit name. The header itself has no background and no bottom border — only the rail conveys structure.
  • 6. Per-row connectors form a continuous rail. Scan down the in-circuit rows. Expected: the coral rail continues unbroken from the header through every middle in-circuit row. Each row has a small coral stub (~14 px) joining the rail to the row content at the row's vertical centre. The vertical line should look like ONE continuous coral piece, not three stacked-but-misaligned segments.
  • 7. Last row terminates with a clean └ corner. Look at the bottom in-circuit row. Expected: the vertical rail stops at the row's vertical centre (not at the row's bottom edge); combined with the horizontal connector at that same centre line, it forms a clean └ corner that visually closes the group. The next item (a standalone exercise or a plan-level rest) sits free of any rail.
  • 8. Cycles chip reads ×N only. Look at the right side of the circuit header. Expected: the pill chip shows just the multiplier (e.g. ×3) — no “ROUNDS” / “rounds” suffix, no extra space. Same coral pill chrome as before.

5 · Fix 2(c) — duplicate-header bug

  • 9. Exactly one header per circuit. Scroll through every circuit in the session. Expected: each circuit shows ONE coral uppercase header at the top of its group — never two. (The bug Carl reported showed two stacked headers on the first circuit; the close-and-reopen on rest-inside-circuit is the root cause and is now fixed.)
  • 10. Rest-inside-circuit renders inside the rail. If your test session has a Rest exercise sitting between two circuit exercises (same circuit_id), look for it in the lobby. Expected: the rest row appears INSIDE the circuit group, with the coral rail still running through it; the rail does NOT break and re-form below. The circuit header above stays single. (If you don't have such a session, mark this Pass-with-note and move on — it's the rare repro path.)

6 · Fix 3 — no row numerals

  • 11. Default-named exercises show the auto-name as-is. If you've got a row whose name is the Studio default (e.g. Exercise 3), look at its info column. Expected: the name renders verbatim — the “3” is part of the name itself, not a separate prefix numeral. No 3. in front of Exercise 3.
  • 12. Renamed exercises show only the custom name. Scroll to a row whose name was edited in Studio (e.g. Goblet squat). Expected: the name is just Goblet squat — no leading 2. or any other numeral; no extra dot, no extra space, no leftover .lobby-info-titlebar column eating horizontal space. Hero card width unchanged from before.

7 · Regression sanity

  • 13. Pill ↔ row coupling still works. Scroll the lobby's hero list slowly. Expected: the matrix's active pill keeps tracking the row whose vertical centre is closest to viewport centre. Tap a pill — the corresponding hero row scrolls into view. PR 234's nearest-to-centre rAF reducer is untouched.
  • 14. Treatment switching still propagates globally. Tap Line / B&W / Colour in the bottom CTA bar. Expected: all rows switch in lockstep. Locked treatments still show the padlock + open the self-grant modal. (This was Fix 4 of PR 234 — the always-write override path.)
  • 15. Plan-level rest rows still render OUTSIDE any circuit. If your session has a Rest exercise that sits between two top-level (non-circuit) items, it should render as a sage Rest row outside any group. Expected: ONE Rest row, sage chrome, no rail, no coral wash — same as before this PR. The rest-inside-circuit fix should only affect rests whose circuit_id matches an open circuit group.
  • 16. Start Workout still hands off cleanly. Tap the coral Start Workout CTA. Expected: the lobby dismisses, the deck takes over, the prep countdown fires, and the first exercise plays as normal. Nothing about the lobby chrome interferes with the handoff.
