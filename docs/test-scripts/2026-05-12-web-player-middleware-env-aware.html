Device QA — Web-player middleware env-aware OG unfurls

Scope: web-player/middleware.js previously hardcoded the prod Supabase URL + anon key AND the prod OG-card host. Bot user-agents (WhatsApp, Slack, Twitter, facebookexternalhit, LinkedIn, Telegram) hitting staging.session.homefit.studio/p/<id> queried PROD Supabase for a plan that lives in STAGING, producing “Not found” unfurls.

Fix (A6): middleware now reads NEXT_PUBLIC_SUPABASE_URL + NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY (fallback NEXT_PUBLIC_SUPABASE_ANON_KEY) from process.env. Strict-fail policy mirrors build.sh: missing env => log + pass through (never silently route to prod).

Fix (A6b): the OG og:url is now derived from the incoming request origin, so staging unfurls show staging.session.homefit.studio/... instead of the prod host.

Prerequisites

  • PR fix/web-player-middleware-env-aware merged to staging, Vercel preview/staging deployment live at staging.session.homefit.studio.
  • Have a published plan on staging (so a real staging.session.homefit.studio/p/<id> URL exists) and another on prod (so a session.homefit.studio/p/<id> URL exists).

1 · Staging publish gives staging URL

  • 1. Publish a plan from the staging dev app and confirm the share URL starts with https://staging.session.homefit.studio/p/ (not the prod host).

2 · Staging unfurl in WhatsApp shows real data

  • 2. Paste the staging URL in a WhatsApp chat (or Slack / Twitter). The unfurl preview shows the actual plan title and hero image — not “Not found”.

3 · Staging cURL with WhatsApp UA returns real OG title

  • 3. Run curl -A "WhatsApp/2.0" https://staging.session.homefit.studio/p/<id>. The response HTML contains a <meta property="og:title"> with the real plan title (not the “Your exercise plan” fallback) and og:url points at staging.session.homefit.studio.

4 · Prod still works

  • 4. Run the same curl -A "WhatsApp/2.0" against a prod plan URL (session.homefit.studio/p/<id>). Still returns the real plan title + thumbnail — prod env vars resolve correctly.

  • 5. Hit https://staging.session.homefit.studio/p/00000000-0000-0000-0000-000000000000 with curl -A "WhatsApp/2.0". Returns a sensible 404 response (or the SPA fallback) — no crash, no stack trace, no leaked env values.

6 · Normal browser passes through

  • 6. Open the staging plan URL in Safari / Chrome (no special UA). The middleware passes through (returns undefined), the SPA loads normally and the plan plays. No OG-only HTML body rendered to the user.
