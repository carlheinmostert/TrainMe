2026-04-22 · Wave 18.5 — polish + pacing redesign

Device QA — Wave 18.5

Purpose: four items from Wave 18.4 device QA feedback + a pacing-section redesign. (1) Chip-row spacing tightened further (2pt gaps, 10pt chip padding, 6pt [+] padding) so three customs + canonical + [+] fit one line on iPhone. (2) [+] reshaped from a dashed circle into a 32pt-tall, 16pt-radius dashed pill matching the chip visual family. (3) Section header summary font bumped 11pt → 13pt to balance the 13pt label; differentiation carried by weight + uppercase + colour alone. (4) PACING redesign: the old "Use video length" toggle + "Custom duration" row are retired in favour of a single "Duration per rep" widget with two modes — segmented control on video-with-probed-duration (From video / Manual: Xs), single tappable pill on photos or videos without probed duration. Manual seeds at 5s × reps. No "Custom duration" copy survives in the card.

In scope: mobile-only. preset_chip_row.dart (chip tightening + [+] reshape) + studio_exercise_card.dart (header summary font + PACING redesign).

Out of scope: anything already covered by Wave 18 / 18.1 / 18.2 / 18.3 / 18.4. Web player. Portal. The useVideoLengthAsOneRep bool is NOT a new model field — the storage semantics of customDurationSeconds are unchanged (null ↔ "From video" / auto; set ↔ "Manual" / practitioner override).

Prerequisites

  • Wave 18.4 merged (SHA 4fa6dd4). Wave 18.5 branch fix/wave-18.5-polish-pacing installed via ./install-device.sh.
  • At least one session with: (a) a rest period; (b) one VIDEO exercise with a probed video length (capture a 3-4s clip, let it finish converting); (c) one PHOTO exercise; (d) at least one custom rest value already stored (tap [+] and add e.g. 20) so the multi-custom layout is tested.
  • iPhone 17 Pro or similar 393pt-wide device (same viewport Wave 18.4 failed on).

1 · Chip row — one-line fit on iPhone

  • 1. Open a session, expand the PLAN group on any exercise card. The Reps chip row renders [5] [8] [10] [12] [15] + [+] + one custom chip all on ONE horizontal line. No horizontal scroll needed to reach the [+]. If reps has no custom yet, tap [+] + enter e.g. 20 first so there's a custom in the row.
  • 2. On the rest bar (swipe-reveal rest between exercises), the chip row renders [15s] [30s] [60s] [90s] + [+] + up to 3 custom values all on ONE horizontal line. No wrap to a second row until a 4th custom is added.
  • 3. Inter-chip spacing is visibly tighter than Wave 18.4 (2pt vs 4pt). Look at the gap between two adjacent canonical chips on the rest row — it's a hair-width gap now, not the 4pt Wave 18.4 gap. Chip padding is also slightly tighter (10pt vs 12pt horizontal) — "15s" reads slightly narrower than before.

2 · [+] tail — dashed pill

  • 4. Inspect the [+] tail on any PLAN row. It renders as a dashed pill — 32pt tall, 16pt border radius, same family as the text chips. NOT a perfect square, NOT a pure circle (the old 18.4 shape derived from shortestSide/2). Slightly narrower than "15s" or "10" text chips, but reads as the same PILL aesthetic.
  • 5. Tap [+] on any row — inline input opens (Cancel · numeric field · Done). No modal, no bottom sheet. Enter a value + Done → new chip appears in the row. Haptic selectionClick fires on tap. Behaviour unchanged from Wave 18.4.
  • 6. Long-press [+] — no-op. No haptic, no action, no undo SnackBar. Compare with long-press on a value chip (e.g. [15s]) which DOES remove + undo. Unchanged from Wave 18.4.

3 · Section header summary — 13pt

  • 7. Collapse all four group headers (PLAYBACK / PLAN / PACING / NOTES). On a row with a non-default summary (e.g. PLAN · 10 reps · 3 sets · 5s hold), the label and summary render at the same size (13pt). The label stands out via weight (w700 vs w500) + uppercase + coral colour. The summary stays normal-case, secondary-grey, w500. No more 13pt-vs-11pt size mismatch.
  • 8. Expand PACING. The label stays 13pt, summary (when present) also at 13pt, joined by · . No regression back to 11pt summary. The label is still visibly the emphasised element in the row (weight + case + colour carry the differentiation).

4 · PACING — "Duration per rep" redesign

  • 9. On a video exercise with a probed video length (i.e. a clip you recorded and let convert), expand PACING. Under the PREP row you see a "Duration per rep" row with a two-segment control: [ From video ] [ Manual: Xs ]. One and only one segment is coral-filled; the other is surfaceRaised. If the exercise was fresh, From video is selected by default (since customDurationSeconds starts null).
  • 10. On the same card, tap Manual: Xs. Segment flips coral. Value shows 5s (the seed). Effective duration in the card's summary row (above PLAYBACK) updates to 5s × reps (e.g. 50s manual for 10 reps). The card collapses summary row and the PACING row's non-default coral dot appear. Behaviour is one-and-only-one — From video is now unfilled.
  • 11. Tap Manual: 5s AGAIN (the already-selected segment). Inline numeric input opens inside the Manual segment — no modal, no bottom sheet. Type 8, Done (or blur). Segment updates to Manual: 8s. Persists after card close/reopen. Runtime effective duration recomputes as 8s × reps.
  • 12. Tap From video. Segment flips coral. Manual segment goes unfilled. Effective duration reverts to the video-length × reps auto-calc. PACING non-default coral dot disappears on the section header (since customDurationSeconds is now null again and no prep override is set).
  • 13. On a photo exercise, expand PACING. The "Duration per rep" row renders a single tappable value pill (no segmented control). Value reads 5s. Tap it → inline numeric input opens. Commit a new value → persists. The photo exercise now shows a coral non-default dot on PACING (because customDurationSeconds is now set — expected side-effect of the 5s seed).
  • 14. Scan the ENTIRE card (collapsed + every expanded section + summary row). No copy reading "Custom duration" survives anywhere. The PACING summary uses "Xs per rep" (no "custom" prefix); the card row summary uses X manual (not X custom); the PACING body is labelled "Duration per rep".
Wave 18.5 closes the Wave 18.4 polish-fail items + rolls in a pacing-section redesign. Wave 18.1 remains the canonical baseline (run if anything around PLAN / PACING / rest bar feels off). Storage semantics of customDurationSeconds are unchanged — legacy sessions read through cleanly, a previously-toggled video-length exercise simply renders as Manual: Xs with X = video length.
