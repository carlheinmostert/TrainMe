2026-05-13 · Embedded Preview "No internet connection" fix

Device QA — Embedded Preview Strict-Fail Bug

Purpose: verify the mobile workflow Preview step renders against the local homefit-local://plan/ scheme handler. The bug: web-player/api.js's strict-fail check (commit 2cda208, 2026-05-11) threw at module load when window.HOMEFIT_CONFIG was empty AND the host wasn't in the loopback-exception list. The custom-scheme host (plan) wasn't in that list, so the embedded WebView fell into app.js's catch-all and rendered "No internet connection".

Fix: added plan to the strict-fail loopback exception list, in lockstep with isLocalSurface()'s existing host check.

In scope: Workflow Preview WebView boot, local-scheme media serving, regression check on the Vercel-served staging.session.* surface.

Out of scope: haptic bridge (unchanged), audio session (unchanged), CSP / middleware (PR #307 territory).

Prerequisites

  • Build from fix/embedded-preview-no-internet branch installed on Carl's iPhone via ./install-device.sh.
  • At least one client session with one video exercise + one photo exercise (so both media-path branches exercise).
  • Staging Supabase branch reachable (vadjvkmldtoeyspyoqbx.supabase.co).

  • 6. Open https://staging.session.homefit.studio/p/<planId> on Safari (use a known-published staging plan). Lobby renders, no console errors about missing HOMEFIT_CONFIG. Workout starts cleanly. (Confirms the strict-fail logic still gates non-local surfaces correctly.)
Root cause: web-player/api.js commit 2cda208 added a strict-fail guard with a loopback-only exception that omitted host plan (the Wave 4 Phase 2 custom-scheme host). The strict-fail check fires at IIFE load time, before isLocalSurface() is exposed — so even though isLocalSurface() handles plan, api.js threw before app.js could call it. window.HomefitApi ended up undefined, app.js's fetchPlan threw TypeError, and the catch block surfaced "No internet connection".
