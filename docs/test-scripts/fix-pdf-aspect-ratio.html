2026-05-15 · Lobby PDF aspect ratio (BUG 6c)

Device QA — PDF export aspect ratio no longer stretched

Bug (BUG 6c): Exporting a plan PDF from the homefit.studio lobby produced a horizontally stretched image when opened on Mac — images and text inflated by ~7%, making brick-wall photos and hero shots look squashed landscape.

Root cause: .lobby-export-page in web-player/styles.css declared width: 794px + box-sizing: border-box + padding: 28px 28px 22px. html2canvas captured the full 794-px bordered-box width but only 738 px of that was actual content. pdf.addImage then stretched the 1588-px canvas (scale=2) to fill A4's 210 mm width, inflating content horizontally.

Fix: the 28 px horizontal padding now lives on an inner .lobby-export-page-inner wrapper. The outer wrapper's 794 px IS the content width, so html2canvas captures the right width and addImage stamps without horizontal distortion. The padding renders inside the rasterised image as real PDF margin space.

Files touched: web-player/styles.css, web-player/lobby.js, app/assets/web-player/* (bundle synced byte-identical).
