In scope: Home + Client header + Studio shell (toolbar, summary chip), Exercise card (accordion, chip rows, status), Rest rows (swipe, chips), Media Viewer (mute pill, coral dots), Publish flow (23505 fix + dirty-on-capture + SessionCard cleanup), Migration (Wave 18 presets carry forward).

  • 2. Client detail header. Open any client. The header row under the client name shows only the consent chip — no "N sessions" text beside it. Session count lives only in the list items below (mobile saves space; portal keeps its count).
  • 3. SessionCard clean. In the client-detail session list, each session card shows title + chevron only — no Publish icon, no Share icon on the card. Publishing lives inside Studio now.
  • 4. Studio toolbar exists. Open a session. Directly under the session title is a new toolbar with 4 icons in order: Import / Preview / Publish / Share, separated by small right-pointing coral triangles. Icons centred as a group.
  • 5. Triangle dim echoes state (Variant C1). On a never-published session, the triangle feeding Share is dim (45% opacity) because Share itself is dim. Once published, that triangle is full coral.
  • 6. No exercise-count summary chip. Above the exercise list — nothing. No "N exercises" pill, no summary row. Straight from toolbar to first exercise card.
  • 7. Toolbar is tight. Visibly tighter than Wave 18 (40pt vs 52pt). Icons at 22pt visible glyph, coral triangles between. Compact density — tap targets still respond reliably to taps on/near the icons.
