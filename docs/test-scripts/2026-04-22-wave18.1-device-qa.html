Device QA — Wave 18.1 patch wave

Purpose: five locked follow-ups from live device feedback on Wave 18. The biggest landing is the atomic replace_plan_exercises RPC that closes the 23505 regression on publishing reordered sessions. The exercise card retires its auto-expand-on-non-defaults rule in favour of a proper single-open accordion (DOSE / PACING / NOTES share one open slot; PLAYBACK toggles independently). Preset chip rows unify canonical + custom into one mutable list; long-press removes any chip. Studio toolbar tightens from 52pt → 40pt. Rest row gets its own Dismissible + non-scrolling chip row so swipe-to-delete works again.

In scope: mobile-only (no R-10 mirror — practitioner-facing surfaces only). Exercise card accordion; preset chip row; studio toolbar; rest row; publish RPC.

Out of scope: web player, portal. Wave 17 verification + analytics consent still open.

Prerequisites

  • Main through Wave 18 merged. Wave 18.1 branch fix/wave-18.1-patch installed via ./install-device.sh.
  • Supabase migration supabase/schema_fix_publish_replace_exercises.sql applied via supabase db query --linked --file ... (already done during implementation — re-running is idempotent).
  • At least one published session on the device with 3+ exercises (needed for the reorder-and-publish test). Ideally one session already in the cap-3 preset-memory state (custom values stored under Wave 18).

1 · Exercise card accordion

  • 1. Open any fresh (all defaults) exercise card. PLAYBACK is expanded by default — treatment tiles + Muted toggle visible. DOSE / PACING / NOTES are ALL collapsed with a chevron + label + em-dash summary on each row.
  • 2. Tap DOSE header. DOSE expands (reps + sets + hold chip rows visible). PACING + NOTES stay collapsed. PLAYBACK stays open — unaffected by the accordion.
  • 3. Tap PACING header. PACING expands. DOSE collapses automatically (only one of the three accordion members can be open at a time). PLAYBACK still open. Chevron rotates 180ms ease, haptic selectionClick on the tap.
  • 4. Tap NOTES header. NOTES expands; PACING collapses. Tap NOTES again → it collapses, all three accordion members now closed. PLAYBACK still open (independent toggle).
  • 5. Tap PLAYBACK header. PLAYBACK collapses to header row. Tap again → expands. The DOSE/PACING/NOTES accordion state is unchanged by these taps — PLAYBACK is an independent toggle.
  • 6. Set a non-default PACING value (e.g. edit prep-seconds to 8 or turn on the video-length-as-rep toggle on a video). Close + reopen the card. The PACING header row renders with (a) a 5pt coral dot between the chevron and the label, and (b) the label text bolds to w800. The card DOES NOT auto-expand PACING — the accordion starts closed per Wave 18.1 spec.
  • 7. Add a short note (e.g. "Keep back straight"). Close + reopen the card. NOTES header renders with the same coral dot + bolded label. Still collapsed by default — no auto-expand on non-default content.
  • 8. Collapsed PLAYBACK summary on a VIDEO exercise reads Line · Audio on (or B&W · Muted / Original · Audio on, depending on the exercise's preferred treatment + mute state). On a PHOTO, the audio segment is omitted — e.g. Line on its own.
  • 9. Collapsed DOSE summary reads 10 reps · 3 sets · 5s hold (standalone) or 10 reps · 5s hold (inside a circuit — sets hidden). A hold of 0 renders as Hold off. Changing any field updates the summary live when the card is next collapsed.
  • 10. Collapsed PACING summary reads defaults when all pacing fields are at the default, or a comma-separated list when not: e.g. 2s prep, video-length on, custom 24s per rep.
  • 11. Collapsed NOTES summary reads empty when no note, or the first 28 chars quoted with an ellipsis if longer: e.g. "Keep back straight, slow on…".

2 · Preset chip row unification

  • 12. Expand DOSE on any exercise. Long-press [12] in the Reps chip row. Haptic selectionClick fires, the chip is removed from the row, and a Undo SnackBar appears reading "Removed reps 12".
  • 13. Tap Undo in the SnackBar from item 12. The [12] chip reappears in the row in the same sorted position. Fires selectionClick haptic.
  • 14. Long-press on [Custom] tail. NOTHING happens — no haptic, no action, no SnackBar. Tap (not long-press) still opens the inline input.
  • 15. All chips in the row look identical — flat fill, no little coral dot in any corner on any chip. Selected chip is coral-filled with white label; unselected chips are surfaceRaised-filled with textOnDark labels. [Custom] tail stays dashed-coral.

3 · Rest row + toolbar

  • 16. Swipe-left on a rest row. A red background with the label Delete + trash icon slides in from the right; releasing past the threshold removes the rest row + shows the Undo SnackBar. Swipe works identically to exercise-card swipe-to-delete.
  • 17. Rest bar's chip row does NOT scroll horizontally. All four canonical chips ([15s][30s][60s][90s]) + the [Custom] tail fit in the visible row. Try to swipe horizontally inside the chip row — the gesture is not captured by the chip row; it triggers the rest-row swipe-to-delete instead.
  • 18. Compare the Studio toolbar to the Wave 18 screenshot (or a side-by-side device). The toolbar is visibly tighter — it's 40pt tall, not 52pt. Icons stay 22pt visible glyph; triangles stay the same visual size. No change to any of the four icons' semantics (Import / Preview / Publish / Share).

4 · Publish + migration

  • 19. Core regression fix. Open a published session with 3+ exercises. Reorder at least one exercise (long-press + drag) so positions change. Tap the Publish icon in the toolbar. The publish spinner shows, then the success toast ("Published v{N+1}"). NO 23505 error toast appears. Open the plan URL on a second device / incognito browser — the new order loads.
  • 20. Preset-memory migration. On a device that already had Wave 18 installed with at least one custom preset (e.g. a custom 13 reps chip under the old cap-3 model), install the Wave 18.1 build. Open any DOSE section. The Reps chip row renders a unified list — the canonical [5][8][10][12][15] seeds AND the prior custom [13], sorted in numerical order. Nothing was lost in the migration.
