Wave 42 · consolidated gear panel + per-exercise client overrides

Web player gear panel — per-exercise overrides

The standalone mute speaker icon, the standalone treatment segmented control, and the gear-popover Body Focus switch have all been retired. The gear button now opens one panel with three rows (Mute, Treatment, Body focus) plus a Reset to practitioner defaults button. Each control sets a per-exercise client override that layers on top of the practitioner's per-exercise default. Overrides persist per plan in localStorage. 9 items in 2 sections.

A · Defaults + override scope

  • A1. Practitioner defaults respected on first open.
    1. Open a published plan whose exercises have varied practitioner settings — at least one with body_focus=false, one with preferred_treatment set to grayscale or original, one with include_audio=true.
    2. Walk every slide. Confirm each plays the practitioner's chosen treatment + body-focus state. Audio plays where include_audio=true.
  • A2. Gear panel opens with three neutral rows + greyed Reset.
    1. Tap the gear icon (top-right vertical chrome).
    2. Confirm rows: Mute · Treatment (Line / B&W / Original) · Body focus · Reset to practitioner defaults.
    3. No coral border on any row (no overrides yet). Reset button is greyed and not tappable.
  • A3. Treatment override on exercise 3 turns control coral + persists through navigation.
    1. Navigate to exercise 3. Open gear panel. Tap Treatment > Line.
    2. Confirm the Line pill goes coral. Treatment row gains a coral border. Video swaps to the line drawing.
    3. Swipe forward to exercise 4. Open gear panel. Confirm exercise 4 plays its OWN practitioner default (not Line).
    4. Swipe back to exercise 3. Confirm Line override is still applied (coral cue + line drawing playing).
  • A4. Override survives a full reload (localStorage).
    1. With the Line override on exercise 3 still active, hard-reload the page.
    2. Navigate to exercise 3. Confirm Line is still selected (coral cue, line drawing playing).
  • A5. Tapping the same override pill again clears it (back to default).
    1. On exercise 3, with Line override active, tap Line again.
    2. Confirm coral cue clears, treatment row border returns to neutral, video reverts to the practitioner default for that exercise.

B · Reset, scope, mute

  • B6. Reset to practitioner defaults clears all overrides.
    1. Set overrides on at least three exercises (mix of mute, treatment, body focus).
    2. Confirm Reset button is now coral and tappable.
    3. Tap Reset. Confirm: every control returns to neutral, every exercise reverts to its practitioner default, Reset goes back to greyed.
  • B7. Override scope is per-exercise — exercise 3 doesn't bleed into exercise 4.
    1. Set treatment=Line on exercise 3.
    2. Navigate to exercise 4. Confirm exercise 4 plays its own practitioner default, NOT Line.
    3. Open gear panel on exercise 4. Confirm no coral cues — overrides on exercise 3 do not show up here.
  • B8. Per-exercise mute — exercise 5 muted does not affect exercise 6.
    1. Find an exercise (5) with include_audio=true. Tap Start Workout so audio is unmuted by default.
    2. Open gear panel on exercise 5 → tap Mute. Audio cuts. Mute row shows coral cue.
    3. Navigate to exercise 6 (also include_audio=true). Confirm audio plays per its default. Mute row on exercise 6 has no coral cue.
  • B9. Service worker bumped — homefit-player-v81-wave42-overrides loads on hard refresh.
    1. DevTools → Application → Service Workers (or Safari Web Inspector → Storage). Hard reload the player.
    2. Confirm the active cache name is homefit-player-v81-wave42-overrides (or higher). Old caches purge on activation.
    3. Confirm the build-version marker in the footer reads v81-wave42-overrides.

Out of scope

  • Mobile / Flutter player (PR #146 already shipped the schema + mobile half).
  • Web portal client-consent UI.
  • No localStorage migration of legacy homefit-muted / homefit.playback.* keys — clean cutover.
