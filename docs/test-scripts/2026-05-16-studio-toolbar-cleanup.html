2026-05-16 · studio toolbar cleanup (PR #371)

Device QA — Studio toolbar visual cleanup

Scope: the Studio bottom workflow pill drops the sideways triangle separators and the dark round chip behind each glyph; icons grow to 28pt; the Adjust glyph swaps to a rotated Icons.tune (vertical sliders with horizontal handles).

Why: tapping any cell launches a different surface — none is ever "selected" — so the directional triangle was misleading, and the dark chip duplicated information the cell-opacity readiness signal already carried. Five launchers, not five stages.

What did NOT change: readiness rules (Adjust/Preview/Publish/Share dim while their preconditions are unmet), cell order, tap-targets and tap behaviour, the Unlock pill on locked plans, publishing spinner, the right-aligned Download utility, web player (this is mobile-only).

Prerequisites

  • PR #371 merged to staging + ./install-device.sh staging from the repo root. Confirm Settings → About shows the new short SHA.
  • Have at least one client with a session that has captures (for the "lit" state) and one with an empty new session (for the "dim" state).

1 · Triangles are gone

  • 1. Open any session in Studio. The bottom workflow pill shows 5 cells (Capture · Adjust · Preview · Publish · Share) sitting flush in a single rounded pill with NO coral triangles, chevrons, arrows, dots, or any other glyph between cells. Just the cells, separated by whitespace.

2 · No dark chip behind glyphs

  • 2. Look closely at each cell's icon. Each glyph sits DIRECTLY on the pill surface — no darker round chip / circle / disc behind it. The icons just float on the pill background.

3 · Icons feel bigger

  • 3. Compare to your memory of the prior shipping build (or the screenshots in PR #371). Icons are noticeably larger / more present — roughly 28pt vs the prior ~20pt. They dominate the cell instead of looking lost in the chip.

4 · Adjust icon is vertical sliders

  • 4. Look at the Adjust cell (slot 2). The glyph is now three VERTICAL slider bars with horizontal handles bisecting them — the "tune / equalizer" look. NOT the old list-of-rows glyph. Visually it now reads as "adjustment controls", which is what Adjust actually does.

5 · Readiness opacity still works

  • 5. Open a brand-new empty session. Capture is bright/white; Adjust, Preview, Publish, Share are clearly DIMMED (~40-45% opacity on both glyph and label). Tapping a dim cell does nothing.
  • 6. Capture one exercise, return to Studio. Adjust and Preview wake up (now bright). Publish stays bright too if you have credits + the session has unpublished changes. Share stays dim until the session is published. The "lit" / "dim" pattern is exactly as before — only the chrome around it changed.

6 · Tap targets and behaviour

  • 7. Sanity-tap every cell. Capture opens the camera. Adjust opens the editor sheet for the topmost capture. Preview opens the workout preview. Publish publishes (or opens the unlock pill on a locked plan). Share opens the share sheet on a published plan. None of these regressed when the triangles and chips were stripped.
  • 8. Right-aligned Download utility unaffected. The cloud-download icon still sits outside the workflow pill on the far right. Tapping it still downloads the session's media to camera roll with the usual progress toast. No shift, no resize.

7 · Lock state preserved

  • 9. If you have a plan that's been opened by a client more than 14 days ago (or you can simulate it), open Studio for it. The Publish slot is replaced by the coral lock + "UNLOCK" label, same as before. No regressions there — the lock-state version of the cell got the same chip-strip + icon-bump treatment so it stays visually consistent.
