Device QA — Wave 18.11

Purpose: three-point correction on Wave 18.10's expanded-body treatment after Carl's live review. (1) Swap recessed+coral-hairline for elevated neutral sub-card. 18.10 made the open accordion body darker (surfaceBg) with a 2pt coral left-edge stripe. Feedback: too much coral in the app already, and the asymmetric "only left edge has a stroke" read lopsided. Now the body sits on surfaceRaised with rounded 8pt corners and a thin neutral 1pt border at 6% white alpha — one elevation tier brighter than the outer card's surfaceBase @ radius 12, mirroring the card's grammar. Zero coral on the body; the existing coral chevron rotation + persistent summary already signal "this section is live". (2) Right-padding fix. 18.10's EdgeInsets.fromLTRB(12, 8, 4, 8) crammed right-aligned values (REPS / SETS / HOLD / PREP / DURATION PER REP / Muted toggle) against the container edge. Symmetric 12pt horizontal padding now gives them proper breathing room. (3) Inline-editor fill swap. InlineNumericEditor's fillColor was surfaceRaised — now camouflaged against the new surfaceRaised sub-card. Swapped to surfaceBase so the field reads as a darker recessed well. Section-header bottom padding also bumped 2 → 6 so the new visible sub-card top edge has breathing room from its header.

In scope: mobile-only. One file — studio_exercise_card.dart (_ExpandedBody rewrite + _GroupHeader bottom padding + InlineNumericEditor fillColor).

Out of scope: anything already covered by Wave 18 / 18.1 through 18.10. Web player. Portal. Font sizes / weights unchanged. Storage semantics unchanged. Chevron alignment + section-header top padding (Wave 18.10) unchanged.
