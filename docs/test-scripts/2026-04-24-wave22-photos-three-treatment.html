Device QA — Wave 22 photos three-treatment

Purpose: verify photos now flow through the same three-treatment pipeline as videos. A photo is just a video with one frame — the practitioner can pick Line / B&W / Original per exercise; the client can flip "Show me" globally; consent gates Colour + B&W just like videos.

What landed: mobile publish now ships TWO files per photo — line-drawing JPG to public media bucket (unchanged) AND raw colour JPG to private raw-archive bucket at {practiceId}/{planId}/{exerciseId}.jpg, gated on the client's video_consent.original. Supabase get_plan_full now branches on media_type: photos get the same grayscale_url + original_url keys videos do (signed; pointing at the .jpg). Web player buildMedia for photos routes through resolveTreatmentUrl, applies .is-grayscale when slide treatment is bw; CSS rule img.is-grayscale mirrors the existing video.is-grayscale. rebindVideoSources got a parallel photo-img loop so "Show me" flips hot-swap photo src + class instantly.

Surfaces touched: Supabase migration schema_wave22_photos_three_treatment.sql, mobile upload_service.dart (new photo loop in _uploadRawArchives) + api_client.dart (parametric contentType on uploadRawArchive), web-player app.js (buildMedia photo branch + rebindVideoSources photo loop) + styles.css (extended grayscale rule).

SW cache name: homefit-player-v51-photos-three-treatment (was v50-stack-polish). First Safari visit after install should show the new name.

What did NOT change: video three-treatment plumbing (segmented + mask sidecars stay video-only), the rep stack, soft trim, "Show me" override mechanics, consent flag wiring on the practitioner side, the practitioner Studio per-exercise treatment picker — all preserved.

Prerequisites

  • Latest ./install-device.sh (today). Build SHA on Home footer should match the Wave 22 commits.
  • Safari hard-reload on any plan URL so the service worker advances to homefit-player-v51-photos-three-treatment.
  • A test client (e.g. Lauren) with all three video consent flags GRANTED (Settings → Clients → toggle on Colour + B&W).

A · Mobile publish — raw photo lands in raw-archive

  • 1. Capturing + publishing a photo writes both files.
    1. Open the consent-granted client. New session. Camera mode → short-press to capture a photo. Wait for line-drawing conversion.
    2. Publish. Wait for the success snackbar.
    3. In Supabase Studio Storage → media bucket → confirm {plan_id}/{exercise_id}.jpg exists (the line-drawing JPG, public, has been there pre-Wave-22).
    4. Storage → raw-archive bucket → confirm {practice_id}/{plan_id}/{exercise_id}.jpg exists. Click it → expect a colour photo (the raw camera capture), NOT the line drawing.
    5. If the raw .jpg is missing in raw-archive, the client probably hasn't granted Colour consent — the upload skips silently in that case. Check the client's consent and re-publish.

C · Web player — "Show me" enables Colour + B&W on photos

  • 3. All three "Show me" segments enabled on a photo slide (with consent).
    1. Plan URL open in Safari. Tap the gear icon top-right → "Show me" segmented control appears.
    2. Navigate to the photo slide via the dot indicators or chevrons.
    3. Expect: Line · B&W · Colour all three segments are tappable (no lock glyph, no greyed pill). Active segment matches the practitioner's preferred_treatment for that photo.
    4. Tap B&W → expect: photo INSTANTLY desaturates in place (no reload flash, no spinner). The src does NOT change visibly — same image, just the CSS filter applied.
    5. Tap Colour → expect: full-colour raw photo appears (the actual camera capture, not the line drawing).
    6. Tap Line → expect: snaps back to the line-drawing JPG.

D · B&W is a CSS filter, not a separate file

  • 4. Confirm via inspector that B&W on photos is the same src as Colour, just with the .is-grayscale class.
    1. On the photo slide with "Show me" set to B&W, Web Inspector → Elements → find the <img> inside .card-media.
    2. Expect: class contains is-grayscale + data-treatment="bw".
    3. Toggle "Show me" → Colour. Re-inspect the same <img>.
    4. Expect: same src attribute (no change), data-treatment="original", .is-grayscale class removed.
    5. Computed styles → confirm filter: grayscale(1) contrast(1.05) applies in B&W and disappears in Colour. Same selector that videos use — single source of truth.
