2026-05-13 · web-player CI hygiene

SW cache name auto-bumps per deploy

web-player/sw.js CACHE_NAME now contains the deploy's 7-char git SHA, injected by build.sh. Every push that triggers a Vercel deploy ships a unique cache name → SW invalidates stale resources on next visit. No more manual unregister after CSP / app.js / api.js changes.

  • 1. After staging redeploys, visit staging.session.homefit.studio → DevTools → Application → Cache Storage → cache name reads homefit-player-<sha> (NOT homefit-player-__BUILD_SHA__).
  • 2. Trigger another deploy (any push to staging) → revisit the page → cache name updates to the new SHA.
  • 3. Existing SW registration continues to work after the cache-name change — no broken offline behaviour.
  • 4. After a deploy with a CSP / api.js change, hard-refresh the page → new content loads without manual SW unregister.
  • 5. Build log on Vercel shows sw.js CACHE_NAME → homefit-player-<sha> line — confirms the rewrite step ran.
  • 6. Local vercel dev with the script also produces a cache name with a real SHA (fallback to git rev-parse).
