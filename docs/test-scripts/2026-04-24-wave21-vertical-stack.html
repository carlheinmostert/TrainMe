Device QA — Wave 21 vertical rep-block stack

Purpose: verify the new vertical rep-block stack on the LEFT edge of the video, the trailing rest after the LAST set, and the updated total-duration math (every set followed by a rest).

What landed: the horizontal segmented progress bar at the top of the video viewport is gone. In its place, a vertical column on the LEFT shows one micro-block per rep + one thinner block per rest, stacked bottom-up: Set 1 (bottom) reps fill upward, then a rest block, then Set 2, etc. Section labels (S1, R, S2, R, …) sit in a left gutter outside the column. Rep fill is discrete (one rep per video loop seam, 200ms ease-in) — no time-based drift. Rest fill is time-based, sage, bottom-up over the rest duration. The active block (next-to-land) carries a 1Hz pulse on its outline.

State-machine update: advanceSetPhase no longer short-circuits on the last set — every set is now followed by a rest. calculateDuration updated: total = sets × perSet + sets × breather (was (sets − 1) × breather). A sets=3 / breather=15 / per-set=10 exercise totals 75s now (was 60s).

Hidden when: rest slide, photo, single-set + no breather, or legacy slide with reps null/0 (cleanest fallback per Carl — no fake-block fabrication).

SW cache name: homefit-player-v45-vertical-rep-stack (was v44-soft-trim). PLAYER_VERSION bumped in lockstep — both must match.

Punted: nothing. The stack is fully wired across mobile preview (WebView wrapper) + Safari since both surfaces share the same web-player/ bundle.
