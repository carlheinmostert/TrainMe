Wave 17 · web player QA

Analytics consent banner + event emitters + transparency page

Web player only. Open a published plan URL in Safari / Chrome (not the mobile app preview). The plan must belong to a client with analytics_allowed not disabled (default ON for all clients). Clear localStorage for the plan domain before testing to get a fresh consent state. 15 items in 4 sections.

A · Consent banner (4 items)

  • 1. Banner shows on first open of a plan URL.
    1. Clear localStorage for session.homefit.studio (DevTools → Application → Local Storage → Clear).
    2. Open a published plan URL. After the plan renders, a coral-accented banner slides in from the top.
    3. Banner text reads: "Help {TrainerName} help you. We'll share which exercises you complete, and when. Nothing else. You can stop this anytime."
    4. Two buttons: "No thanks" (outline) and "Yes, share" (coral filled). Below them, a subtle "What's shared? →" link.
  • 2. Banner remembers "Yes, share" and does not re-prompt.
    1. Tap "Yes, share". Banner slides away.
    2. Reload the page (hard refresh). The banner does NOT reappear.
    3. Check localStorage: key homefit-analytics-consent-{planId} should be "yes".
  • 3. Banner remembers "No thanks" and does not re-prompt.
    1. Clear localStorage again. Reload. Banner appears.
    2. Tap "No thanks". Banner slides away.
    3. Reload. No banner. localStorage key is "no".
  • 4. "What's shared?" link opens the transparency page in a new tab.
    1. Clear localStorage. Reload. When banner shows, click the "What's shared? →" link.
    2. A new tab opens at /what-we-share?p={planId}.
    3. The banner remains on the original tab (not dismissed).

B · Event emitters (6 items)

  • 5. plan_opened fires on load (even without consent).
    1. Clear localStorage. Open the plan URL. Watch the Network tab for a POST to log_analytics_event.
    2. Before tapping any consent button, you should see a plan_opened event posted.
    3. The payload includes event_kind: "plan_opened" and optionally a referrer field.
  • 6. exercise_completed fires when an exercise timer finishes.
    1. Accept consent. Start Workout. Let the first exercise run to completion (timer reaches 0).
    2. In Network tab, a POST with event_kind: "exercise_completed" should appear.
    3. Payload includes watched_ms and threshold_met: true.
  • 7. treatment_switched fires when the "Show me" override changes.
    1. During a workout (or browsing), open the gear popover. Switch from "Auto" to "B&W" (if consent available) or "Colour".
    2. Network tab shows event_kind: "treatment_switched" with from and to fields.
  • 8. pause_tapped and resume_tapped fire on tap-to-pause/resume.
    1. During a workout, tap the video area to pause. Network tab shows pause_tapped with elapsed_ms.
    2. Tap again to resume. Network tab shows resume_tapped with pause_duration_ms.
  • 9. rest_shortened fires when swiping past a rest slide early.
    1. During a workout, wait for a rest slide. Before its timer finishes, swipe to next.
    2. Network tab shows rest_shortened with scheduled_ms and actual_ms (actual < scheduled).
  • 10. plan_closed fires on navigate-away / tab close.
    1. With consent accepted, navigate away from the plan page (type a new URL or close the tab).
    2. Check Supabase plan_analytics_events table: a plan_closed row should exist with elapsed_ms and slide_index_at_close.
    3. (This uses fetch + keepalive so it survives page teardown.)

C · Completion CTA (2 items)

  • 11. CTA appears on the workout-complete overlay after the last exercise.
    1. Accept consent. Start Workout. Let all exercises run through to the end.
    2. The "Workout Complete!" overlay shows. Below the total time, a coral-bordered link reads "See what's been shared with {TrainerName} →".
    3. Below that, the existing "Close" button.
  • 12. CTA links to /what-we-share?p={planId}.
    1. On the completion overlay, click the CTA link.
    2. New tab opens at session.homefit.studio/what-we-share?p={planId}.
    3. The transparency page loads with the practitioner's name in the heading.

D · Transparency page (3 items)

  • 13. Generic variant renders at /what-we-share (no query param).
    1. Navigate directly to session.homefit.studio/what-we-share.
    2. Page renders with dark theme. Title: "What homefit.studio shares with your practitioner."
    3. Four sections visible: What we collect, What we don't, How long, How to stop.
    4. "Stop sharing" button is disabled. Helper text: "Open this page from inside your exercise plan to stop sharing."
  • 14. Contextual variant renders with practitioner name at /what-we-share?p={planId}.
    1. Open a plan URL first (so a session ID lands in localStorage). Then navigate to /what-we-share?p={planId}.
    2. Title reads: "What {practitionerName} at {practiceName} sees about your exercises."
    3. "Stop sharing" button is enabled (not greyed out).
  • 15. Stop-sharing button works and shows confirmation.
    1. On the contextual variant, click "Stop sharing for this plan".
    2. Button disappears. Green confirmation shows: "Stopped. {practitionerName} won't see new data from this plan."
    3. Return to the plan URL. Reload. No new analytics events should fire (localStorage consent key flipped to "no").
  • 16. Stop-sharing button on contextual /what-we-share?p={planId} is enabled and fires revokeAnalyticsConsent.
    1. Open a plan URL, accept the consent banner ("Yes, share").
    2. Open /what-we-share?p={planId} in the same browser (new tab or click the "What's shared?" link).
    3. The "Stop sharing for this plan" button is enabled (not greyed out), hint text is gone.
    4. Click the button. It shows "Stopping..." then green confirmation: "Stopped. {name} won't see new data from this plan."
  • 17. Tapping a progress pill to jump forward generates an exercise_navigation_jump event.
    1. Open a plan with 3+ exercises. Accept analytics consent.
    2. Start the workout. Let the first exercise play briefly.
    3. Tap the 3rd pill in the progress matrix to jump forward.
    4. Verify in Supabase (plan_analytics_events) that an exercise_navigation_jump row appeared with from_slide, to_slide, and method: "pill" in event_data.
    5. Tap a pill behind the current slide to jump backward. Verify another event row.
