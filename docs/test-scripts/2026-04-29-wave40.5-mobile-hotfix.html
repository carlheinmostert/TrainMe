Wave 40.5 - Mobile hotfix

Haptics + ultrawide + photo refresh

Four fixes from Wave 40.4 device QA feedback. A Publish icon whiten. B Native haptics via platform channel (bypasses iOS AVCaptureSession suppression). C 0.5x ultrawide via CameraDescription swap. D Photo-as-last-exercise conversion UI refresh regression.

A - Publish icon white

A1 - Publish icon renders white
Open Studio for any unpublished session with exercises. The publish (cloud-upload) icon in the bottom toolbar should be white (matching camera/preview/share icons), NOT coral. The lock-state Unlock padlock icon CAN still be coral (it is a state cue). Verify both the active (publishable) and dim (no exercises / already published clean) states render in white tones.

B - Native haptics (camera active)

B1 - Press-hold start haptic
Open Camera mode. Press and hold the shutter button. You should feel a heavy thud haptic the instant video recording starts (even while the camera mic is hot). This is the critical fix -- Wave 40.4 reported no haptic at all during recording.
Pending
B2 - Per-second tick haptic
While recording video, feel for a heavy haptic tick every second in sync with the pulsing red dot. Should be unmistakable even with the mic recording. Count at least 5 ticks.
Pending
B3 - Lock-engage haptic
While recording, slide finger up onto the lock target (the round lock icon above the shutter). Feel for a light haptic when entering the armed zone, then release your finger while over the lock target. You should feel a heavy thud confirming lock-engage. The shutter inner morphs to a white square.
Pending
B4 - Stop recording haptic
Stop a recording (either release finger for non-locked, or tap the white square for locked). Feel for a light tap confirming the stop. For the 30s auto-stop: let a recording run to 30s -- feel for a double heavy thud (~120ms apart).

C - 0.5x ultrawide lens

C1 - Tap 0.5x pill
Open Camera mode. The vertical lens pills on the right edge should include 0.5x (only if the device has an ultrawide camera -- iPhone 11+ with dual/triple rear cameras). Tap the 0.5x pill. The viewfinder should switch to the ultrawide field of view (noticeably wider). The 0.5x pill should be highlighted (white background, black text).
Pending
C2 - Pinch crossover
While on the 0.5x ultrawide, pinch outward to zoom. When the zoom passes ~0.7x the viewfinder should seamlessly switch back to the main wide camera at 1x. Similarly, from 1x, pinch inward past ~0.7x and it should switch to ultrawide. The transition should be smooth (brief re-init is acceptable).
Pending
C3 - Switch back to 1x/2x/3x
While on 0.5x ultrawide, tap the 1x pill. The viewfinder should switch back to the main wide camera. The 1x pill should highlight. Verify 2x and 3x also work after returning from ultrawide. The 0.5x pill should NOT appear during recording (lens switching mid-clip is disabled).

D - Photo last-exercise refresh

D1 - Photo conversion updates Studio card
Create a new session. Capture a single photo (tap shutter). Swipe left to Studio. The photo card should show the raw thumbnail initially. Wait a few seconds for line-drawing conversion to complete. The card's thumbnail should update in-place to the converted line-drawing version without requiring any interaction (no scroll, no tap, no navigation). The conversion spinner (if visible) should clear. This is the regression fix -- previously the last exercise's UI did not update after conversion.
