Device QA — Studio hotfix (4-bug rollup)

CRITICAL. TestFlight v1 build SHA f6f7bce shipped with the back chevron + edge-swipe both non-responsive in Studio mode. This hotfix re-enables both, drops the CAPS labels 30% so they fit, and routes the ADJUST cell at the editor sheet (not the playback canvas).

Scope: four bugs in one PR.

1. Studio AppBar back chevron used maybePop() which respected the shell's PopScope(canPop:false) and silently no-op'd. Switched to explicit pop() which bypasses canPop.

2. The shell's PopScope(canPop:false) wrapper was a defence against an iOS edge-swipe vs PageView page-paging conflict on the Capture page. Re-enabled iOS edge-swipe-to-pop now that the AppBar exists; if the page-paging conflict re-surfaces in the wild, we'll add per-page conditional logic.

3. CAPS toolbar labels were 10pt and pinching their cells. Dropped 30% to 7pt + wrapped each Text in FittedBox(fit: BoxFit.scaleDown) as belt-and-braces for unusual font scaling. Sub-cell glyph circle (36pt) is unchanged.

4. ADJUST cell on the CAPS toolbar was opening MediaViewerBody (full-screen playback canvas) instead of the per-exercise editor sheet. Now mirrors the card-tap path: opens showExerciseEditorSheet(...) with 4 tabs (Plan / Notes / Defaults / Preview) and the onExerciseChanged callback wires through to _updateExercise for Studio refresh.

Prerequisites

  • PR for branch hotfix/studio-back-swipe-caps-adjust merged + ./install-device.sh from repo root.
  • One client with at least one non-rest exercise (for items 5, 6, 7) and one client with no exercises (for item 7).

1 · Studio back navigation (Bug 1)

  • 1. Open a session in Studio. Tap the back chevron in the top-left of the Studio AppBar (the labelled "← Carl" leading chevron). The shell pops back to the ClientSessionsScreen for that client. The back button now WORKS.

2 · iOS edge-swipe-to-pop (Bug 2)

  • 2. From Studio, drag from the very left edge of the screen rightward (iOS-native edge-swipe gesture). The shell pops back to the ClientSessionsScreen. The iOS edge-swipe is RE-ENABLED.
  • 3. From Studio, swipe horizontally LEFT-to-right OR RIGHT-to-left within the body of the screen (not from the edge) to page between Studio and Camera in the shell PageView. Page-paging still works — the page-paging gesture wasn't broken by re-enabling edge-swipe. (Re-test: Camera → Studio swipe still pages within the shell, doesn't pop the shell.)

3 · CAPS labels fit (Bug 3)

  • 4. Inspect the CAPS toolbar. The five labels — CAPTURE, ADJUST, PREVIEW, PUBLISH, SHARE — all FIT inside their cells without overflow. No ellipsis visible at default iPhone text scale. Glyph circle above each label is unchanged in size (36pt). PUBLISH (longest label) reads cleanly.

4 · ADJUST opens editor sheet (Bug 4)

  • 5. Open a session with at least one non-rest exercise. Tap the ADJUST cell on the CAPS toolbar. The bottom-sheet editor opens with 4 tabs (Plan / Notes / Defaults / Preview) — the SAME sheet that opens when tapping an exercise card body. NOT a full-screen media viewer.
  • 6. Inside the ADJUST-opened editor, change a value (e.g. bump reps from 10 to 12) and dismiss the sheet. Studio refreshes and the card now reflects the new value — parity with the card-tap path.
  • 7. Open a session with NO exercises (or only rest periods). The ADJUST cell renders dim and tapping it does nothing (no-op). Confirms the dim-state gate is intact.
