2026-05-14 · PDF export tainted-canvas hotfix (cloud surface)

Hotfix QA — PDF export pre-fetches cross-origin hero images

Symptom: tapping Share on the lobby of a published plan at staging.session.homefit.studio opened the export modal, then sat forever on "Rendering page 1 of 2…". Embedded preview (Flutter workflow Preview tab) worked fine because it serves images same-origin via homefit-local://.

Root cause: hero <img> tags on the cloud surface point at Supabase signed URLs (cross-origin to the page). PR #325 already pre-fetches and swaps to data: URLs, but the preload fetch() had no timeout — a stalled response left it pending forever, so the spinner never advanced. html2canvas also had no internal timeout, so a partially-swapped DOM could hang its image loader on a leftover signed URL.

Fix: (1) AbortController 8s budget per preload fetch — failures land in preloadErrors instead of hanging forever. (2) onclone backstop in html2canvas — re-walks the cloned DOM, swaps any unmapped <img> src to its data URL, or strips the src to prevent useCORS-triggered fetches on a leftover Supabase URL. (3) imageTimeout: 4000 inside html2canvas + a 15s Promise.race ceiling on the whole rasterise call. (4) Per-page img.decode() wrapped in a 2s race guard matching the existing initial-decode pattern. (5) Hard early-fail when dataUrlMap.size === 0 && preloadErrors.length > 0 — surfaces the error modal with Retry instead of letting the pipeline drift into the page loop.

Gotcha memory: ~/.claude/…/memory/gotcha_html2canvas_cors_taint.md ("Adding crossorigin=anonymous doesn't help on already-loaded pages — pre-fetch via fetch+blob, swap in onclone").

Out of scope: the lobby gear popover settings unification (parallel agent). Studio card layout. Any other lobby pipeline.
