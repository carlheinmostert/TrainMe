Item 5 (photo filmstrip softness) is intentionally NOT in this script. Investigation: app/lib/widgets/session_card.dart line 990 — photo filmstrip cells use ex.displayFilePath which is the converted line-drawing JPG. That path doesn't read preferred_treatment or body_focus, so PR #309 cannot have caused it. Most likely root cause: Image.file's cacheWidth: 240 at line 951 is sized for a 4-cell strip (~97px wide each); a single full-width cell upscales 4.8× and reads soft. Stacked for a follow-up that needs a cacheWidth scaled to the cell's actual width.
