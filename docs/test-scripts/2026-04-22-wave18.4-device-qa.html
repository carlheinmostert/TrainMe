Device QA — Wave 18.4 patch

Purpose: two polish items from live device feedback on Wave 18.3. Delta-only — this script validates ONLY the items that changed in 18.4. Items covered: (1) the [Custom] text pill has been replaced with a [+] icon in a 32×32 dashed coral/sage circle (saves ~40pt per chip row, so a full canonical + a few customs fit one line on iPhone); inter-chip spacing tightened 6pt → 4pt; (2) section header labels (PLAYBACK / PLAN / PACING / NOTES) bumped 11pt → 13pt so the label reads clearly against the 13-14pt body content; summary stays 11pt to preserve hierarchy.

In scope: mobile-only (no R-10 mirror). Preset chip row (preset_chip_row.dart) + group header inside the studio exercise card (studio_exercise_card.dart).

Out of scope: everything covered by Wave 18 / 18.1 / 18.2 / 18.3. Web player. Portal.

Prerequisites

  • Wave 18.3 merged to main (SHA 0a269c6). Wave 18.4 branch fix/wave-18.4-polish installed via ./install-device.sh.
  • At least one session with: (a) a rest period; (b) one exercise with a non-default PACING (e.g. 8s prep override or a custom duration); (c) one exercise with a long NOTES entry (40+ chars) so the NOTES header wraps in collapsed state.
  • At least one custom rest value already saved from prior QA (tap [+] on the rest row and add e.g. 20). If the rest chip list is empty of customs, add 1-2 before running items 2 + 3.

1 · Rest chip row — 4 canonicals + [+] on one line

  • 1. Open a session, find a rest bar with no custom rest values stored yet. The chip row renders [15s] [30s] [60s] [90s] + [+] all on ONE horizontal line. No wrap, no clipping. The [+] tail is visibly a compact round icon — not the old ~70pt text pill reading "Custom".
  • 2. Tap [+] on the rest row, enter 20, Done. Then tap [+] again, enter 45, Done. You now have 4 canonicals + 2 customs + [+]. All seven chips still fit on ONE horizontal line on iPhone (no wrap). This is the main win of the Wave 18.4 icon swap — pre-18.4 the text-pill took enough horizontal room that 1 custom already pushed to a second row.
  • 3. Keep adding customs (e.g. 25, 75, 120, 180) until the chips stop fitting on one row. The row now wraps cleanly to a second line. All chips remain reachable; none clip off the right edge. Wave 18.3's wrap behaviour is preserved.

2 · [+] tail — dashed circle + inline input

  • 4. Inspect the [+] tail. It renders as a 32×32pt dashed coral circle (sage on the rest row) with a clean + glyph centred inside. Dashed outline, not solid. Same visual weight as a canonical chip (32pt tall) so they sit flush on the row.
  • 5. Tap [+] → inline input row opens (Cancel · numeric text field · Done). No bottom sheet, no modal. Haptic selectionClick fires on tap. Enter a value + Done → commits + row re-renders with the new chip. Cancel → restores the chip row unchanged.
  • 6. Long-press [+] — it's a no-op. No haptic, no action, no input opens, no undo SnackBar. Compare with long-press on any value chip (e.g. [15s]) which DOES fire selectionClick + removes the chip + shows an undo SnackBar. The tail is NOT removable because it isn't a value.
  • 7. Expand PLAN on any exercise card. The Reps / Sets / Hold chip rows each end with a coral [+] circle — same 32×32 shape, dashed outline, same + glyph. NOT a text pill reading "Custom". Tap works identically (opens inline input); long-press is still a no-op.

3 · Inter-chip spacing tightened 6pt → 4pt

  • 8. On the rest bar (non-scrollable Wrap variant) the gap between chips is visibly tighter than Wave 18.3. 4pt instead of 6pt — subtle but discernible when you look at the gap between each pill. Vertical run-spacing (when the row wraps to a second line) is also 4pt.
  • 9. On the PLAN rows (scrollable ListView variant — Reps / Sets / Hold) the gap between adjacent chips is also 4pt. Chips sit closer together than in Wave 18.3. Horizontal scroll still engages if the row outgrows the card width (unchanged from 18.3).

4 · Section headers — 13pt label

  • 10. Collapse all four group headers (PLAYBACK / PLAN / PACING / NOTES). Each label renders at 13pt uppercase coral w700 — visibly larger than Wave 18.3's 11pt. Letter-spacing stays at 0.8. Compare with screenshots of 18.3: the label is now clearly legible at normal arm's-length reading distance, no longer "diminutive" against the 13-14pt body content.
  • 11. On a collapsed header row with a non-default summary (e.g. PACING · 8s prep), the label (PACING) reads at 13pt coral w700 — visibly larger + heavier than the summary text (8s prep) which stays at 11pt w500 secondary-grey. The · separator sits between them. Label clearly reads as the emphasis; summary as supporting.
  • 12. Expand PACING. The label stays at 13pt in the expanded state too — no regression back to 11pt. It no longer reads as "diminutive" against the row of body controls (PrepSecondsRow, toggles, etc.) directly below. The summary still persists with single-line ellipsis on the same header row, rendered at 11pt.
  • 13. On a non-default row the trailing 5pt coral dot stays aligned to the first-line label baseline even after the label's taller line-box. Put a thumb across the row — chevron left, label+summary middle, dot right, all sitting on the same visual centreline. No drift, no floating-above-the-text, no clipped dot.
