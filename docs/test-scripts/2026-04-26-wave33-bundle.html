  • 8. Console.app capture: every availableCameras() entry is logged.
    1. Plug iPhone into Mac. Open Console.app. Filter on subsystem avatar.capture.
    2. Tap empty avatar slot on a client. Console fills with diagnostic lines.
    3. Expect: availableCameras() returned N entries with each one's name + lens + sensorOrientation.
    4. Expect: native getPreferredBackCameraName: name=... uniqueID=... availableTypes=[...] — if this doesn't appear, the native shim isn't wired and the substring fallback is in play.
    5. Expect: avatar capture picked camera: name=... matchedNativeCanonical=true — this is the one to look at. matchedNativeCanonical=false means the plugin's enumerated camera names don't agree with AVFoundation's canonical wide-angle, and we fell through to the substring filter.
    6. Expect: avatar capture zoom: min=... max=... applied=1.0 (uniqueIdHint=...).
    7. Copy the relevant lines into the note field if the lens is still wrong, OR check off pass if everything reads correctly + the capture itself looks good.
