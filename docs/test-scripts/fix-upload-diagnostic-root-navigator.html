2026-05-15 · modal-stacking fix · useRootNavigator

Device QA — Upload diagnostic sheet opens over publish progress sheet

Bug: on the failure state of the publish progress sheet, the “Show which files →” link rendered correctly but was non-responsive when tapped — nothing happened. The diagnostic sheet never opened.

Root cause: Flutter modal-stacking issue. UploadDiagnosticSheet.show() called showModalBottomSheet against the same Navigator that already had the PublishProgressSheet open. Pushing a new modal onto a navigator that already owns a modal is a no-op — Flutter silently rejects the push, the new sheet never appears, and the tap looks like a dead button.

Fix: single-line change in app/lib/widgets/upload_diagnostic_sheet.dart — add useRootNavigator: true to the showModalBottomSheet call so it pushes onto the ROOT navigator (which has no modal on it), letting the diagnostic sheet layer cleanly on top of the publish progress sheet.

Sensitivity: low — a single named parameter on a single call site. No behaviour changes outside the diagnostic-sheet open path.

Prerequisites

  • Build from fix/upload-diagnostic-sheet-root-navigator installed on Carl's iPhone via ./install-device.sh.
  • One fully-consented test client (B&W + Original ON) with a fresh session of 3+ video captures.
  • Wi-Fi reachable and Control Center one-tap away so Airplane mode can be flipped mid-upload.
