Inter-set rest in Preview + Studio chevron + photo segmentation no-bands + Home avatar + badge font −10%

Five mixed-surface follow-ups. (A) The rep-block stack in the in-app Preview was missing the rest blocks between sets for any pre-Milestone-Q exercise (i.e. those captured before 2026-04-23 when withPersistenceDefaults() started seeding 15s). The bridge passed interSetRestSeconds raw; web-player/app.js treats null as 0 and the stack collapses sets together. One-liner fix at the local_player_server boundary coerces null → 15s so Preview matches the canonical default. Cloud RPC replace_plan_exercises already includes the column (verified live) — no migration. (B) Studio exercise card now has a chevron at the top-right of its header. Right-pointing › collapsed → down-pointing — rotates 90° clockwise on expand, 200ms ease, coral when active. Whole header row stays the toggle target (R-04 thumb-reach). (C) Photo body-focus output had ~5 horizontal bands behind the subject on portrait (9:16) photos. Hypothesis was stride / in-place-blur. Root cause: second tent-convolve pass was reading and writing plate in-place without a temp buffer, which vImageTentConvolve_ARGB8888 does not reliably support — bands are half-updated rows. Fix: split into a true ping-pong with a separate scratch buffer; pass1 source→plate, pass2 plate→scratch, composite reads scratch. Same kernel + edge mode. Diagnostic os_log lines added at category:avatar.capture for input dim, stride, kernel, mask state — if bands persist Carl can capture under Console.app and we have data. (D) Home client cards now render the body-focus avatar PNG when the client has one. Same ClientAvatarGlyph three-tier fallback (local PNG → signed URL → initials). Cards without an avatar look exactly as before (coral person glyph). (E) Session-card count badge: font −10% from W36's 13/10 to 12/9. Container stays 21×21.
