2026-05-04 · web-player lobby fixes (round 1)

Device QA — lobby Carl's-round-1 fixes

Scope: follow-up to PR 231 (lobby surface). Carl's iPhone QA on the live session.homefit.studio lobby surfaced 7 issues; this PR fixes them in the shared web-player/ bundle (so mobile workflow Preview gets the same fixes for free).

What landed:
1. Row index numerals (1. 2. 3. …) on each non-rest row — circuits stay grouped, numbering goes through their rows once per circuit.
2. Pill highlight: nearest-to-centre reducer (rAF-throttled scroll) replaces the IntersectionObserver. First non-rest row defaults active at mount so the matrix is never unhighlighted at scroll-top.
3. Pill tap accuracy: lobby pills bumped to 36px tall (was 20px), 24px min-width per column, 6px gap between pills, plus touch-action: manipulation. Click delegation already targets root .pill[data-slide] via closest().
4. Treatment override propagation: applyTreatmentOverrideToAllExercises now ALWAYS writes the global pick on every exercise — the prior "clear when matches practitioner default" branch was the bug (a per-exercise B&W default would survive a global Line tap, leaving that row in B&W).
5. Line treatment animates: switched from static <img> JPG thumbnail to <video> with line_drawing_url (the actual line-drawing video). Same lazy-load gating as B&W/Colour.
6. Circuit visual matches Studio: brand-tint background container with header row carrying a coral mini-rail + uppercase practitioner-set name (plan.circuit_names[circuit_id]) + ×N cycles chip on the right. Falls back to "Circuit" when no name is set.
7. Drop synthesised breather rows: the lobby was rendering 3 "Breather" rows for ONE plan-level rest because circuit unroll multiplies a rest into N rounds. Fixed by deduping rest slides by exercise id; rename label "Breather" → "Rest" (per-set inter-set rests already surface inside the dose-line as "30s rest").

SW cache bumped: homefit-player-v35-lobby → homefit-player-v36-lobby-fixes.

Prerequisites

  • This PR merged to main + Vercel auto-deploy of session.homefit.studio live.
  • Hard-refresh (kill + reopen Safari) on iPhone CHM so the new SW takes over.
  • Test plan: ideally one with 6-10 exercises, ONE plan-level rest period (not inside a circuit), ONE 3-round circuit with 3+ exercises and a custom name set, plus at least one photo exercise. A mix of practitioner-default treatments helps test fix 4.

Fix 1 · Row index numerals

  • 1. Standalone exercises numbered 1, 2, 3, … Open the lobby. Expected: each non-rest row shows a small numeral (1. 2. 3.) in Inter 14, secondary colour, baseline-aligned to the left of the exercise name. Names align across rows (number column has consistent width).
  • 2. Circuit rows numbered through grouping. A 4-exercise circuit positioned after 3 standalone exercises shows numbers 4. 5. 6. 7. on its 4 rows — regardless of how many rounds the circuit repeats (the matrix conveys round count). Numbering continues 8., 9., … on rows after the circuit.
  • 3. Rest rows have no numeral. Plan-level rest rows render as a non-numbered sage strip (item 14 confirms label). Numbering on the next non-rest row continues without skipping.

Fix 2 · Pill highlight reducer

  • 4. First row's pill highlights at scroll-top. Open the lobby — before scrolling, the first non-rest exercise's pill should already show the active highlight. The corresponding row also has the subtle coral border. Bug it replaces: the prior IntersectionObserver-with-rootMargin coupling left every pill unhighlighted until you scrolled past row 1.
  • 5. No flicker mid-scroll. Scroll the lobby slowly. Expected: the active pill flips smoothly to the next row when its centre crosses the viewport centre — no toggling between two adjacent pills. Scroll back up and the active pill flips smoothly the other way. Drives: the new "row whose centre is closest to viewport centre" reducer; rAF-throttled so 60fps scrolls don't queue handlers.
  • 6. Active pill auto-centres horizontally. Scroll to a row whose pill is far right in the matrix. Expected: the matrix scrolls horizontally so the active pill centres in the visible matrix area.

Fix 3 · Pill tap accuracy

  • 7. Tap pill scrolls list to the matched row — no neighbour mis-fires. Tap several pills in succession (cherry-pick: ones near the edges + narrow circuit pills). Expected: each tap scrolls the hero list to centre the matching row; no taps land on the neighbouring pill. The active pill highlight follows the scroll. Pills are now ~36px tall (up from 20px) and ≥ 24px wide — finger-tap-able.
  • 8. Long-plan matrix scrolls horizontally instead of squashing pills. Open a plan with many exercises (10+ slides). Expected: the matrix lets you swipe left/right to see all pills; individual pills retain their min-width (24px). No pills below tap accuracy.

Fix 4 · Treatment override propagation

  • 9. Global Line tap forces every row to Line — even practitioner-default-B&W rows. On a plan where the practitioner has set ANY exercise's preferred treatment to B&W (swipe-to-set in Studio), open the lobby and tap Line. Expected: EVERY hero now plays the line-drawing video. No row is left in B&W. Bug it replaces: the prior clear-on-match logic deleted the override when it equalled the practitioner default, leaving that row to fall back to its B&W default.
  • 10. Same logic for B&W and Colour. Tap B&W — every row goes grayscale (CSS filter on Line video frames OR grayscale_url for B&W when consented). Tap Colour — every row goes full colour. No row is left on a stale practitioner-default treatment.

Fix 5 · Line treatment animates

  • 11. Line treatment plays the line-drawing VIDEO (not the JPG thumbnail). With Line selected, scroll to a video exercise. Expected: the hero loops through the line-drawing video (motion). Off-screen rows pause; current ± 1 plays. Spec restores Q7 of the lobby spec: "Line treatment → loops the line-drawing video."
  • 12. Photos remain static stills under Line. A photo exercise (no video source) still renders a static <img> hero — no broken video element. Rest rows have no media at all.

Fix 6 · Circuit visual + named circuits

  • 13. Circuit container matches Studio chrome. A circuit renders inside a brand-tinted rounded container with a 36px header row up top: a coral mini-rail on the left, uppercase circuit name in the middle (Inter 14/600 with 0.5 letter-spacing), and a ×3 cycles chip on the right (brand-tint pill, JetBrainsMono 12). The contained exercise rows inherit a surface-base background to stand out from the brand-tint container.
  • 14. Custom circuit name renders. In Studio, set a circuit name (e.g. "Lower body"). Republish + open the lobby. Expected: the header reads LOWER BODY (uppercased) followed by the ×N chip. Unnamed circuits fall back to CIRCUIT.

Fix 7 · Plan-level rests only; "Rest" label

  • 15. One plan-level rest = ONE rest row. Open the test plan with one explicit plan-level rest (e.g. between two exercises, NOT inside a circuit). Expected: exactly one sage rest row in the lobby. Bug it replaces: the prior render iterated over the unrolled slides array, which multiplies an in-circuit rest by N rounds; one rest in a 3-round circuit was emitting 3 "Breather" rows.
  • 16. Rest row label reads "Rest · Ns". The label is "Rest" (not "Breather"), followed by " · " and the rest seconds (e.g. Rest · 30s). Per-set inter-set rests still surface inside the exercise dose-line as "30s rest" — they are not a separate row in the lobby.
