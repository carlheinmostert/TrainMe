Lobby hotfix round 3 — matrix backdrop (real fix), 2-col rail, default-name fallback, top-left chip, last-pill scroll-bottom

Five follow-ups Carl surfaced after PRs 235 + 236 landed on his iPhone. Same shared web-player/ bundle — mobile Workflow Preview WebView gets the fixes for free.

Fix A. Pill matrix STILL had a coral wash after PR 235. Root cause: PR 235 zeroed .lobby-matrix-inner .matrix-circuit { background: transparent } but the actual painter is .matrix-circuit-row::before (the per-row coral tint band rendered as a pseudo-element). Real fix: explicitly disable that pseudo in the lobby context with content: none; background: transparent; display: none.

Fix B. Circuit rail rebuilt around a real two-column gutter layout. Rail pieces are real <span> children of a fixed-width .lobby-row-gutter column, NOT pseudo-elements on the row. Carl's directive: “the vertical line must run outside the left edge of the exercise cards” — with the gutter at 22 px and the rail at left: 11px, the line sits just outside the card column's left edge. The active-row coral border (intentional UX) lives on the inner content card and now cleanly wraps just the card without covering the rail. Mockup at docs/design/mockups/lobby-circuit-header.html updated to match.

Fix C. Default-named exercises rendered as just “Exercise” (no number). Investigation: Studio's create-exercise factory at app/lib/models/exercise_capture.dart:416 doesn't set name at all — it arrives as null from get_plan_full. The mobile UI's Exercise N fallback only fires at display time (progress_pill_matrix.dart), never persists. Fix: in the lobby, when slide.name is null/empty, synthesise Exercise N from a position counter that increments per de-duped non-rest exercise (rests excluded; circuit-grouped). Custom-named exercises render their name as-is (no number).

Fix D. Build-marker chip moved out of the .lobby-meta strip into a fixed top-left corner overlay (position: absolute; top: 6px; left: 8px; z-index: 51) inside #lobby. Same content, same dim mono treatment. Hides automatically when Start is tapped (since the parent #lobby hides).

Fix E. Last exercise's pill never highlighted at scroll-bottom. The nearest-to-centre reducer never fired for the last row because at scrollTop ≈ maxScroll the last row's centre never reaches viewport centre. Special-case the scroll-bottom edge (within 4 px of max scroll): mark the last visible row active regardless of distance.

SW cache bumped: homefit-player-v38-version-chip-labels → homefit-player-v39-hotfix-round3. Build-marker chip in the lobby will read plan v{N} · web-player v39-hotfix-round3 · cache v39-hotfix-round3 once the new SW takes over.
