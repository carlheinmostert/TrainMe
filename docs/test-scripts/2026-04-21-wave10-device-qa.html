5 · Analytics — share_events writes

Every share action (Copy / Open-in-app / Download-PNG) appends to a new append-only table, indexed by practice_id, channel, occurred_at.

  • 11. Click each of the 3 template Copy buttons (WhatsApp 1:1, broadcast, email) + Open-in-app + Download-PNG + Copy image — query share_events via Supabase dashboard, rows appear within 5s with correct channel + action fields.
  • 12. Try to insert a share_events row as anon via the REST API — RLS blocks. Only authenticated with matching practice membership can write.
  • 13. Share telemetry surfaces somewhere on /network below the fold (or on /audit per design review) — count of shares per channel over last 30 days. Matches actual event rows.
  • 14. A failed share (e.g. clipboard permission denied) does NOT write a successful event row. Any error path that does log should write with an error flag, not a bare success.
