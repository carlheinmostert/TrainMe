2026-05-13 · Hero Resolver — Flutter Bundle 2a

Device QA — Exercise Hero Resolver Flutter adoption (Bundle 2a: PR 4 + PR 5)

Purpose: Bundle 2a of the photo-vs-video treatment audit migration. Adds app/lib/services/exercise_hero_resolver.dart — Flutter mirror of web-player/exercise_hero.js (Bundle 1). Single stateless function decides how each mobile surface (Studio card · ClientSessions filmstrip · MediaViewerBody Preview tab · camera peek) picks treatment-correct file + ColorFilter + body-focus caps + fallback chains.

Audit fixes in this PR:
· PR 4 — Flutter resolver adoption. Migrates MiniPreview (_treatmentFor, _videoPathFor, _PhotoFrame, _HeroFrameImage, _VideoFrame), CaptureThumbnail (_resolveSource, _buildImage), _FilmstripCell, and MediaViewerBody helpers (_hasArchive, _effectiveTreatmentFor, _sourcePathForTreatment) to call the shared resolver. Deletes 3 duplicate ColorFilter constants in favor of one canonical kHeroGrayscaleFilter (Rec.709 weights, matches web).
· PR 5 — F17 filmstrip rule change. _pickFilmstripHeroes changes from "videos first up to N; zero videos → first photo only (single cell)" to "videos first up to N; if zero videos, photos up to N". A photo-only session with 3 photos now renders 3 filmstrip cells instead of clamping to 1.

Stacked for Bundle 2b (parallel agent, do NOT regress on these but they don't need to be FIXED here):
· F21 (photo thumbnail variant pipeline — adds _thumb.jpg / _thumb_color.jpg / _thumb_line.jpg for photos to match the video pipeline). The resolver degrades gracefully when these files don't exist — photos still render via the current files-only chain (line → converted JPG, B&W/Original → raw JPG + ColorFilter).
· Cloud upload of photo thumbnail variants — backlog after F21.

R-10 parity: The Flutter resolver contract mirrors the JS resolver shape from Bundle 1. Field semantics align: caps.hasBodyFocus (videos only), caps.availableTreatments (line baseline + grayscale/original when raw exists), caps.treatmentLockedTo (null or Treatment.line when fallback applies). Surfaces: HeroSurface.studioCard, filmstrip, mediaViewer, peek.

In scope: Flutter app only. Resolver + 4 widgets (mini_preview, capture_thumbnail, session_card, studio_mode_screen) + filmstrip picker rule.

Out of scope: web-player/ bundle (shipped in Bundle 1 already; do NOT touch). conversion_service.dart / upload_service.dart / Swift native code / unified_preview_scheme_bridge.dart / SQLite schema (Bundle 2b territory — parallel agent running now).
