2026-04-24 · Wave 24 · Video reps per loop

Device QA — Wave 24 video reps per loop

Purpose: verify that the practitioner can record a video of N repetitions (default 3, was implicitly 1) and the player times the slide correctly. The manual DURATION PER REP editor is gone — per-rep time is now ALWAYS derived from video_duration ÷ video_reps_per_loop; per-set time = target_reps × per_rep. Single source of truth.

What landed: Supabase column exercises.video_reps_per_loop INT NULL (CHECK > 0) + replace_plan_exercises RPC pass-through. Mobile ExerciseCapture.videoRepsPerLoop field, persistence default 3 on fresh video captures (photos + rest skip). Capture screen has a new muted "Aim for ~3 reps per video" line beneath the existing coral "Hold for video" hint. Studio PACING accordion gains REPS IN VIDEO at the TOP (above PREP) on video exercises only; DURATION PER REP + the From-video / Manual toggle pair are GONE from the UI. Web-player calculatePerSetSeconds rewritten to derive from video_duration_ms / video_reps_per_loop. SW cache + PLAYER_VERSION bumped v52-stack-nav → v53-video-reps.

Backwards compatibility: rows with video_reps_per_loop = NULL are treated as 1 rep per loop on both surfaces — every pre-Wave-24 plan plays exactly as it did before. Legacy plans with a stored custom_duration_seconds override (e.g. position 0 video on Lauren plan 8f64489f-f712-4c1c-87b3-f958819cd2ae) still honour the override on read; the editor is just gone.

SW cache name: homefit-player-v53-video-reps (was v52-stack-nav). First Safari visit after install should show the new name.

Prerequisites

  • Latest ./install-device.sh (today). Build SHA on Home footer should match the Wave 24 commits.
  • Safari hard-reload on any plan URL so the service worker advances to homefit-player-v53-video-reps.
  • Have a fresh client to publish a brand-new plan against (so video_reps_per_loop writes through the new code path, not as null).

    2. Expected per-set: (6000 / 1000) / 3 × 10 = 20s.
    3. Open the plan URL in Safari. Tap Start Workout, then watch the timer chip on the active slide for that exercise. The set-phase countdown should start at ~20s (or whatever your math says) and tick down to 0 before the breather phase begins.
    4. If the timer chip starts at the WRONG number, suspect either (a) video_reps_per_loop is null on this row (legacy fallback to 1 rep / loop) or (b) the SW cache hasn't advanced (hard-reload Cmd+Shift+R).
