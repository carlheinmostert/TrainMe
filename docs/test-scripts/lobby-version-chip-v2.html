Device QA — plan version + relabel build-marker chip

Scope: the small dim line at the bottom of the lobby metadata strip. Two related tweaks:
1. Add the plan version as the first segment so a freshly-republished plan is distinguishable from a stale tab on the same URL. Reads plans.version from the loaded plan; drops out cleanly when null on legacy rows.
2. Relabel the existing two segments. The bundle gets the prefix web-player and the SW cache gets the prefix cache — both describe the whole web-player surface, not the lobby specifically.

Net format:
plan v3 · web-player v38-version-chip-labels · cache v38-version-chip-labels

Legacy plan (no version):
web-player v38-version-chip-labels · cache v38-version-chip-labels

Bumps: PLAYER_VERSION → v38-version-chip-labels; CACHE_NAME → homefit-player-v38-version-chip-labels.
