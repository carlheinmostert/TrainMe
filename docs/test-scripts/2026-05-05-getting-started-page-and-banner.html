Device QA — Getting started page + first-launch banner

Scope: new public route at manage.homefit.studio/getting-started (web-portal) plus a one-time coral banner at the top of Home in the trainer app that links to it. The banner shows once per device, opens the URL in Safari (external), and disappears once tapped or dismissed.

Why: a new practitioner who's just signed in needs prescriptive walk-through, not feature reference. Page mirrors the bottom workflow pill — Camera → Refine → Preview → Publish → Share — with a dedicated Refine section covering reps/sets, hold position, trim slider, Hero frame, notes, circuits, reorder.

What did NOT change: existing Home chrome, sync banner, client list, all of Studio. Banner sits BELOW any sync error so transient infrastructure errors take precedence.

Prerequisites

  • PR for branch claude/xenodochial-borg-0ec6ee merged + Vercel auto-deploy of manage.homefit.studio live (page section).
  • Run ./install-device.sh from the repo root after merge for the banner to ship to your iPhone.
  • Banner first-launch is gated by SharedPreferences flag getting_started_banner_seen_v1. To re-trigger it for testing: delete + reinstall the app, or use Settings → Reset state if available. Otherwise the banner only fires once per fresh install.

  • 14. Force-quit + relaunch the app. Home opens; banner stays gone. Persistence is correct — the SharedPrefs flag survived the relaunch.
  • 15. Sign out and back in (different account, or same). Banner stays gone — the flag is per-device, not per-user. Expected behaviour.
