Wave 38 · device QA

Studio bottom-toolbar refactor + SessionCard inline rename

Two tightly-coupled mobile refactors. (A) Studio AppBar retired. The session title, padlock chip, preview / publish actions, and the Wave 33 "First/Last opened" analytics row all left the top of the screen. They re-grouped at the bottom in a fixed three-row stack: stats → subtitle (`{date} · {client}`) → toolbar (back / preview / publish). Locked plans show a coral-bordered "🔒 Unlock (1 credit)" pill in place of Publish, right-aligned. Mid-grace and prepaid-unlocked plans both keep Publish — only the post-14-day-grace lock surfaces the pill. (B) SessionCard title is now inline-editable on the per-client detail page. Dashed grey underline at rest signals editability; tap drops into a TextField with autofocus + select-all; Enter or tap-outside fires rename_session through SyncService's offline queue and the new title persists immediately. Tapping the badge / version row / lock row / chevron still opens Studio (separate hit zones). Schema migration: supabase/schema_wave38_rename_session.sql — SECURITY DEFINER + practice-membership gated, mirrors rename_client.

A · Studio bottom toolbar

  • 1. Top of Studio is exercise list with ~12pt breath above first card — no AppBar.
    1. Open any session with at least one captured exercise. The status bar (time / signal) sits at the top of the safe area; immediately below is the first exercise card with a small (~12pt) breather.
    2. No back button, no title row, no padlock chip, no Preview / Publish icons at the top of the body. The whole top is just exercises.
    3. Scroll the list. The bottom toolbar STAYS pinned (no auto-hide on scroll; that was Carl's locked design choice).
  • 2. Bottom toolbar in normal state: back ‹ left, spacer, preview ▶, publish (coral filled circle) right.
    1. On a draft (never published) session: bottom stack shows ONLY the subtitle row + toolbar. No analytics strip above the subtitle. Three buttons total in the toolbar — back left, preview (eye / slideshow glyph), publish (coral cloud-upload circle).
    2. Tap each in turn. Back pops. Preview opens UnifiedPreviewScreen. Publish either fires upload (if you have credits + conversions are done) or surfaces the existing publish error / unlock-sheet path.
    3. All buttons are at least 44×44 hit targets — the coral publish circle reads as the most visually prominent affordance.
    4. On a published-and-clean session, the publish glyph mutates to a sage / circuit-coloured check (existing W18 behaviour, just relocated). On a published-and-dirty session, it's coral cloud-upload again.
  • 3. Locked plan (past 14d grace): unlock pill replaces publish, right-aligned.
    1. Find a session whose first_opened_at is more than 14 days ago and whose unlock_credit_prepaid_at is null. (Or fake one by manually backdating the SQLite row in the simulator with sqlite3.)
    2. Open Studio. Bottom toolbar: back ‹ left, preview ▶ next to it, then a spacer, then a coral-bordered "🔒 Unlock (1 credit)" stadium pill on the right. Publish glyph is GONE.
    3. Tap the unlock pill → existing unlock-for-edit bottom sheet opens (1 credit pre-pay). Confirm: card flips to the prepaid-unlocked state.
    4. Stats strip line 2 above the toolbar shows a red dot + bold "Republish costs 1 credit" while locked. After unlock it shows a light-coral dot + "Unlocked · republish free".
  • 4. Stats strip + subtitle ellipsise long content; drafts skip the stats strip.
    1. Open a published, opened session. Stats strip line 1: "First opened {date} · Last opened {date}" (or "First & last opened {date}" if same day). Line 2: coloured dot + lock state ("Free Edits · 11d 4h left", or amber/coral as the deadline approaches, or red+bold when past grace).
    2. Open a published-but-never-opened session. Stats strip line 1: "Not yet opened". No line 2 (no lock state to show until the client opens).
    3. Open a draft (version 0). Stats strip is HIDDEN entirely. Bottom stack shrinks by ~40pt. Just subtitle + toolbar.
    4. Subtitle row reads `{DD MMM YYYY} · {client name}`. With a long client name ("Lauren post-knee-op week 4"), the row stays single-line and ellipsises with "...". No wrap, no two-line breakage.

B · SessionCard inline rename

  • 5. Dashed underline visible at rest on every SessionCard title.
    1. Open any client with one or more sessions. Each row's title (e.g. "26 Apr 2026 11:09" or a previously-set custom title) has a faint dashed grey underline beneath the text. Reads as the same "edit me" affordance the client name in the header above uses.
    2. Lock-state dot row, version line, and chevron have NO dashed underline — only the title carries it.
  • 6. Tap title → edit mode + autofocus; Enter persists via RPC; reload shows new title.
    1. Tap directly on the title text. The card does NOT navigate to Studio. The title morphs into a coral-underlined TextField with the existing value selected; the keyboard slides up.
    2. Type a new name (e.g. "Lower body recovery"). Hit Enter on the keyboard.
    3. Field collapses back to text + dashed underline; new title shows immediately. Pull-to-refresh: still the new title (it persisted to local SQLite).
    4. Force-quit the app, relaunch, drill back to the same client. Title is still the new value — the offline queue flushed to plans.title on resume / reconnect.
    5. Edge case: rename to an empty / whitespace-only string. Field exits edit mode without firing the RPC; old title stays.
    6. Edge case: tap title → type something → tap somewhere ELSE on the screen (outside the keyboard, outside the card). The change auto-commits via onTapOutside.
  • 7. Tapping rest of card (badge / version row / lock row / chevron) still navigates to Studio.
    1. From the client detail list, tap the leading list-alt icon + count badge area. Studio opens for that session.
    2. Back out. Tap the version line ("v3 · 25 Apr") OR the lock-state row ("· Free Edits...") OR the chevron on the right. Each opens Studio.
    3. Back out again. Tap directly on the dashed-underlined title. ONLY the title cell drops into edit mode — doesn't navigate.
    4. While in edit mode, swiping left on the card does NOT trigger the swipe-to-delete — the Dismissible is paused so a sloppy swipe near the keyboard area can't soft-delete the row out from under you.

Out of scope — reminder

  • The StudioToolbar widget (top toolbar with Import / Preview / Publish / Share triangles) was retired entirely. Import is now reachable via the gutter rail's insertion tray, which still works. Empty-state copy was updated to point at swipe-right-to-Capture only.
  • Network share via the studio toolbar was retired. The NetworkShareSheet from W30 is still the canonical share path on published sessions.
  • Web player has nothing to update — this is a mobile-only refactor. R-10 parity is preserved (the practitioner side is "configuration"; the consumer-facing web player has no rename surface).
  • Portal does not get an inline session rename in this wave. The mobile is the source of truth for session names; the portal still shows them read-only.
