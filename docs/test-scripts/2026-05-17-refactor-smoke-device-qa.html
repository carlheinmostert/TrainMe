Why this script. Pure refactor — agent claimed zero behaviour change and CI green (24 tests passing, flutter analyze clean). But the publish flow is the load-bearing money path and Carl's feedback_sensitive_code_review_before_merge.md rule applies. This script verifies the four publish paths (fresh photo, fresh video, metadata-only republish, hero-drag republish) still behave identically before and after the refactor.

Walks in tandem with. 2026-05-16-thumbnail-republish.html (PR #376) and 2026-05-16-photo-thumb-bw.html (PR #377). If those scripts pass on this build, the refactor is transparent — the underlying behaviours they assert are unchanged by definition. This script is the explicit refactor-smoke layer on top.

Prerequisites

  • Staging tip b0a18fa installed on iPhone CHM via ./install-device.sh staging.
  • Signed in with the staging QA account; ≥2 credits available (this script publishes twice minimum).
  • A client (any consent profile) with at least one open session, OR willingness to capture a fresh one.

Fresh-publish smoke (slow-path)

  • 1. Fresh video publish. Open a fresh session, capture one short video (3-rep default fine). Wait for conversion to complete. Publish. The progress sheet should show N of N files reaching completion with a «Published ✓» toast. device
  • 2. Open the published plan URL in Safari. Switch through Line / B&W / Original treatments — all three render the video without broken-tile glyphs or 403s in the network tab. device
  • 3. Fresh photo publish. Add a photo to the same (or new) session. Publish. _thumb_bw.jpg uploads alongside _thumb_line.jpg (per PR #377). Reload Safari lobby → switch to B&W. The photo renders pre-baked greyscale — not the CSS-filter path. device

Republish flow (fast-path + hero-drag)

  • 4. Fast-path stays fast. Edit a non-thumb field on any exercise (bump reps from 3 to 4). DO NOT touch the Hero star or hero-crop drag. Republish. The progress sheet completes in < 3 seconds with the «metadata-only republish» fast-path — no thumb uploads in the file counter. device
  • 5. Hero-drag republish (PR #376 still works through refactor). Open the editor sheet on the video → Demo tab. Drag the Hero star to a visibly different frame. Republish. Wait for the toast. Reload Safari lobby — the card hero now reflects the new frame. (If this fails, the refactor broke the thumbnailsDirty override path.) device

Regression sweep

  • 6. Audit log on manage.homefit.studio shows a plan.published entry for each publish above with the right plan name, trainer email, and timestamp in browser-local time. No media_thumb_*_upload_failed kinds in any of the entries' detail. device
  • 7. Studio session list still loads, client switcher still works, embedded preview opens (workflow pill Camera → Preview). Smoke-grade. device
  • 8. flutter analyze + flutter test were clean prior to install (24 tests passing per CI on commit b0a18fa). pre-device
