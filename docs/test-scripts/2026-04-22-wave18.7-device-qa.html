2026-04-22 · Wave 18.7 — polish round 4

Device QA — Wave 18.7

Purpose: five items from Wave 18.6 live QA. (1) [+] structural fix — CustomPaint now wraps the padded Container so the dashed border paints on the full pill area and the 12pt horizontal padding lives INSIDE the border. Before, the border was drawn on the canvas inside the padding, so the pill hugged the 18pt icon with ~0 visible padding. Fix makes the pill finally read as a ~42pt-wide padded pill with visible room around a centred icon. (2) PREP row drops the · default suffix — dashed-underline affordance is the only signal needed; source label was noise. (3) DURATION PER REP redesigned to mirror PREP — label + dashed-underline value + source-toggle pair ([From video] [Manual]) below. Editing the value auto-flips source to Manual; tapping From video auto-fills with video length. Photos + video-without-probed-length hide the toggle entirely. Seeds Manual from the client's sticky custom_duration_per_rep default (or 5s). (4) Inline numeric editors (PREP + DURATION PER REP) got explicit < Cancel   [_N_]   Done > buttons — iOS numeric keyboard has no return key, so explicit affordances are the only reliable commit path. (5) Section headers bumped 13pt → 14pt to overcome coral-on-dark perceptual-contrast penalty. Inner labels + values + summaries all stay 13pt.

In scope: mobile-only. preset_chip_row.dart ([+] nesting swap) + studio_exercise_card.dart (PREP suffix drop, section header font bump, DURATION PER REP redesign, InlineNumericEditor widget) + studio_mode_screen.dart (sticky custom_duration_per_rep plumbing).

Out of scope: anything already covered by Wave 18 / 18.1 / 18.2 / 18.3 / 18.4 / 18.5 / 18.6. Web player. Portal. Storage semantics of customDurationSeconds unchanged (still TOTAL = per-rep × reps). Chip long-press remove/undo, [+] long-press no-op, inline (not modal) input — all unchanged. Sticky propagation for the other six existing fields (reps / sets / hold / audio / treatment / prep) — unchanged.

Prerequisites

  • Wave 18.6 merged (SHA d94b5cd). Wave 18.7 branch fix/wave-18.7-polish installed via ./install-device.sh.
  • At least one session with: (a) a video exercise WITH a probed duration (record a 3–5s clip); (b) a photo exercise; (c) ideally, a second session for the same client so sticky-seed propagation can be verified across sessions.
  • iPhone 17 Pro or similar 393pt-wide device.

1 · [+] padding structural fix

  • 1. Open any exercise card, expand DOSE. The [+] tail renders as a ~42pt-wide pill with visible 12pt padding between the dashed border and the centred + icon. Compare against Wave 18.6: the old pill appeared to have no padding (icon pressed against the border) even though the width was correct. Now the icon floats in the middle with air around it.
  • 2. Long-press [+] → no-op (no haptic, no action). Tap [+] → inline Cancel · numeric field · Done opens (NOT a modal, NOT a bottom sheet). Visual weight of the pill now matches the text chips [15], [12], [10] — same 32pt tall × 16pt radius family, just slightly narrower and dashed.

2 · Section header font bump (13pt → 14pt)

  • 3. Expand an exercise card. Section header DOSE (coral w700 uppercase) reads visibly at least the same size or slightly larger than the inner label REPS (grey w600 uppercase). No perceptual inversion. Wave 18.6 had both at 13pt and the coral looked smaller due to contrast penalty; the 1pt bump re-establishes the header as the primary read.
  • 4. Section header summary (text after the · separator — e.g. "DOSE · 10 reps · 3 sets · 5s hold") stays at 13pt — visibly smaller than the 14pt label. The row reads as label-then-summary cleanly, with the label dominating.
  • 5. Inner label tier is unchanged: REPS / SETS / HOLD / PREP / DURATION PER REP all render at 13pt grey w600 uppercase. Inner value tier is unchanged: 45, 30s, 5s all at 13pt mono w700 white.

3 · PREP row — no · default suffix

  • 6. Expand PACING on any fresh exercise (no PREP override). The row reads PREP on the left and 5s with dashed underline on the right. The text " · default" is NOT visible anywhere in the PREP row. The dashed-underline value alone is the "tap to edit" affordance.
  • 7. On a fresh (no-override) PREP row: tap 5s → inline editor opens with < Cancel   [_5_]   Done >. The editor field is pre-filled with the current displayed value (5, matching the default). Tap Cancel → editor closes, value stays 5s with dashed underline, no write happened. Tap 5s again, type 7, tap Done → row now reads 7s in white with dashed underline (override set). Coral non-default dot now lights on PACING row header.
  • 8. On the same PREP row (now at 7s): tap 7s, clear the field, type 5 (matches the global default), tap Done → row reverts to "no-override" state (muted colour, non-default dot gone). Committing a value identical to the global default clears the override — no "coincidence override" persists.

4 · DURATION PER REP — PREP-style redesign

  • 9. On a video exercise with probed duration, expand PACING. The DURATION PER REP row now renders as TWO rows: first row DURATION PER REP label (left) + {video-length}s value with dashed underline (right), matching the PREP row layout exactly. Second row: [ From video ] (coral-filled, selected) [ Manual ] (grey-filled, unselected). No more segmented-control-with-long-label from Wave 18.5/18.6.
  • 10. Tap [ Manual ] on the fresh video exercise → segment flips coral. The value above updates to the seed (5s by default, or the client's sticky custom_duration_per_rep if one was stored from a prior commit). [ From video ] flips to unselected. No haptic delay.
  • 11. Tap [ From video ] → segment flips coral. Value returns to the video length (e.g. 3s). [ Manual ] unselects. customDurationSeconds is now null; the effective duration falls back to video-length × reps at runtime.
  • 12. With [ From video ] currently selected: tap the value (dashed underline) directly. Inline editor opens < Cancel   [_N_]   Done >, pre-filled with the current video-length number. Type 7, tap Done → value updates to 7s AND the toggle auto-flips to [ Manual ] (no separate tap needed). Editing the value implies Manual.
  • 13. On a photo exercise, expand PACING. DURATION PER REP row renders WITHOUT the toggle pair — just the label on the left and the {N}s value with dashed underline on the right. Tap the value → inline editor opens. Commit a new value — persists. No mode indicator shown because the photo has only one source.
  • 14. On a fresh video exercise with [ From video ] active: the PACING row header shows NO coral non-default dot — "From video" is the implicit default state for a video-with-probed-length, not a practitioner override. After Item 12 (editing the value flipped to Manual), the coral dot now lights on PACING because customDurationSeconds is set.
  • 15. Capture a second video exercise for the same client, in a NEW session. The sticky custom_duration_per_rep default propagated forward. Bonus: if you tap [ Manual ] on a fresh video exercise BEFORE any Manual commit has ever landed for this client, the seed is 5.
