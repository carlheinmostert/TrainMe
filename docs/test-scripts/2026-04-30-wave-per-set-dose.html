Per-set PLAN wave · tabbed editor sheet · redesigned exercise card

Per-set PLAN table + tabbed bottom-sheet editor

Architectural shift. The exercise card collapsed from an inline accordion to a compact row (thumbnail + Plan button + Notes button + gear). All editing now happens in a four-tab bottom sheet (Plan / Notes / Preview / Settings) with two detents and drag-to-dismiss. PLAN itself moved from uniform reps×sets@hold to a per-set table with a new weight_kg column and per-set breather. Schema migration ran, mobile data layer rewrote, web-player rep-stack now renders per-set with weight chips + change-load indicators + end-of-exercise marker. 22 checks across 7 sections.

Known caveat · rest-period durations

  • The wave-1 schema migration dropped exercises.hold_seconds — which was carrying rest-period duration on cloud. The wave-2 fix added a fresh rest_seconds column, but cloud values for the 24 pre-existing rest exercises were lost. Mobile cache preserved them locally; on next publish they go back to cloud.
  • If a rest period plays back at 30 s when you remember setting it to something else, it's likely cloud-side data loss. Re-set the value → republish → cloud picks up.
