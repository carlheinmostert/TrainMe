2026-05-15 · web-player lobby gear popover CSS-override fix

Device QA — gear popover stays above gear in landscape on iOS

Why: PR #338 added repositionLobbySettingsPopover() JS that correctly computes a viewport-aware "open above the gear" position via position: fixed + top = gearRect.top − gap − popH. BUT a stale .lobby-settings-popover { position: absolute; bottom: calc(100% + 8px); right: 16px; } rule in styles.css won the cascade, anchoring the popover to the fixed CTA bar's containing block instead of the viewport. In landscape on iOS this pushed the popover BELOW the gear and off the bottom of the screen.

Fix: drop the conflicting position: absolute / bottom / right declarations from the base .lobby-settings-popover rule; switch to position: fixed so there's no FOUC; JS-set top + left are now the only positional rules in play.

Bumps: none — CSS-only change, no JS / bundle / cache slug movement.
