Storage bucket RLS policies recovery

Why this matters. The 2026-05-11 baseline migration was generated via pg_dump --schema=public, which silently DROPPED every storage.objects policy (storage is a separate schema). Since the cutover, every Supabase Branching-cloned DB (staging + every per-PR preview) has zero storage policies. Authenticated INSERTs to either bucket fail with 42501 RLS violation. This is the root cause of Carl's “0 of 6 uploaded” P0.

The migration. supabase/migrations/20260515135502_storage_bucket_policies_recovery.sql restores the 7 policies verbatim from supabase/archive/schema_milestone_c.sql (media bucket, 4 policies) + supabase/archive/schema_milestone_g_three_treatment.sql (raw-archive bucket, 3 policies). Wrapped in DO blocks with insufficient_privilege exception handling; each policy individually DROP-IF-EXISTS-then-CREATE so it's idempotent.
