2026-05-03 · Studio settings card · Phase A (UI-only)

Device QA — Studio settings card

Purpose: verify the new plan-settings sheet ships behind a gear icon on the Studio bottom toolbar (replacing the prior cloud-download icon). The sheet renders five sections per the locked mockup — Pacing, Display defaults, Web viewer, Plan actions, Plan info — and routes Save-to-Photos plus Soft-delete plan to the existing handlers.

Phase A scope: UI only. The persisted field (preferred_rest_interval_seconds) writes through; every other toggle / chip / stepper is in-memory state with TODO(phase-b) markers. Reopening the sheet resets in-memory fields to their defaults — that's expected, schema-half lands in Phase B.

Out of scope: persistence for the in-memory fields, web-player consumption of any Web-viewer toggle, Duplicate plan (renders disabled), web-portal mirror.

Prerequisites

  • Branch feat/studio-settings-card-ui installed via ./install-device.sh.
  • Carl's iPhone 00008150-001A31D40E88401C.
  • At least one client with at least one published session (so Plan info has real values).

1 · Toolbar entry-point + sheet open

The cloud-download icon is gone; a gear sits in its place. Tapping opens the sheet at ~85% detent.

  • 1. Open any client → tap into a session. Studio mode renders. The bottom toolbar's right edge shows a gear icon (settings_outlined). The old cloud-download icon is no longer present anywhere on the toolbar.
  • 2. Tap the gear. A bottom sheet rises to ~85% of screen height. Header reads Plan settings with a subtitle {DD Mon YYYY HH:MM} · client: {Name}. A drag handle sits above the header; an X icon top-right.
  • 3. Tap the X (or tap outside the sheet). The sheet dismisses with the standard modal-bottom-sheet slide-down animation. No haptic glitches; no flash of black.

2 · Pacing section

Rest-between-exercises persists to preferred_rest_interval_seconds; auto-insert threshold is in-memory only.

  • 4. In Pacing, the first row is Default rest between exercises. The stepper shows the current value (default 30s for fresh plans). Tap the + button — value bumps in 5s increments (30 → 35 → 40, etc). Tap minus — value drops by 5s. Floor 5s, ceiling 180s.
  • 5. Bump rest to 45s. Close the sheet → reopen. Value still shows 45s (persisted to SQLite via the existing _touchAndPush path). The session card's "dirty" indicator may flip to coral cloud-sync since the edit counts as a content change.
  • 6. Auto-insert rest every stepper bumps in 1-min increments (default 10 min). Change to 7 min → close sheet → reopen → value resets to 10 (Phase A in-memory only — not persisted, expected).

3 · Display defaults section

All three controls in this section are in-memory only this wave. Header subtitle reads "applies to new exercises".

  • 7. Default treatment chip group shows three pills: Drawn / B&W / Colour. Tap each — selected chip flips to coral fill + coral border. Only one chip is selected at a time. Default selection: B&W.
  • 8. Body focus on toggle: pill-shaped, 42x24, tap flips coral on / surface off; thumb slides left<->right with a 140ms ease. Default state: ON.
  • 9. Audio on for new exercises toggle works the same way. Default state: OFF.
  • 10. Flip all three to non-default. Close + reopen the sheet. Every value resets to default (Phase A in-memory only — expected).

4 · Web viewer section

Six toggles, all default ON. Header subtitle reads "what your client sees".

  • 11. All six toggles render in this order with their help text: Show exercise title above video, Show practitioner notes, Show progress pill matrix, Show rep stack, Allow client to override treatment, Allow client to skip / extend rests. Every toggle defaults ON.
  • 12. Flip a few toggles off. Close + reopen the sheet. Every toggle resets ON (Phase A in-memory only — expected).

Save-all and Delete route to existing handlers. Copy-URL writes to clipboard. Duplicate is disabled.
