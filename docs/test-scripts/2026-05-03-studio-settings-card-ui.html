Device QA — Studio settings card

Purpose: verify the plan-settings sheet behind the gear icon on the Studio bottom toolbar. The sheet now splits its five sections across three tabs — Now (Pacing), Defaults (Display defaults + Web viewer), Plan (Plan actions + Plan info) — so no single page exceeds one screen. Save-to-Photos and Soft-delete plan continue to route through the existing handlers.

Phase A scope: UI only. The persisted field (preferred_rest_interval_seconds) writes through; every other toggle / chip / stepper is in-memory state with TODO(phase-b) markers. Reopening the sheet resets in-memory fields to their defaults and resets the active tab to Now — both intended; persistence + tab-memory wave is Phase B. Phase B rows ship a "Coming soon" chip + ~45% opacity dim + tap-blocking so practitioners aren't misled.

Out of scope: persistence for the in-memory fields, web-player consumption of any Web-viewer toggle, Duplicate plan (renders disabled), web-portal mirror.
