2026-05-03 · web-player rep stack 4-bug sweep

Device QA — rep stack bug sweep

Scope: four bugs surfaced from a Carl screenshot of the workflow Preview step. All four live in the shared web-player/ bundle and ride to both the client surface (session.homefit.studio) and the embedded mobile workflow Preview WebView.

What changed: (A) every set now gets a trailing R block in the rep stack including the LAST set; (B) the sage "✓ End of exercise" badge is render-time hidden and only revealed when the workout has reached the trailing rest of the last set (post-Start-Workout); (C) the rep-stack column is narrower (50px → 34px width, 18px → 13px gutter labels) so it stops colliding with the centered notes overlay; (D) per-set pills now render as a SINGLE uniform pill with weight + hold folded together, fixed min-width: 68px + min-height: 18px so pills are visually identical regardless of content length.

Out of scope: Flutter mobile, RPC / schema, anything not in web-player/.

Prerequisites

  • PR for branch worktree-agent-a80ff3958977673b0 merged + Vercel auto-deploy of session.homefit.studio live (cache name homefit-player-v26-rep-stack-fixes).
  • Mobile build picks up the bundle from app/assets/web-player/ — re-run ./install-device.sh if you want the embedded workflow Preview surface to reflect the fixes too.
  • Hard-refresh / pull-to-refresh on session.homefit.studio so the new SW takes over (or kill + reopen Safari).
  • At least one published session on the device with: 2+ sets per exercise, varying weight + hold values, and a notes string.

1 · Bug A — last set's trailing rest block

  • 1. Trailing R block on every set, including the last. Open a published plan with an exercise that has 3+ sets. The vertical rep-stack column on the left edge should show, bottom-to-top: S1 reps · R · S2 reps · R · S3 reps · R. The R block under S3 should look identical in shape + colour to the R blocks under S1 and S2. Previously S3's trailing R was missing if the last set had zero breather.
  • 2. Set with non-zero breather still works. Verify on an exercise where the last set DOES have a configured breather: the trailing R block uses the configured duration and the End-of-exercise marker still rides on top of it (visible after Start Workout reaches it).

2 · Bug B — End-of-exercise badge gating

  • 3. Pre-Start-Workout — no badge. Open the plan URL fresh. Before tapping Start Workout, scroll through every slide. The sage ✓ End of exercise chip must NOT appear on any slide.
  • 4. Mid-flight — no badge until last rep is done. Tap Start Workout. As the prep countdown ticks and reps tick through (S1, R, S2, R, …), the End-of-exercise chip stays hidden. It must NOT appear on the prep slide, on any non-last set's rest, or partway through the last set.
  • 5. Last rep landed — badge appears. When the workout reaches the trailing R block of the LAST set on this slide, the sage ✓ End of exercise chip appears in the aside, riding on top of the R block. Navigate to the next slide; the chip stays gated correctly there too (hidden until the last rep of the last set lands).

3 · Bug C — rep stack narrower, no notes overlap

  • 6. Notes panel sits clear of the rep stack. Open an exercise with notes (e.g. "Keep back straight, slow on the descent."). The notes overlay must NOT visually collide with the S1 / S2 / S3 labels or the rep blocks at the bottom-left of the canvas. The rep stack lives in the leftmost ~34px of the canvas; the notes overlay should breathe to the right of it.
  • 7. Long notes also clear. Test with a deliberately-long notes string ("dhdiddo so then I said we should go to the office and check…" — at least one full screen of wrap). The rep stack still sits independent of the notes panel; nothing overlaps.
  • 8. Stack still readable. The S1 / S2 / S3 labels in the gutter are still legible at 13px width — not crushed against the rep blocks, and the rep blocks themselves don't visually disappear.

4 · Bug D — uniform per-set pills

  • 9. 3+ sets with varying values render uniform pills. Open an exercise with 3+ sets that have DIFFERENT weight + hold combos (e.g. set 1 = 5 kg · 2 s, set 2 = 45 kg · 5 s, set 3 = 120 kg · 12 s). Every per-set pill in the aside should render at visually identical width and height. Ragged dimensions = fail.
  • 10. Combined weight + hold in one pill. Each per-set pill should read NN kg · MM s (no separate "X s hold" sub-line below the chip). Bodyweight sets read just Bodyweight. Sets with zero hold read just NN kg.
  • 11. Long content fits. A pill with content like 120 kg · 12 s still fits inside the chip without truncation or ellipsis; text is centred within the pill.

  • 12. Client web player reflects all 4 fixes. Open the published plan URL on session.homefit.studio in Safari (or share to WhatsApp + tap the unfurl). Items 1, 5, 6, 9 above all pass.
  • 13. Mobile workflow Preview reflects all 4 fixes. In the trainer app, open a session → tap the workflow's Preview chevron. The embedded WebView pulls from app/assets/web-player/. Items 1, 5, 6, 9 above all pass identically to the client surface.
