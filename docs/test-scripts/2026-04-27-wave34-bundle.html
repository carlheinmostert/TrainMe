Wave 34 · device QA

Native AV avatar camera + W33 carryover

(A) The avatar capture surface drops the Flutter camera plugin entirely. Native AVCaptureSession bound to .builtInWideAngleCamera directly — no virtual multi-cam device, no auto-lens-switch, no fish-eye. Live preview is a UiKitView hosting AVCaptureVideoPreviewLayer with videoOrientation = .portrait pinned. New channel com.raidme.avatar_camera; new AvatarCameraChannel.swift + AvatarCameraPreviewFactory. Diagnostics now via os_log (subsystem com.raidme.raidme, category avatar.capture) so they actually surface in Console.app under iOS Flutter profile builds. (B) SessionCard count badge shrinks ~15% (16×16 → 14×14, fonts 10/8 → 9/7) so it reads as a glanceable accent against the now-larger leading icon. (C) Photo preview controls: treatment switcher + Body Focus pill now render for photos (was video-only through W33). Audio + Rotate stay video-only. (D) Client + SessionCard leading icons bumped +50% (40×40 → 60×60, glyph 22 → 33).

A · Avatar camera native AV

  • 1. Avatar capture opens portrait + standard wide lens (no fish-eye).
    1. Tap empty avatar slot on a client detail. Camera opens locked PORTRAIT.
    2. Subject in centre frame looks proportional — no Ultra Wide warping. Compare with the Camera.app default rear-wide lens to sanity-check FOV.
    3. Capture a still. The composite preview shows clean subject + blurred background.
    4. Tap "Use this". Avatar lands on the client header upright, not sideways.
  • 2. Live preview stays portrait when you rotate the phone.
    1. Open avatar capture. Rotate the phone to landscape.
    2. The preview frame stays upright — subject's head still points to the top of the screen, NOT to the side. (Wave 33 / 32 used to leak landscape.)
    3. Rotate back to portrait. Preview unchanged.
    4. Tap shutter while landscape. Composed PNG comes out upright (subject head at top).
  • 3. Background + foreground lifecycle works.
    1. Open avatar capture. Swipe up to send the app to the background.
    2. Wait ~3 seconds. Tap the homefit icon to return.
    3. Live preview re-attaches without reopening the screen — no black frame, no "Camera unavailable" error. (If preview goes dark, log the time and capture in the note.)
    4. Capture still works.
  • 4. Console.app: native diagnostics surface under avatar.capture category.
    1. Plug iPhone into Mac. Open Console.app and select the iPhone in the sidebar.
    2. Filter on subsystem com.raidme.raidme AND category avatar.capture (or just the substring avatar.capture).
    3. Tap the empty avatar slot on a client. Console fills with os_log lines.
    4. Expect AvatarCameraChannel initialised on first call (might have appeared earlier on app launch).
    5. Expect startSession: available back devices types=[...] names=[...] — full device list.
    6. Expect startSession: started device=<name> uniqueID=<...> deviceType=AVCaptureDeviceTypeBuiltInWideAngleCamera minZoom=1.000000 maxZoom=... — the load-bearing line. deviceType MUST be BuiltInWideAngleCamera; if it reads BuiltInDualWideCamera / BuiltInTripleCamera, AVFoundation gave us the virtual device and the fix didn't land. Capture the deviceType raw value in the note.
    7. Tap shutter. Expect capturePhoto: requesting capture outPath=... followed by capturePhoto: wrote ....
    8. Close screen. Expect stopSession: stopped.
  • 5. Re-capture works (avatar is replaced cleanly).
    1. On a client that already has an avatar, tap the avatar (or the camera-pencil affordance) to re-capture.
    2. Avatar capture screen opens. Take a different still + tap "Use this".
    3. Header avatar updates immediately to the new still. The old PNG is replaced (not stacked, not stale).

B · Count badge −15%

  • 6. Count badge fits naturally against the now-larger leading icon.
    1. Open a client with multiple sessions. Inspect SessionCard rows.
    2. The little count badge bottom-right of the leading icon reads as a small accent — not visually competing with the icon. Single-digit (1-9) and 2-digit (10-99) both fit cleanly inside the smaller (14×14) footprint.
    3. No truncation. No overflow into the icon body. Empty session (0 exercises) still hides the badge entirely.

C · Photo preview controls

  • 7. Photo preview shows treatment switcher + Body Focus pill.
    1. Open a session that has at least one PHOTO exercise (short-press the shutter in capture mode to add a photo if needed).
    2. Tap the photo's thumbnail in Studio to open _MediaViewer.
    3. Treatment pill (vertical book-spine, left edge) is visible — same widget videos use. Tap segments to switch Line / B&W / Original.
    4. Line treatment renders the on-device line drawing JPG. B&W renders the raw colour photo with a grayscale CSS-style filter applied. Original renders the raw colour photo untouched.
    5. Body Focus pill is visible bottom-left. Audio pill + Rotate pill ARE NOT visible (those stay video-only).
  • 8. Locked segments stay locked on legacy photos with no raw colour file.
    1. If you have an old photo exercise from a pre-Wave-22 capture (where the raw colour JPG was pruned), open it in _MediaViewer.
    2. Treatment pill: B&W and Original render with a lock glyph — tapping shows the "Older capture — re-record to enable." tooltip. Same gating videos use when the local archive mp4 is missing.
    3. If you don't have a legacy photo: skip this item with a note — the gating logic is the same as the existing video gating, so it's covered by W22's prior verification.

D · Card icons +50%

  • 9. Client + Session card leading icons read as confident anchors.
    1. On Home (clients list) inspect the leading person icon on a few _ClientCard rows. The coral-tint square is now 60×60 (was 40×40); the person glyph is ~33pt.
    2. Open a client → sessions list. Inspect the leading list-alt icon on each SessionCard. Same 60×60 coral square + 33pt glyph.
    3. Both card types feel visually consistent — the icons read as a confident left anchor, not chip-sized.
    4. The count badge (bottom-right of SessionCard's icon) is anchored properly — doesn't float, doesn't overlap into the body of the icon.
