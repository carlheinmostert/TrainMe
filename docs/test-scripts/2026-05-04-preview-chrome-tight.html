Device QA · iPhone

Preview chrome tight — close button + Dynamic Island clamp

Two iPhone-only chrome fixes on the workflow Preview surface.

Bug A — preview close (X) button. The top-left close button on the Flutter Preview screen sat ~8 px below the safe-area top + ~8 px in from the left, AND was wrapped in a SafeArea inside an already-safe-area-aware Positioned — so the inset was being doubled. Carl: “needs to move to top left corner, tight.” Fix: drop the redundant SafeArea wrap and zero the offsets so the chip sits flush against the status-bar inset, against the left edge.

Bug B — fullscreen mode under the Dynamic Island. When the player goes fullscreen on iPhone, body.is-fullscreen #app { padding-top: 0 } stripped the safe-area top inset, so matrix pills + the active-pill clock slipped under the Dynamic Island. Fix: padding-top: var(--safe-top) in the same rule keeps the existing env(safe-area-inset-top) token so content stops below the island. Phones without an island still report 0 here — no-op for them.

SW cache bumped: homefit-player-v33-fractional-seconds-fix → homefit-player-v34-island-clamp.
