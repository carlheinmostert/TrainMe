Wave 28 · device QA

Landscape orientation support

Single landing for the entire landscape rollout. (A) Camera mode now reflows correctly to landscape; preview no longer warps. Recording-in-flight orientation lock prevents mid-record rotation. (B) _MediaViewer reflows chrome for landscape — treatment + body focus + mute + new rotate-90 pill stack horizontally; tune sheet becomes a popover; trim panel is compact. (C) Web player has landscape CSS for the first time (zero @media (orientation: landscape) rules existed before today). Schema migration adds aspect_ratio + rotation_quarters to exercises; get_plan_full auto-flows. Mobile + web read both. Home / ClientSessions / Settings / Sign-In / Auth gate / Studio mode are explicitly portrait-locked. iPad locked to iPhone-only orientations. PlanPreviewScreen retired entirely. Latest cache v64-landscape. SQLite v28.

A · Orientation policy — stay portrait where it should

  • 1. Sign-in, Home, Client list, Client detail, Settings all stay portrait when rotated.
    1. Sign out. Rotate iPhone — Sign-in screen stays portrait.
    2. Sign in. Home screen — rotate, stays portrait.
    3. Tap a client → ClientSessionsScreen — rotate, stays portrait.
    4. Open Settings — rotate, stays portrait.
  • 2. Studio mode stays portrait — bottom-anchored reach unchanged.
    1. Open a session → Studio mode. Rotate — stays portrait.
    2. Studio's chip rows + InlineActionTray + bottom rest bars all unchanged from previous waves.

B · Camera capture — landscape works (Wave 28 marquee)

  • 3. Rotating the phone in Camera mode reflows the preview correctly. No warp.
    1. Open a session → swipe to Camera mode. Verify preview is correct in portrait (full screen, fills bounds).
    2. Rotate to landscape (left or right). Preview should reflow to fill the now-wider canvas. No more "warps to portrait".
    3. Rotate back to portrait. Preview reflows back. No artifacts.
  • 4. Recording locks orientation while in flight.
    1. In Camera, rotate to landscape. Long-press shutter to start recording.
    2. Mid-recording, try to rotate to portrait. Phone orientation should NOT change until you release the shutter (lock acquired at landscape on record-start).
    3. Release the shutter (recording ends). Orientation lock releases — phone can now rotate freely.
    4. Repeat starting in portrait — record-start locks at portrait until release.
  • 5. Landscape-recorded clip plays back as landscape on the Studio card thumbnail + in _MediaViewer.
    1. Record a video in landscape.
    2. Swipe back to Studio (forces portrait). Card thumbnail shows the landscape source proportionally — no squashed faces.
    3. Tap the thumbnail → _MediaViewer opens. Video renders correctly (letterboxed inside portrait viewport, OR fills if you rotate the phone to landscape).

  • 9. Rotate-90 pill in _MediaViewer rotates playback 90° clockwise per tap; long-press resets to 0.

    3. Tap again. 180°. Tap again. 270°. Tap again. Back to 0°.
    4. Long-press the pill — instantly resets to 0° regardless of current state.
    5. Pill is hidden on photo / rest rows (only visible for videos).
  • 10. Rotation persists across app restart and ships on publish.

    2. Reopen the same exercise → still rotated 90°.
    3. Publish the plan. Open the published URL on Mac Safari (hard refresh for new SW cache).
    4. Same exercise renders rotated 90° in the web player.
  • 11. Rotation composes correctly with crossfade, body focus, treatment switch, trim.

    4. Drag trim handles — trim window respected against the source clock (rotation doesn't affect trim).
