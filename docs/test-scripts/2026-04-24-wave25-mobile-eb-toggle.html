Device QA — Wave 25 mobile Enhanced Background toggle

Purpose: verify the Studio per-exercise _MediaViewer on mobile now plays the segmented raw file for B&W + Original by default, and surfaces an Enhanced Background toggle pill stacked under the treatment pill — R-10 twin of the web player's gear-popover switch (Milestone P+G).

The bug it fixes: previously _sourcePathForTreatment always returned absoluteArchiveFilePath for B&W + Original. The segmented variant (same Vision body mask the line drawing uses, body-pop + dimmed background) was written alongside the raw archive since v22 but never played by the practitioner-side preview. Practitioner saw a different rendering than the client.

What landed: two mobile commits on this worktree branch. (1) _sourcePathForTreatment now consults a _enhancedBackground flag — when true (default) AND the segmented file exists locally, B&W + Original prefer it; otherwise falls through to the untouched archive. Line is unaffected. (2) New _EnhancedBackgroundTogglePill stacked under TreatmentSegmentedControl in the same left-edge column — book-spine rotated label, coral fill when ON. Persisted per-device under SharedPreferences key homefit.preview.enhanced_background. Greys out + tooltip when the current treatment is Line.

Surfaces touched: Flutter only — app/lib/screens/studio_mode_screen.dart. No web player changes (it already had the gear popover). No Supabase changes. No model changes (absoluteSegmentedRawFilePath accessor was already present from v22).

What did NOT change: existing treatment switching, mute pill, page dots, play/pause overlay, vertical-swipe-to-cycle treatments, horizontal-swipe-between-exercises, long-press menu (Wave 19.4), soft-trim editor (Wave 20). All preserved.

Prerequisites

  • Latest ./install-device.sh (today). Build SHA on Home footer should match the Wave 25 commits.
  • A test client with Colour + B&W consent GRANTED (Settings → Clients → toggle on).
  • At least one freshly-captured video exercise on that client (post-v22 — has both archive + segmented files locally).
  • For item 6 (legacy fallback): one exercise captured BEFORE v22 (no segmentedRawFilePath). If unavailable on the device today, mark item 6 as Carl-confirms-later and skip.
