Device QA — hide treatment selector behind a settings gear (round 4)

Why: the practitioner has already set per-client preferred_treatment per-exercise as part of plan setup, so 95% of clients should never need a global override. The treatment row was earning a premium spot it didn't deserve. Round 4 hides it behind a gear in the sticky CTA bar — the chrome reclaims its visual space; the override is one tap away when actually needed.

Net layout: sticky bar drops the treatment-pills row entirely. Start workout shrinks horizontally (flex: 1 1 auto) to make room for a 48×48 gear button to its right with an 8 px gap. Tap gear → popover anchored above the gear opens upward with the same three pills (Line / B&W / Colour) and the same lock-glyph + self-grant-modal behaviour. Practitioner default still applies on lobby load — clients who don't open the gear get the right treatment automatically.
