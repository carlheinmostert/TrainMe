QA — generic web-player bundle sync + --check mode

Scope: app/tool/sync_web_player_bundle.dart. Local dev-tool change — no device install required, no SW cache bump.

Why this exists: Carl asked “don't we have a sync check for this?” after lobby.js was missing from the iPhone bundle for several rounds (PR #243 fixed it manually). The hardcoded _filesToCopy list was the root cause — every new web-player file required a manual edit.

Fix: walk top-level web-player/, copy everything except an explicit exclude list (sw.js, vercel.json, serve.json, middleware.js, *.results.json). New files in the bundle are picked up automatically. New --check mode reports drift and exits non-zero — clean hook for CI / pre-commit later.

Run from app/: dart run tool/sync_web_player_bundle.dart (copy) or dart run tool/sync_web_player_bundle.dart --check (drift report).

Prerequisites

  • PR for branch worktree-agent-af6a1a36ecb864e60 merged (or checked out locally).
  • Repo at main with app/ as cwd. Dart SDK available (any version that runs the existing app dev tools).
  • None of the test items require a device install — this is pure local tooling.

1 · Normal mode — copy works

  • 1. Run normal mode — succeeds, prints copied files, exits 0. From app/: dart run tool/sync_web_player_bundle.dart. Expected: one copied … line per non-excluded source file, a final web-player bundle synced (N files) summary, exit code 0. Re-running is idempotent — second run produces identical output.
  • 2. Every non-excluded source file is in dest with identical bytes. After running normal mode, for each file in web-player/ that is NOT in the exclude list, the corresponding app/assets/web-player/<name> file exists and diff <src> <dst> is empty. Spot-check a few: diff web-player/index.html app/assets/web-player/index.html → no output. Same for app.js, api.js, styles.css, lobby.js, what-we-share.html, what-we-share.js.
  • 3. Excluded files are NOT in app/assets/web-player/. Confirm none of sw.js, vercel.json, serve.json, middleware.js sit in the dest dir: ls app/assets/web-player/ | grep -E '^(sw\.js|vercel\.json|serve\.json|middleware\.js)$' → no output (or they were there from a previous run and the script doesn't actively remove them — in that case --check will flag them as stale, see item 6).

2 · Auto-pickup — new file gets bundled without code change

  • 4. Hypothetical new file is auto-picked-up. From the repo root: echo "marker" > web-player/test-marker.txt. Then from app/: dart run tool/sync_web_player_bundle.dart. Expected: the output includes a copied …test-marker.txt line and app/assets/web-player/test-marker.txt exists with the same content. Cleanup: rm web-player/test-marker.txt app/assets/web-player/test-marker.txt — do not commit.

3 · --check mode — drift detection

  • 5. --check on a clean bundle exits 0. Immediately after running normal mode (item 1): dart run tool/sync_web_player_bundle.dart --check; echo $?. Expected: web-player bundle in sync (N files) on stdout, exit code 0.
  • 6. Force drift → --check exits non-zero with a clear error. Delete a file from the dest: rm app/assets/web-player/api.js. Run dart run tool/sync_web_player_bundle.dart --check; echo $?. Expected: stderr says web-player bundle DRIFT detected: with a missing from … - api.js entry, and the final line tells you to run the script to repair. Exit code is 1 (or any non-zero).
  • 7. Stale dest file is flagged. Create a stray file: echo "stale" > app/assets/web-player/stale-bogus.txt. Run dart run tool/sync_web_player_bundle.dart --check. Expected: stderr lists stale in … - stale-bogus.txt and exits non-zero. Cleanup: rm app/assets/web-player/stale-bogus.txt.
  • 8. Repair via normal mode + --check passes again. After the drift in item 6 (and cleanup of any stale files from item 7): dart run tool/sync_web_player_bundle.dart, then dart run tool/sync_web_player_bundle.dart --check; echo $?. Expected: web-player bundle in sync (N files), exit code 0.
Source-of-truth lives in app/tool/sync_web_player_bundle.dart. Exclude list is the literal-filename set + the *.results.json suffix; subdirectories are skipped entirely. If the bundle ever needs nested assets, the walker needs updating — see the file's top-of-file comment.
