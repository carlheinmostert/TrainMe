2026-05-03 · Hero frame everywhere

Device QA — Hero frame on Studio cards + editor header

Purpose: the Studio bottom-sheet exercise cards and the editor sheet's small header thumbnail used to play the source video on a loop. They now render the static Hero shot — auto-picked motion-peak by default, or whatever the practitioner has dragged the trim panel's white star to. The Preview tab inside the editor still plays motion (that's where motion belongs).

In scope: app/lib/widgets/mini_preview.dart — new staticHero flag + _HeroFrameImage branch + _HeroFileImage (FileImage subclass that includes the Hero offset in its cache identity, so an in-place JPG rewrite forces a re-decode). app/lib/widgets/studio_exercise_card.dart — passes staticHero: true. app/lib/widgets/exercise_editor_sheet.dart — header MiniPreview passes staticHero: true.

Out of scope: the Preview-tab MediaViewerBody (still motion), the camera peek box / capture-mode strip / ThumbnailPeek (those already used the JPGs via CaptureThumbnail), the regenerateHeroThumbnails implementation (untouched), the web player.
