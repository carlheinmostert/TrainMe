Device QA — Hero tab cloud prefetch

Purpose: verify the editor sheet's Hero tab no longer hard-fails with "Raw video file not found." on cloud-hydrated sessions. Stacks on top of PR #190 (sessions hydrate from cloud) and PR #189 (Hero tab feature). Tapping a video exercise → Hero tab now silently kicks off the same raw-archive prefetch the locked-segment tap path uses.

In scope: _HeroTab + _HeroTabState in app/lib/widgets/exercise_editor_sheet.dart; new practiceId + planId threading from _buildHeroTab; MediaPrefetchService.prefetchRawArchive integration; coral spinner + "Downloading original…" placeholder; failure copy "Original video unavailable — recapture to use Hero." Photo/rest placeholders unchanged.

Out of scope: the Studio locked-segment tap path (already shipped); analytics events; consent gates; the line-drawing prefetch for Studio MiniPreview cards.

Prerequisites

  • Branch claude/strange-hypatia-5b5736 installed on Carl's iPhone 00008150-001A31D40E88401C.
  • At least one cloud-only session with video exercises whose raw mp4 has NOT been pulled locally yet (force via fresh re-install + sign-in).
  • Same client must have video_consent.original = true so the signed URL path actually returns a URL.
  • Stream device logs in a terminal: idevicesyslog --udid 00008150-001A31D40E88401C 2>&1 | grep -E 'MediaPrefetchService|_HeroTab|prefetchRawArchive'.

1 · Cloud-only video → Hero tab triggers prefetch

The headline test. The bug was the Hero tab bailing with "Raw video file not found." instead of pulling the file from cloud.

  • 1. Reinstall from main, sign in, wait for clients list to populate. Tap a client → tap a session that was published from another device (so the raw mp4 is NOT on this iPhone). Tap any video exercise card to open the editor sheet. Tap the Hero tab. Expect: a coral spinner above "Downloading original…" — NOT the red "Raw video file not found." string. os_log shows MediaPrefetchService: prefetchRawArchive exerciseId=<id> — ….
  • 2. Within ~3-15s the spinner clears and the live video preview + scrubber render. The video plays muted, soft-trim window is honoured, scrubber is interactive (drag → video seeks to that frame). The "Set as Hero" button is disabled until the scrubber position differs from the saved focus offset.

2 · Set Hero, regen succeeds

  • 3. Drag the scrubber to a frame about a third of the way through the video. The "Set as Hero" button highlights coral. Tap it. Expect: brief inline spinner, button greys out, then re-enables marked as committed. Dismiss the sheet and look at the Studio exercise card — the thumbnail / MiniPreview poster reflects the new Hero frame on the next paint. No crashes, no SnackBars.

3 · Re-open later — local-first, no spinner

  • 4. Without restarting the app, dismiss the sheet, then tap the same exercise + Hero tab again. Expect: video preview + scrubber render immediately (the prefetch has stamped the local mp4, so the local-first branch wins). os_log shows no second prefetchRawArchive for this exercise. Force-quit the app, relaunch, and re-open the same Hero tab — same result, instant render with no download.

No crash, no SnackBar leak from the editor sheet. Dismiss the sheet — Studio looks normal.
