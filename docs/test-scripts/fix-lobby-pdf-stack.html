2026-05-15 · Lobby PDF export stack hotfix

Hotfix QA — gear popover + Download bridge + multi-page timeout + copy

Four related lobby fixes from the 2026-05-14 QA wave land in one branch (fix/lobby-pdf-export-stack):

BUG 14 — gear popover offscreen on iOS. The popover was position: absolute nested inside .lobby-cta-bar (position: fixed; bottom: 0). Desktop laid it out fine; iOS WKWebView rendered it partially outside the viewport. Fix: on every open, JS computes the gear button's getBoundingClientRect() and sets position: fixed with viewport-aware top/left (bottom-right anchored to the gear, 16px margin clamp). Resize / orientation / scroll re-anchor while open.

BUG 8 — Download PDF does nothing on iOS WKWebView. The Download button was a bare <a download> anchor; WKWebView silently ignores those clicks. Fix: on iOS embedded surfaces, Download routes through the existing homefitBridge.shareFile bridge (same path as Share — base64 over the channel, Dart writes a temp file, UIActivityViewController includes Save to Files). Desktop / live web fall through to the native anchor behaviour.

BUG 10a — page-2 render timeout on multi-page PDFs. The per-page html2canvas rasterise had a 15s ceiling; if one of 8/9 images didn't fully pre-fetch, html2canvas would try to refetch during page-2 layout, blow the budget, and the error modal would fire. Fix: per-page timeout bumped 15s → 30s; image pre-fetch budget 8s → 15s on iOS embedded WebView (faster paths keep 8s); explicit pageCanvas.width = 0; pageCanvas.height = 0; pageCanvas = null; after each page so the GC reclaims the 1588×N bitmap before the next render.

Polish — Cmd+Shift+R copy. Two error messages told the user to "Refresh with Cmd+Shift+R" — useless on iOS WKWebView where that gesture doesn't exist. Fix: rewrote both to "Reload and try again" / "Reload the page and try again" so the copy is device-agnostic.

Out of scope: any non-lobby surface (deck player, Studio, settings panes). The deck's .settings-popover uses its own absolute layout against the body and is unaffected.
