Device QA — Refine button on Studio bottom toolbar

Scope: a 5th icon — Refine — lands on the Studio bottom workflow pill, between Camera and Preview. Tapping it opens the per-exercise editor sheet for the topmost (most recent) capture in the Studio stack. Disabled (dimmed) when the session has no real exercises yet.

Why: aligns the in-app toolbar with the public help page narrative at manage.homefit.studio/getting-started, which already walks practitioners through Camera → Refine → Preview → Publish → Share. The icon was missing from the actual app.

What did NOT change: Camera, Preview, Publish, Share behaviour. The right-aligned Settings gear utility. The editor sheet itself. No schema, no SQLite bump, no SharedPrefs key.

Prerequisites

  • PR for branch worktree-agent-aa42a3eb9a441e83b merged + ./install-device.sh from repo root.
  • Have at least one client with at least one published or draft session you can open in Studio. Have one client with an empty session (no exercises) for the dimmed-state check.

1 · Icon present and positioned correctly

  • 1. Open a session in Studio that already has at least one captured exercise. The bottom workflow pill now shows 5 icons inside the raised pill, in this exact order from left to right: camera → list (the new Refine glyph) → play/slideshow → upload (or unlock) → share. Coral chevron triangles between each icon as before.
  • 2. Long-press the new icon to surface its tooltip. Tooltip reads "Refine top card". Glyph is Icons.view_list_rounded — horizontal bars with leading dots, a list-of-rows look. White (active state), 28pt, same size as the other 4 icons. Right-aligned Settings gear is unchanged.

2 · Tap opens the editor for the topmost card

  • 3. Note the card visually nearest the bottom of the Studio stack (the most recent capture). Read its name. Now tap the new Refine icon. The full-screen editor sheet opens. Its header shows the same exercise name you noted — Refine targets the most recent capture, not a random one and not the oldest.
  • 4. Capture a new video or photo (Camera mode), let it convert, swipe back to Studio. The new capture appears at the bottom of the stack. Tap Refine — the editor opens onto THAT new capture, not the previous one. The "topmost = most recent" promise survives a fresh capture.

3 · Rest periods are skipped

  • 5. If the session has a trailing rest period at the bottom (auto-inserted between exercises), tap Refine. The editor opens onto the most recent real exercise — rest periods don't have media to edit, and Refine should hop over them. If the bottom of the stack is a rest bar, the editor still opens onto the exercise above it.

4 · Empty-session disabled state

  • 6. Open a brand-new session that has no exercises captured yet (or use the New Session FAB on a client). The Refine icon and its surrounding chevron triangles render dimmed (~45% white) just like the Preview/Publish/Share slots in an empty session. Tapping it does nothing — no editor opens, no haptic, no error.

5 · Existing toolbar behaviour intact

  • 7. Run a sanity check on the other 4 toolbar icons. Camera (slot 1) opens the camera. Preview (slot 3) opens the workout preview. Publish (slot 4) publishes or shows the unlock pill if locked. Share (slot 5) opens the share sheet on a published plan. None of these regressed because Refine slipped into slot 2.
  • 8. Right-aligned Settings gear is unchanged. Still sits outside the workflow pill on the far right, still opens the plan-settings sheet. Coral deviation dot still appears when defaults are deviated from. Refine did NOT push the gear off-screen or into the pill.
Implementation: app/lib/widgets/studio_bottom_bar.dart (new onRefine field + 5th _ToolbarIconButton at slot 2) and app/lib/screens/studio_mode_screen.dart (new _canRefine getter + _refineFromToolbar handler that picks the last non-rest exercise and calls _openMediaViewer). No schema, no RPC, no SQLite version bump.
