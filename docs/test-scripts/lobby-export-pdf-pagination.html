2026-05-14 · Lobby export pivot — multi-page PDF (PNG retired)

Device QA — Lobby Exports A Multi-Page PDF

Purpose: verify the lobby's Share button now produces a multi-page PDF instead of a single rasterised PNG. The PDF chunks the plan into pages of up to 5 entries, never splits a circuit across pages, ships a full header on page 1 (plan title + summary line) and a compact running header on subsequent pages, with a footer ("Page N of M" left, "Visual plans clients follow. · homefit.studio" right) on every page.

Pivot from: the 2026-05-05 PNG flow generated one giant raster that was unreadable on phones and a pain to share. The PDF is a print-friendly artifact clients can save, print, or forward. Active highlights are stripped (#10.1), circuit chrome is contained (#10.3), and the full exercise list is captured (#10.4) by structural pagination — no viewport clipping possible.

iOS Save / Share: embedded WKWebView routes through HomefitBridge.shareFile (Dart writes a temp file + presents UIActivityViewController — Save to Files, AirDrop, Messages, Mail are all in the share sheet). Mobile Safari uses navigator.share({ files: [pdfFile] }) with a window.open fallback. Desktop downloads via a real <a download> anchor.

Modal shape: spinner with progress text ("Rendering page 2 of 3…"), then page-1 PNG thumbnail with a "1 of N pages" coral badge, then Download PDF + Share buttons. On generation failure: visible error message + a Retry button (replaces the silent hang).

In scope: lobby PDF generation; multi-page layout; circuit no-split rule; page chrome (full/running header + footer); modal preview + buttons; native bridge Save / Share; desktop download; error + retry path; R-10 parity (Mobile Safari at staging.session.homefit.studio); drift CI gate.

Out of scope: PayFast / billing flows; non-lobby player surfaces; PDF content beyond the listed visual rules (no fonts beyond the bundled stack); Android / desktop browsers other than what's listed; PDF accessibility metadata.

  • Build from feat/lobby-export-pdf-pagination on Carl's iPhone via ./install-device.sh, OR Vercel staging deploy of the same branch reachable at staging.session.homefit.studio.
  • At least one published staging plan with 3+ exercises (short plan), one with 8+ (medium), one with 20+ (long), one with a circuit small enough to fit on one page, one with a circuit large enough that it would have to span a page-break.

4 · Circuit no-split rule

A circuit is treated as ONE atomic entry on a page. If it doesn't fit in the remaining row slots, a fresh page begins BEFORE the circuit so the whole group lives on the next page.

  • 4. Plan with a circuit positioned EARLY in the list (so it fits on page 1 within the 5-entry budget). The circuit's header, in-circuit rows, and rounds chip all appear together on page 1 — never split. Circuit chrome (SVG lane outlines + spiraling tracer settled to a static frame) is symmetric, no asymmetric clipping at the corners (issue #10.3 fix). (device)
  • 5. Plan whose 5th slot would land mid-circuit (e.g. 4 standalone rows + 1 circuit with 3 in-circuit rows). The PDF starts a new page BEFORE the circuit so the whole circuit lives on page 2. Implementation notes: web-player/lobby.js orchestrates the pipeline. jsPDF is vendored at web-player/jspdf.umd.min.js (358KB; gzip ~85KB) and loaded BEFORE lobby.js in index.html so the global window.jspdf.jsPDF constructor is ready. The page chunker (chunkLobbyItemsForPdf) walks #lobby-list children at 5 entries per page; circuits are atomic units (no split). For each page chunk, the live DOM elements are cloned into an off-screen .lobby-export-page wrapper with appropriate header (full on page 1, compact running on pages 2+) and footer, then html2canvas-rendered at scale=2 to a JPEG (quality 0.92) and added to the jsPDF doc at A4 portrait (210mm wide, height derived from canvas aspect). Bridge: web-player/app.js adds homefitBridge.shareFile(base64, fileName, mimeType); app/lib/screens/unified_preview_screen.dart handles the matching share_file case by writing a temp file and invoking Share.shareXFiles. CSS suppressions: html.is-exporting + .lobby-export-page strip active-row coral highlight (#10.1), force overflow:visible on circuit chrome containers (#10.3), and force header text wrapping with right-padding to stop overruns.
