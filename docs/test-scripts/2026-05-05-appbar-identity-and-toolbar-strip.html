2026-05-05 · appbar identity + toolbar strip

Device QA — AppBar identity overhaul + Studio bottom-strip cleanup

Scope: three pre-TestFlight cosmetic items bundled because they all rewrite the same surface — Home / Client detail / Studio top + bottom chrome. Item #1: a help (?) icon lands left of the gear in the Home AppBar; tap opens manage.homefit.studio/getting-started in external Safari (same external-Safari pattern as the first-launch banner). Item #2: AppBar identity overhaul — every screen's AppBar title becomes what you're looking at: client detail shows a 32px avatar plus the dashed-underline client name; Studio gets a brand-new top AppBar with back-labelled-with-client-name + dashed-underline session title + gear on the right. Camera mode has no AppBar (full-screen viewfinder). Item #4: Studio's bottom toolbar drops the stats strip, subtitle, back arrow, and gear (now all carried by the new AppBar). The bottom is JUST the workflow pill (Camera → Refine → Preview → Publish → Share). The metadata that used to live above the toolbar moves into a new Statistics tab on the plan-settings sheet — Now / Defaults / Plan / Statistics.

What did NOT change: workflow pill icons + names + glyphs (item #5 CAPS rename lands in a follow-up PR). Refine handler / Camera handler / Preview / Publish / Share callbacks. Plan-settings sheet's existing Now / Defaults / Plan tabs. No schema, no SQLite bump, no SharedPrefs key.

Prerequisites

  • PR for branch chore/appbar-identity-and-toolbar-strip merged + ./install-device.sh from repo root.
  • Have at least one client with at least one published session, and one client with a draft (unpublished) session.
  • Optional bonus: a client with a session that's past its 14-day grace (locked state) so you can verify the Statistics tab's red-dot copy.

1 · Home AppBar — help icon (item #1)

  • 1. Open Home (the clients list). The top-right of the screen now shows a help icon (a question-mark glyph, Icons.help_outline) sitting immediately to the left of the existing gear. Both icons are white, 28pt for the help icon and 26pt for the gear, with comfortable hit targets. Long-press surfaces the tooltip "Help".
  • 2. Tap the help icon. The trainer app drops to the background and Safari opens at manage.homefit.studio/getting-started. NOT an in-app browser; the URL bar reads "Safari" in the system task switcher. Switching back to the trainer app preserves your spot on Home.
  • 3. Tap the gear. The Settings screen opens as before. Help icon did NOT push the gear off-screen or break its hit area. The top-left network-share icon is also unchanged.

2 · Client detail AppBar (item #2)

  • 4. From Home, tap a client to open the detail screen. The AppBar shows: a back chevron labelled "Back to Clients" (long-press tooltip), a small 32px circular avatar, and the client's name with a dashed underline beneath it. The body of the screen no longer has a duplicate big avatar+name+dashed-underline header above the consent chip.
  • 5. Avatar is the consent-gated photo or fallback. If the client has avatar consent granted AND a captured avatar PNG, the AppBar avatar shows that photo. Otherwise it shows a coral-tinted disc with the initials monogram (same three-tier fallback the Home cards use). Tap the avatar — capture flow opens (or consent sheet if avatar consent isn't granted yet).
  • 6. Tap the client name in the AppBar. An inline-rename TextField appears just below the AppBar (above the consent chip), pre-filled with the current name and selected. Type a new name — press Return. The AppBar title updates to the new name with the dashed underline, and the rename persists across navigation back to Home and back into the client.
  • 7. Body shows: consent chip, sessions list, New Session FAB. No duplicate header in the body. The 3-dot overflow menu in the AppBar still works (Delete client). The back chevron still pops you back to Home.

3 · Studio AppBar (item #2)

  • 8. From the client detail screen, tap a session. Studio mode opens with a brand-new AppBar at the top: a chevron + the client's name on the left (e.g. "← Carl"), the session title (the auto-formatted date like "5 May 2026 14:30") with a dashed underline in the centre/title slot, and the settings gear on the right. Camera mode previously had no AppBar — that's still true (this AppBar is Studio-only).
  • 9. Tap the session title in the AppBar. The title flips to a coral-cursored TextField inline. Type a new title (e.g. "Knee rehab session 1") and press Return — the title updates, dashed underline returns. Pop back to the client list and re-open: the rename persists.
  • 10. Tap the gear in the AppBar. The plan-settings bottom sheet opens. The gear is on the AppBar's right edge now (it used to live at the bottom-right of the studio toolbar — that location is dropped per item #4). Closing the sheet returns you to Studio.
  • 11. Coral deviation dot on the gear. Open the sheet, bump the "Default rest between exercises" stepper to a value other than 30s, close the sheet. The settings gear in the AppBar shows a tiny coral dot in its top-right corner. Reset rest to 30s — the dot disappears.
  • 12. Tap the back chevron + client label. You pop back to the client detail screen as before. The AppBar back chevron is the only path; iOS edge-swipe-to-pop is intentionally disabled (it conflicts with the Studio ↔ Camera page-swipe). 48pt hit area — no fiddly chevron-only target.

4 · Camera mode (no AppBar)

  • 13. From Studio, swipe right (or tap the Camera icon on the workflow pill). Camera mode opens. The Studio AppBar disappears entirely — no chrome at the top, full-screen viewfinder. Camera's own affordances (shutter button, lens pills, library peek) are unchanged.
  • 14. Swipe back left to Studio. The Studio AppBar reappears with the same back/title/gear. No flicker, no half-rendered AppBar during the swipe transition. Camera → Studio → Camera is smooth.

5 · Studio bottom toolbar stripped (item #4)

  • 15. The bottom of Studio is noticeably shorter than before. What's gone: the "First opened {date} · Last opened {date}" stats strip, the lock-state line with coloured dot, the "{date} · {client name}" subtitle row, the back arrow on the far left, AND the settings gear on the far right. What remains: just the workflow pill, centred horizontally on the bottom edge.
  • 16. Workflow pill: 5 icons inside a raised-surface pill. Camera, Refine (list-rows glyph), Preview (slideshow), Publish (cloud-upload), Share (iOS share). Coral chevron triangles between each. Hit each one — same callbacks fire as before (Camera opens viewfinder; Refine opens editor for top card; Preview opens unified preview; Publish publishes or shows unlock pill if locked; Share opens iOS share sheet on a published plan).
  • 17. Locked plan — Publish slot replaced by Unlock icon. If you have a session past its 14-day grace, the Publish (4th) slot becomes a coral lock_outline icon. Tap — the unlock-1-credit bottom sheet opens. Same behaviour as before this PR.

6 · Plan-settings sheet — Statistics tab (item #4)

  • 18. Open the plan-settings sheet from the AppBar gear. The tab strip now has 4 tabs in this order: Now, Defaults, Plan, Statistics. The sheet still opens on Now by default. Tab labels are visible without truncation on a standard iPhone width.
  • 19. Tap Statistics on a published session that's been opened by the client. The tab body shows "First & last opened {date}" or split "First opened {date} · Last opened {date}" (depending on whether they're the same day) PLUS a lock-state row: coloured dot + label. Sage dot for fresh free-edit window; amber for <3 days remaining; coral for <24h; red dot + "Republish costs 1 credit" when locked; light-coral dot + "Unlocked · republish free" if unlock has been pre-paid.
  • 20. Drafts — "Not yet published" placeholder. Open settings on a draft (unpublished) session and tap Statistics. Body reads "Not yet published." plain text — not blank, not error-y. Now / Defaults / Plan tabs still work as expected for drafts (Plan tab shows v0 / "—" placeholders for cloud-only fields).
  • 21. Published-but-never-opened. If you have a session that's been published but the client URL has never been opened, the Statistics tab shows "Not yet opened" only — no lock-state row (the 14-day clock hasn't started). This is the right thing.
Implementation summary: app/lib/screens/home_screen.dart — help icon as Positioned in the existing Stack overlay (not a Material AppBar, since Home doesn't use one) + _openHelp launcher; app/lib/screens/client_sessions_screen.dart — AppBar's title slot now hosts ClientAvatarGlyph + dashed-underline name; body's duplicate header retired; rename TextField re-anchors to a slim section above the consent chip; app/lib/screens/studio_mode_screen.dart — brand-new _buildStudioAppBar with _StudioBackLeading (chevron + client label) + dashed-underline title + _StudioAppBarSettingsButton with optional coral dot; app/lib/widgets/studio_bottom_bar.dart — gutted to JUST the workflow pill; _resolveStatsLines lifted to a free function resolveStudioStatsLines with public StudioStatsLines / StudioLockTone types so the Statistics tab can reuse the truth; app/lib/widgets/plan_settings_sheet.dart — 4th tab "Statistics" added; _buildStatisticsTab renders the analytics row + lock-state line via the new shared resolver. No schema, no SQLite bump, no SharedPrefs key.
