2026-05-04 · fractional seconds + end-of-exercise badge retired

Device QA — clean integer timer + drop the End-of-exercise pill

Scope: shared web-player/ bundle — affects both session.homefit.studio and the embedded mobile workflow Preview WebView (R-10 parity).

Bug 1 — fractional seconds bleeding into the matrix top bar. Carl saw 23:54.66666666666674 in the matrix time and ...6666666666666 in the active pill remaining. Root cause: jumpToRep computed elapsedInPhysical = (repIdx / targetReps) * physicalSet, which is fractional whenever the physical-set seconds don't divide evenly by reps (e.g. 121s / 12 reps = 10.083). The fraction propagated into setPhaseRemaining and remainingSeconds, and the 1-second decrement loop preserved it for the rest of the slide.

Fix: Math.round((repIdx / targetReps) * physicalSet) at the source in jumpToRep. Belt-and-braces: formatTime now wraps its input in Math.floor(Number(seconds) || 0), and the active-pill showSecs wraps remainingSeconds in Math.floor.

Bug 2 — End-of-exercise badge dropped. Carl: “serves no purpose”. The green sage ✓ End-of-exercise pill that appeared on the rep stack during the final rest phase has been removed: the render-path aside is now empty for the final rest section, the paintActiveRepBlock gating is removed (dead code without the DOM element), and .end-marker CSS rules are deleted.

SW cache bumped: homefit-player-v32-vertical-weight-chip → homefit-player-v33-fractional-seconds-fix.
