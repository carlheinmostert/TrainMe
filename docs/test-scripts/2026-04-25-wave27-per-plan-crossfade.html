Wave 27 · device QA

Per-plan crossfade tuner + trim handle fix + play/pause rail

Three bundled changes. (1) Wave 26 item 5 — right trim handle no longer fights the video; _enforceTrimWindow is suspended while a handle is being dragged. (2) Wave 26 item 15 — dual-video crossfade is now per plan, with two practitioner-tunable sliders (lead-time + fade-duration) in a discreet bottom-sheet inside _MediaViewer. The native viewer now runs an actual two-controller crossfade so you feel the values change live as you tune. Web player reads the same per-plan values from get_plan_full; defaults (250 ms / 200 ms) preserved when fields are NULL. (3) Web player gets a YouTube-style play/pause toggle on the right rail above the maximize button — replaces the centered pause disc. Latest cache v60-per-plan-crossfade. Latest install-device commit 020438e.

A · Right trim handle (Wave 26 item 5 fix)

  • 1. Right trim handle scrubs to the dragged frame without yanking back to start.
    1. Open a video exercise (5 s+) in the full-screen _MediaViewer.
    2. Drag the right trim handle inward — video should pause, then scrub to the dragged frame in real time.
    3. Release — playback resumes within the new window. No yank back to start mid-drag.
    4. Drag the left handle inward to confirm it still works (was already correct).

B · Crossfade tuner sheet

  • 2. Tune gear opens the crossfade bottom-sheet.
    1. Open any video exercise in _MediaViewer.
    2. Find the small tune gear icon in the chrome (top-right area, just under the close X). Tap it.
    3. A bottom-sheet slides up titled "Loop crossfade tuning" with two sliders (Lead time + Fade duration) + a coral "Reset to defaults" link.
  • 3. Sliders cover sensible ranges; default values surface when untouched.
    1. On a never-tuned plan, both sliders read defaults: Lead 250 ms · Fade 200 ms.
    2. Lead slider scrubs 100 → 800 ms; Fade slider scrubs 50 → 600 ms.
    3. Current value text under each slider follows the thumb live.
  • 4. Slider changes apply live to the dual-video crossfade you can see.
    1. Drag Lead time → 700 ms and Fade duration → 600 ms.
    2. Watch the loop seam on the active video — should look like a long, slow dissolve. The next video starts well before the seam, blends generously.
    3. Drag both back to 100 / 50 ms. Seam now feels almost like a hard cut — tight overlap, fast fade.
    4. Effect is visible IMMEDIATELY without closing the sheet.
  • 5. Reset to defaults clears values + UI snaps back.
    1. Tune Lead + Fade to non-default values.
    2. Tap "Reset to defaults". Sliders snap to 250 / 200 instantly.
    3. Close + reopen the sheet — sliders still at default. Session row stored as NULL (so a republish writes NULL and the surface defaults take over).
  • 6. Tuned values persist across app restart (per plan).
    1. Tune Lead = 400 ms, Fade = 300 ms on a plan. Close _MediaViewer.
    2. Force-quit the app from app switcher. Reopen.
    3. Reopen the SAME plan + same video → tune sheet shows 400 / 300.
    4. Open a DIFFERENT plan → sheet shows defaults (each plan tunes independently).
