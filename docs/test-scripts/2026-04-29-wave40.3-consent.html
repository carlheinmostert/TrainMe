Wave 40.3 · consent redesign + audit log

Client visibility — collapsed-by-default + Avatar toggle + audit-logged

Three coupled changes across both surfaces. (1) The portal's /clients/[id] consent block is no longer the page's primary visual block — it's wrapped in a <details> accordion that ships collapsed on every route load. The summary header reads Visibility · {granted}/{total} granted. (2) The portal toggle list grows a fourth row, Avatar still, finally surfacing the Wave-30 consent slot the mobile sheet has had since 2026-04-23. (3) Every successful save drops a client.consent.update row into audit_events, with meta carrying a {from, to} jsonb diff. The /audit feed renders these rows with a sage chip + a description that names the keys whose boolean flipped (e.g. “Visibility for Aslan: B&W on, Avatar off”). The Wave 40.1 Client column populates correctly because the row's ref_id is the client uuid. R-11 mobile twin: the existing chip-then-bottom-sheet is the mobile equivalent of the portal accordion; the chip now mirrors the portal's headline (Visibility · {granted}/{total} granted). Avatar toggle was already present in the sheet (Wave 30) — it just gets its first audit-log entries now. 6 items.

Test items

  • 1. Portal · /clients/[id] consent block is collapsed by default; header chip shows the granted count.

    3. Reload the page. Confirm it returns to the collapsed state — expansion is intentionally not persisted.
    4. Navigate away to /clients and back. Same behaviour: collapsed on every route load.
    5. Granted count is correct. For a client with grayscale + colour off + avatar off, chip reads 1/4 granted (only line_drawing). For a client with all on, chip reads 4/4 granted.
  • 2. Portal · tap header — block expands; 4 toggles visible (line_drawing locked-on, grayscale, original, avatar).
    1. Tap (or click) the chip from item 1. The chevron rotates 180°; the panel expands downward.

    3. Below the video group, a second uppercase subhead reads Profile. Underneath it is a single toggle row: Avatar still with helper copy mentioning “Single capture with the background blurred… replaces the initials circle on {Name}”.
    4. Tap header again to collapse. Panel collapses cleanly. No visual jump on the granted-count chip during the animation.
  • 3. Portal · flip the Avatar toggle, save — toast confirms; new row appears in /audit with sage chip + practitioner actor + Client column populated.
    1. Expand the Visibility panel. Note the live granted-count chip at the top — it tracks the LIVE state, so flipping a toggle moves it before saving.

    5. Actor column = your practitioner email (NOT “—”). Client column = the client's name + a link through to /clients/[id].
    6. Description column reads roughly Visibility for {Name}: Avatar on (or “Avatar off” depending on direction). Other toggles unchanged ↠ only the moved key surfaces in the diff.
  • 4. Mobile · same flow on ClientSessionsScreen — chip headline matches portal; saving the sheet emits an audit row.

    3. line_drawing is true on both sides (locked).

  • 6. Regression · existing grayscale / original toggles still save without an avatar change being implied.
