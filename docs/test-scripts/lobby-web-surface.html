1 · Layout — what the client sees on URL open

  • 1. Lobby is the entry point. Open the published plan URL on the iPhone. Expected: the lobby renders directly — no "first card + Start" deck. The legacy in-deck start-workout button is hidden underneath.
  • 2. Default-named plan greets with Hi {ClientName}. For a plan whose title matches the auto-format DD MMM YYYY HH:MM, the headline reads Hi {ClientName} in Inter (not Montserrat). The sub-line reads {N} exercises · ~{N} min · From {Practitioner}. The date stamp sits below in dim mono.
  • 3. Custom-named plan uses the title as headline. Rename a plan to something like "Knee rehab phase 2" on mobile + republish. Expected on the lobby: headline is the custom title (Montserrat 22 bold). Sub-line begins with Hi {ClientName} · ... followed by the exercise/min/practitioner stats.
  • 4. Pill matrix sits above the hero list. The same pill matrix from the deck is rendered at the top of the lobby. On scroll, the matrix is sticky to the top edge so it stays visible while the hero list scrolls underneath.
  • 5. Hero rows: square hero on the left, info on the right. Each exercise row renders a 1:1 square thumbnail at ~40% width on the left; exercise name, dose-line (e.g. 3 × 10 · 30s rest · ~2:30), and italic notes (when present) on the right.
  • 6. Sticky CTA bar pinned at the bottom. A bottom bar with the treatment row (Line / B&W / Colour) above a coral Start workout button is fixed at the bottom of the viewport. iOS safe-area inset is honoured (no overlap with the home indicator). Backdrop is blurred so content underneath shows through faintly.

2 · Two-way coupling — list ↔ matrix

  • 7. Scroll list highlights matching pill. Slowly scroll down the hero list. Expected: as each row crosses centre, the matching pill in the matrix above takes the active-pill highlight; the row gets a subtle coral border. The matrix auto-centres the active pill horizontally.
  • 8. Tap pill scrolls list to row. Tap any pill in the matrix. Expected: the hero list smooth-scrolls so the matching row comes into view, centred vertically. Rest pills work too — tapping a sage rest pill scrolls to the rest row.
