Device QA — Camera capture trio

Purpose: verify three cloud-Claude PRs that all landed today and all touch the camera-capture path. Each section maps to one PR.

#196Raw-archive download suppression. Tapping B&W or Original within ~30s of a fresh capture used to fire the misleading red “Couldn't download original — try again.” toast. Now detects the “still archiving locally” case (video on disk + no archive yet) and shows a polite “{Treatment} still processing — try again in a moment.” toast instead. Auto-unlocks once the archive lands.

#197Auto-save originals to Photos (default ON). Every camera-mode photo + video is now also written to the iOS Camera Roll. Library imports are skipped (no duplicate). Toggle lives at Settings → Session capture → Save originals to Photos. iOS Photos “Add only” permission dialog surfaces naturally on first capture.

#198v8 hand-pose dilation for held equipment. Apple's person segmentation is intentionally person-only — dumbbells / bands / kettlebells used to fade into the background. v8 adds a VNDetectHumanHandPoseRequest pass that paints a filled disc at each detected hand on the upscaled mask, so held implements pop with the practitioner. Edge / line tuning (lo / hi / alpha / bgDim) unchanged — still locked at v7.

Out of scope: web player chrome (PRs are mobile-only); the older Studio long-press “Download” path (#197 is a NEW auto-save on capture, not a refactor of the existing manual save).

  • 12. v6/v7 line-drawing aesthetic preserved. Re-capture a body-weight exercise you've shot before (e.g. a push-up, a squat).
