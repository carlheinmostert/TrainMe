Device QA — Analytics mobile (consent toggle + plan stats + exercise stats)

Purpose: verify the three practitioner-facing analytics primitives: (1) the analytics consent toggle in the client visibility sheet, (2) plan-level stats (opened/completed/last) below each session card, and (3) per-exercise stats (viewed/completed/skipped) in Studio.

In scope: mobile app only. Consent toggle in the existing visibility sheet, plan stats on ClientSessionsScreen, exercise stats on StudioExerciseCard.

Out of scope: web player consent banner, transparency page, web portal analytics, event capture instrumentation.

Prerequisites

  • Branch feat/wave17-analytics-mobile installed to device.
  • At least one client with a published session that has been opened via the web player (so analytics RPCs return data).
  • At least one client with an unpublished session (to verify placeholder state).

A · Consent toggle

  • 1. Open a client detail page. Tap the Visibility chip. The consent sheet opens with an Analytics section at the bottom (below Profile), containing a single toggle labelled "Anonymous usage analytics" with subtitle "Track which exercises are completed or skipped ..." Default state: ON. The consent chip count updates to reflect 5 total slots (was 4).
  • 2. Toggle analytics OFF, tap Save. Consent chip updates count to reflect the change (one fewer granted). Re-open the sheet: the toggle is still OFF (persisted). Toggle back ON, save again: chip count reflects the restored grant.

B · Plan stats

  • 3. On a client with a published plan that has been opened by the web player, a stats line appears below the session card: Opened N× · X/Y completed · last {time} ago. The three numbers are present and legible in muted text.
  • 4. On a client with an unpublished session (draft), NO stats line appears (or a minimal dash placeholder). The card renders cleanly without visual artefacts below it.
  • 5. Relative time formatting works: recent opens show "Xm ago" / "Xh ago" / "Xd ago"; older opens show "Xw ago" / "Xmo ago". Never shows raw timestamps.

C · Exercise stats

  • 6. Open a published plan with web-player activity into Studio. Each exercise card shows a small stats bar below the playback caption: N/M viewed · N/M completed · N skipped in ~10.5pt muted Inter text. Exercises with zero events show no stats bar.
  • 7. The per-exercise stats match what the web player actually recorded (cross-check one exercise: open the web player, view the exercise through to completion, then re-enter Studio and verify the viewed/completed counts incremented).
