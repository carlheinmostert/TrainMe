2026-05-04 · Session card filmstrip hero

Device QA — Session card filmstrip hero

Scope: mobile only — the session-card row in the per-client sessions list now renders the first up-to-4 video Hero frames as a horizontal filmstrip background, with a left-to-right coral-tinted dark gradient over it so the title + chevron stay readable. Card minimum height bumped from ~80px to 104px (+30%) so the filmstrip has room to read.

Decisions (signed off): first N video exercises in session order (N = up to 4) · B&W treatment (matches existing card thumbnails, POPIA-friendly) · static (no carousel / no crossfade) · sessions with NO videos AND NO photos fall back to the existing coral-tinted dark surface · card height +30% (Carl capped at 30% — the mockup proposed +50%).

Surfaces touched: app/lib/widgets/session_card.dart only. No SQLite schema change. No web-player impact (R-10 not affected — this is a practitioner-only surface).

Out of scope: web player, web portal, the editor sheet's Preview tab, Home pre-Milestone-K screen.

Prerequisites

  • Branch feat/session-card-filmstrip-hero built and installed via ./install-device.sh (or simulator via ./install-sim.sh).
  • At least one client with a mix of session shapes:
    • A multi-video session (≥ 4 videos) — for the 4-cell flex check.
    • A 2- or 3-video session — for the equal-flex check.
    • A single-video session — for the 1-cell-fills-card check.
    • A photo-only session (no videos, ≥ 1 photo) — for the photo-fallback check.
    • (Edge case) a rest-only session — for the no-filmstrip fallback. If you don't have one, skip item 9 and note in pass-comments.

1 · Filmstrip rendering

  • 1. Filmstrip on every video session. Open Clients → tap any client. Each session card with at least one video shows a filmstrip background — not the old plain dark surface.
  • 2. Up to 4 equal cells with hairline separators. A multi-video session (4 or more videos) renders exactly 4 equal-width cells. A 1-px black hairline sits between adjacent cells — no rounded gaps.
  • 3. Cell content matches the Hero frame. The image inside each cell looks like the Hero frame for that video (same image you'd see in the editor sheet's small header thumbnail / Studio mini-preview). Confirms it's pulling thumbnailPath, not a random raw frame.
  • 4. B&W treatment. Filmstrip cells render in greyscale — no colour bleed. Matches the look of the existing session-card thumbnails elsewhere (Studio cards, Camera peek box). POPIA-friendly: client's clothing colours / skin tone don't read as colour at all.

2 · Text + chrome legibility

  • 5. Title + version+date readable across light AND dark cells. Compare the title (e.g. “Jan Smith · 19 Apr 2026 17:09”) and the v3 · 25 Apr subtitle on a card whose right-side cell has a light Hero frame (e.g. white wall) vs one with a dark frame. Both stay legible — the text-shadow does its job.
  • 6. Status pills readable with backdrop blur. A session with the “1 converting…” pill or the “N failed” retry pill renders that pill against any cell content (the pill has a subtle backdrop blur so the coral text + spinner stay clear). No bleed-through, no blown-out text.
  • 7. Leading icon + count badge anchored. The 60×60 coral list-icon + the count-badge (e.g. “12”) sit cleanly on top of the leftmost gradient (which is 92% dark). Icon doesn't disappear into a brightly-lit hero behind it. Count badge stays sharp (white digits on solid coral, dark border).

3 · Cell-count flex strategies

  • 8. Single-video session. A session with exactly 1 video renders 1 cell that fills the full card width — no gaps, no hairline.
  • 9. 2- and 3-video sessions flex correctly. A 2-video session renders 2 cells at 50% width each; a 3-video session renders 3 cells at ~33% each. Hairlines between adjacent cells, never trailing.
  • 10. Photo-only session falls back to single cell. A session with no videos but at least one photo renders a single cell using the first photo's line-drawing thumbnail. (Photo cells stay as the line drawing — not B&W — because the converted file is already line art.)
  • 11. Rest-only / empty session — no filmstrip. A session with zero videos AND zero photos (rest-only edge case) falls back to the original coral-tinted dark surface. No broken-image icons, no half-rendered cells. The card still reads correctly. (Skip with a note if you don't have a rest-only session handy.)

4 · Card height + scroll

  • 12. ~30% taller than yesterday's build. Cards are visibly taller than before this PR — previous build was ~80px effective, this build is ~104px minimum. Long client lists scroll smoothly — no jank from image decode (the cells use a 240-px cacheWidth so decode stays cheap).
  • 13. Fresh install before SyncService finishes — graceful fallback. If you reinstall and immediately scroll the client → sessions list before the cloud session pull lands, sessions whose thumbnails aren't on disk yet show a dark fill in their cells — not broken-image glyphs. Once the pull completes and you scroll back, the filmstrip populates correctly.
  • 14. Hero pick updates the cell. Open a video exercise, change the Hero frame on the trim panel (drag the white ★ handle to a different point), close the editor, return to the per-client sessions list. The corresponding cell in the filmstrip updates to the new Hero frame.
