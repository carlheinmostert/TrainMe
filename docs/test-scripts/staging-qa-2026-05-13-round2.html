Device QA — Staging QA Round 2 (tasks A-F)

Purpose: follow-up wave covering Carl's six round-2 QA findings on top of the PR #316 staging build. Six tasks, all fixed in one branch.

Fixes in this PR:
· Task A — AppBar "My Workouts" capsule still truncated after PR #316. Tightened padding (16 → 12px outer, 8 → 6px gap), rebalanced flex (195:100 → 165:100), shortened label to single-word "Workouts".
· Task B — Prep-phase hero image now honours the active treatment class. Photo heroes carry the grayscale class for the whole slide; video heroes carry it during prep. No more colour-flash before B&W playback.
· Task C — Body-focus toggle now renders DISABLED on photo exercises (web gear popover + mobile MediaViewer pill) with the tooltip "Body focus is available for video exercises only."
· Task D — Lobby share-as-static export reliably generates in embedded WKWebView (drops mode: 'cors' for the custom homefit-local:// scheme that was hanging the preload). Embedded surface routes the PNG through HomefitBridge → UIActivityViewController. Modal hint copy switches per environment (desktop / mobile-touch / embedded).
· Task E — Circuit animation: getTotalLength() returns 0 on first read inside WKWebView (layout pass hasn't run); now retried on next animation frame. WAAPI is also wrapped in try/catch + getAnimations() existence check.
· Task F — Photo filmstrip cacheWidth now scales with cell count (1 cell → 720, 2 → 480, 3 → 320, 4 → 240). Single-photo sessions no longer decode-soft.

R-10 parity: all web-player edits synced into app/assets/web-player/ via dart run tool/sync_web_player_bundle.dart.

In scope: Home AppBar capsule layout · web player prep-overlay hero · gear-popover Body focus · mobile MediaViewer Body focus pill · lobby share PNG modal + native share bridge · circuit lane WAAPI tracer · filmstrip cell decode width.

Out of scope: Artifact visual content tweaks (Carl will review separately) · referral / payments paths · Apple Reader-App restrictions.
