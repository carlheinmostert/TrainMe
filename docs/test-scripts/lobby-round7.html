Device QA — pyramid in circuits + hold-qualifier revision + scroll-top guard

Three follow-ups on round 6. One is a real rendering bug Carl just hit live; the other two close gaps round 6 left behind.

1) Pyramid in circuits. Plan 5d24085b-2a3c-4b2f-9592-a842b4882090 has an exercise (position 2, in a circuit) with 3 sets [12, 10, 8]. The lobby was rendering it as 1 × 12 reps instead of 12/10/8 reps. Root cause: buildDoseLine used playSetsForSlide() which is deck-oriented — for a circuit slide it filters to ONE round's single set. The lobby groups circuits and renders one row per exercise (not per round), so it needs the FULL sets[] array. Fix: new allSetsForSlide() helper exposed on HomefitLobbyHandoff that ignores circuitRound and always returns every authored set.
2) Hold-qualifier grammar revision. Round 6 made end_of_set render bare (no qualifier). Carl wants the qualifier ALWAYS present: per_rep → (per rep); end_of_set → (per set); end_of_exercise → (after last rep). All lowercase. Reads more consistently than the implicit-default convention.
3) Scroll-top guard. Round 3 added a scroll-bottom guard for the last row; symmetric guard never landed for scroll-top. Symptom: at scrollTop = 0 the second row wins the nearest-to-centre reducer, so the first row's pill never highlights when the user scrolls back up to the top. Fix: mirror of the bottom guard, scrollTop ≤ 4 → activate first non-rest row.

Bumps: PLAYER_VERSION → v43-grammar-revision-and-fixes; CACHE_NAME → homefit-player-v43-grammar-revision-and-fixes.
