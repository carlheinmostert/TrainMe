Device QA — Studio visual hygiene pass (Wave 18)

Purpose: verify 10 locked-spec UI refinements that together re-shape Studio into a calmer, single-source-of-truth surface. Biggest pieces: the new StudioToolbar that unifies Import/Preview/Publish/Share (with coral triangles echoing the state of the adjacent icon), exercise-card section groups PLAYBACK / DOSE / PACING / NOTES, preset chip rows replacing the three Reps/Sets/Hold sliders + the Rest sage slider, practitioner-wide custom-preset memory (SharedPreferences, MRU cap 3), and the storage-level saveExercise stamp that closes the long-running publish-dirty regression from Camera-mode captures.

In scope: mobile-only. Client detail header loses the session count; SessionCard loses Publish + Share; Studio toolbar gains them; exercise card dividers replaced with labelled groups; chip rows; MediaViewer mute persists via includeAudio; MediaViewer dots coral; _StatusDot deleted.

Out of scope: web player (no R-10 mirror this wave — all changes are practitioner-facing), portal, backend. Still-open Wave 17 verification, analytics consent work.

Prerequisites

  • Main merged through PR for Wave 18. feat/wave-18-studio-visual-hygiene branch landed.
  • Install to device via ./install-device.sh from the repo root.
  • At least one published session on the device (needed for Publish/Share state tests) and one unpublished session (for dim-state tests).

1 · Navigation + Client detail

  • 1. Home (clients list). Settings icon sits top-right of the app bar and is tappable. Tapping opens Settings. No other top-bar glyphs crowd the row.
  • 2. Tap into a client. Below the editable client name there's a row with the Client consent chip and NOTHING ELSE — the "N sessions" count is gone (the session list below is the count).
  • 3. SessionCard (each session in the list) ends with a chevron only. The Publish cloud icon and Share icon are gone — they moved into Studio. Swipe-left delete still works; tap still opens the session.

2 · Studio toolbar

  • 4. Open any session into Studio. Below the title bar is a 52pt-high toolbar with four icons in order: Import · Preview · Publish · Share, each separated by a small right-pointing coral triangle. Icons centred as a group, not edge-spread.
  • 5. On an UNPUBLISHED session (version 0), the Share icon is dim (45% opacity) and the triangle between Publish and Share is also dim (the triangle "feeds into" the dim icon — Variant C1 rule). Tapping Share does nothing and fires no haptic; long-press still shows the tooltip.
  • 6. No exercise-count summary chip renders above the list. The first thing below the toolbar is the exercise list itself (or the empty-state message).
  • 7. On a PUBLISHED session with no unsent edits, the Publish icon is a sage check_circle. Tap it → nothing (clean state, no-op). On a published + dirty session (edit any reps after publish), the Publish icon flips to a coral cloud_upload_outlined; tap → publish fires; a snackbar confirms "Published v{N+1}".
  • 8. While publish is in flight, the Publish icon becomes a 22pt coral CircularProgressIndicator. Triangles feeding into it + Share remain coral (publish and share are "active" during publishing).

3 · Exercise card groups + chip rows

  • 9. Tap an exercise card to expand it. The panel shows four labelled groups in 11pt coral uppercase: PLAYBACK, DOSE, PACING, NOTES. No more grey Divider(height: 1) rules between sections — whitespace + labels carry the separation.
  • 10. On a freshly-captured exercise (all fields at defaults, no notes), PACING and NOTES render collapsed with a trailing em-dash summary: ▸ PACING — defaults and ▸ NOTES — empty. Tap the row to toggle expand; chevron rotates.
  • 11. DOSE section: REPS shows a horizontal chip row [5][8][10][12][15][Custom], coral-tinted. SETS shows [1][2][3][4][5][Custom] (hidden when the exercise is inside a circuit). HOLD shows [Off][5s][10s][30s][60s][Custom]. Each row has the current value on the right of its label.
  • 12. Tap a non-current chip (e.g. [12] for Reps). Haptic selectionClick fires. The chip fills coral with white text; the previous chip un-fills. The exercise row summary ("3 × 12 · ~36s") updates.
  • 13. Tap [Custom]. The chip row replaces in-place with < Cancel [_N_] Done > — no bottom sheet, no modal (R-01). Type 13 on the numeric pad. Tap Done. Haptic mediumImpact fires. A new [13] chip appears in the row, sorted between 12 and 15, with a small coral dot on its top-right corner (the "this is a custom value" affordance).
  • 14. Open a DIFFERENT exercise card. The [13] chip is still there (practitioner-wide MRU via SharedPreferences). Long-press the [13] chip. It's removed immediately; an Undo SnackBar appears reading "Removed reps 13". Tap Undo — the chip returns.
  • 15. Rest bar (swipe right between two exercises to add one, or long-press). The slider is gone. In its place: the rest row renders [icon] Rest [15s][30s][60s][90s][Custom]. Chips are sage, not coral. Tap [60s] → the rest row's hold persists; swipe-left to delete the whole rest row still works.

4 · Status dot + Media Viewer

  • 16. Exercise-card header (collapsed) shows: thumbnail + name + summary line. The right-side status dot (green check / red warning / spinner) is GONE. The thumbnail itself still carries the conversion-state overlays (green check when done, red warning when failed, centre spinner while converting) — zero information loss.
  • 17. Open Media Viewer (tap a video thumbnail). The mute affordance is a coral pill at BOTTOM-LEFT, not a top-right icon button. Pill reads "Audio on" with a coral outline + coral icon when the current exercise has includeAudio = true; reads "Muted" with a coral fill + white icon when false. Tap fires selectionClick haptic and the video's volume flips immediately.
  • 18. Close the Media Viewer. Re-open it on the same exercise. The pill state is the SAME as when you left (it persisted via saveExercise, not a transient flag). Swipe to a neighbour exercise — the pill reflects that exercise's includeAudio, which can differ from the previous exercise.
  • 19. Media Viewer page-dot indicator at the bottom: active dot is coral; inactive dots are a 25%-alpha coral tint (not white / white30 as before).

5 · Publish-dirty storage fix

  • 20. Publish a session so Studio shows the sage check_circle. Swipe to Camera mode and capture ONE new exercise. Swipe back to Studio. The toolbar's Publish icon has flipped to a coral cloud_upload_outlined — the new-capture path now stamps last_content_edit_at inside saveExercise, closing the old regression where Camera captures left the publish indicator stuck on sage.
  • 21. On the dirty session from item 20, publish via the toolbar. The snackbar reads "Published v{N+1}"; Publish icon goes back to sage check_circle; hasUnpublishedContentChanges becomes false. The card on ClientSessionsScreen reads Published v{N+1} · {date} correctly.
