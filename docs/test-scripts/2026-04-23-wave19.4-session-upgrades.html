Device QA — Wave 19.4 session upgrades

Purpose: verify the session's cross-stack work that followed Wave 19.3. Spans native iOS, Flutter Studio, Supabase schema, and the unified web-player bundle.

Feature slices under test:
(A) Capture defaults — fresh captures persist sets=3 + reps=10 so published grammar no longer collapses to "X reps" fragments.
(B) Publish robustness — upsert_client errors now fail hard with a user-visible SnackBar (previously swallowed; produced orphan plans with no client link).
(C) Grammar fix — removed isometric short-circuit; the active-slide title always carries the reps fallback.
(D) iPhone faux-fullscreen — Safari doesn't support Element.requestFullscreen; a body-class toggle now gates ambient mode.
(E) Native v7 segmentation — backgroundDim 0.70 → 0.50 for stronger subject pop on line drawings.
(F) Milestone P — segmented-colour variant — native dual-output produces a second mp4 where the body is pristine + background dimmed; web player prefers it for Colour + B&W treatments, falls back to untouched original for legacy plans.
(G) Segmented-effect toggle — a gear-button popover in the card chrome lets clients opt out of (F); default ON; persisted in localStorage.
(H) Milestone P2 — mask sidecar — third output emits the Vision segmentation mask as its own mp4; uploaded to raw-archive; get_plan_full exposes mask_url. No player consumer today — door open for future playback-time tuning.
(I) Milestone Q — inter-set rest ("Post Rep Breather") — per-exercise inter_set_rest_seconds, default 15s on fresh captures; top-of-video segmented bar renders [Set 1 · 10 reps][Rest 15s][Set 2 · 10 reps]…; video plays continuously, pausing on-frame at set boundaries and resuming into the next set.
(J) Download original video — long-press a video card's thumbnail in Studio → new "Download original" action in the context menu (video-only). Opens a bottom sheet with Save to Camera Roll · Share · Cancel. Local-first (reads the 720p H.264 raw archive under {Documents}/archive/), falls through to a signed raw-archive URL via the new ApiClient.signRawArchiveUrl helper when the local copy is missing / evicted.

In scope: the unified web-player bundle (web-player/*), the Flutter trainer app (app/lib/...), the native video pipeline (app/ios/Runner/VideoConverterChannel.swift), and Supabase schema (Milestones P, P2, Q applied). R-10 parity holds — the mobile practitioner preview runs the same bundle via the homefit-local:// scheme handler.

Out of scope: Portal changes. PayFast webhook. Edge middleware. Login / auth. Audio mux (unchanged from v7). Android (still iOS-only).
