Device QA — Web-player polish round 2

Purpose: verify four targeted polish changes to the unified web-player bundle (web-player/). Lands on feat/web-player-polish-round2. Four changes: (1) Title row promoted above the time row — exercise name + decoded grammar now precedes the 3-number ETA row. (2) Circuit pill grid flipped from column-first to row-first: each row = one round (all exercises left-to-right), rounds stack downward, coral tint band wraps each row. (3) Exercise notes redesigned as a centered dark soft-card box with subtle coral border + backdrop blur (replaces plain coral text). (4) Matrix pill max-width bumped 22 → 32px so single (non-circuit) pills render as rectangles on short sessions instead of near-rounded squares.

In scope: web-player bundle (web-player/index.html + styles.css + app.js + sw.js); propagates to the Flutter practitioner preview via the WKURLSchemeHandler (R-10 parity enforced by the unified-player bundle).

Out of scope: Flutter code. Backend / Supabase. Middleware. Portal. Audio / treatment logic (unchanged). Desktop / landscape layouts.

Prerequisites

  • feat/web-player-polish-round2 deployed to the Vercel preview URL (auto-deploys on push), OR the branch installed onto the device via ./install-device.sh so the embedded WebView picks up the new bundle.
  • Hard-reload the player URL — the service worker bumps from v22-matrix-fit-to-width to v23-player-polish-round2 but needs one cold load.
  • A published plan with: at least one exercise that has practitioner notes, and at least one circuit with ≥ 2 exercises and ≥ 2 rounds (so the row-first grid is observable).

1 · Title row above time row

  • 1. Open the plan link in Safari. The chrome above the video reads, top-to-bottom: (a) plan title + client name (with slide progress on the right); (b) the exercise name + decoded grammar line (e.g. Goblet squat · 3 sets · 10 reps); (c) the 3-number ETA row full-width centered (current · total · ~finish); (d) the progress-pill matrix. The order is title → time → pills (NOT time → title → pills as before).

2 · Circuit pills read row-first (round as band)

  • 2. Open a plan with a circuit of ≥ 2 exercises and ≥ 2 rounds. Inside the progress-pill matrix, the circuit reads as a stack of horizontal bands. Each band = one round, with all that round's exercises left-to-right (A, B, C). Rounds stack downward. The coral tint wraps the ROW, not the column. Contrast with the old layout where each column was one exercise's rounds stacked vertically inside a coral column-band. Non-circuit (single) pills between circuits are unaffected.

3 · Single pills render as rectangles

  • 3. On a short plan (≤ 8 slides so flex-distribution gives each pill real estate), single non-circuit pills look like taller-than-wide rectangles with distinct rounded corners — they do not read as near-round circles anymore. (Previously max-width was 22px and singles squared off to ~14–18px × 20px; Wave 19 bumped max-width to 32px so there's room for a clear rectangular silhouette.)

4 · Notes render as centered soft-card

  • 4. On an exercise slide that has practitioner notes, the note renders as a centered dark soft-card sitting roughly 12% up from the bottom edge of the video (not flush, not left-aligned). The card has a subtle coral border (1px at ~35% alpha), a backdrop blur behind it so the video softens where it overlaps, rounded corners (~10px), and off-white text (not coral — coral is reserved for accent elements). Max width is ~72% of the video; min width keeps single-word notes from shrinking to nothing.

5 · Notes tap expands, tap again collapses

  • 5. Use an exercise with notes long enough to exceed 3 lines. On first render the notes card clamps to 3 lines with an ellipsis. Tap the card → the full note expands inline; the card grows taller but stays horizontally centered + within the max-width. Tap again → it collapses back to the 3-line clamp. Navigating to another slide and returning resets it to the 3-line clamp.

6 · Fullscreen preserves title + time + matrix

  • 6. Tap the fullscreen toggle (top-right of the video). The plan-bar + footer hide (per existing fullscreen ambient mode), but the active-slide title, the 3-number ETA row, and the progress-pill matrix all remain visible — in their new order (title above time). The coral overlays (mute, chevrons, notes card, pause disc) drop to ~30% alpha; the fullscreen toggle itself stays at 100%. Exit fullscreen and the plan-bar / footer restore.

7 · Service worker bumped cleanly

  • 7. Open Safari → Advanced → Web Inspector → Storage on the session.homefit.studio (or preview) page. The Cache Storage name reads homefit-player-v23-player-polish-round2 (not the old v22-matrix-fit-to-width). If the old cache lingers, one more hard reload clears it. No console errors from app.js about missing DOM refs or broken buildMatrixColumns references.
