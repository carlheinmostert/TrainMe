Device QA — CAPS labels bold first letter

Scope: tiny visual polish on the Studio bottom CAPS toolbar. Each label's first letter (CAPTURE, ADJUST, PREVIEW, PUBLISH, SHARE) renders at FontWeight.w900 while the rest of the letters stay at the existing FontWeight.w600 — a passive nod to the CAPS mnemonic. Implementation swaps each cell's plain Text for Text.rich with two TextSpans; the FittedBox(scaleDown) wrapper, the 7pt size, the uppercase letter-spacing, and the disabled-dim colour rule all stay identical. Both _CapsCell and _PublishCapsCell updated.

Prerequisites

  • PR for branch polish/caps-bold-first-letter merged + ./install-device.sh from repo root.
  • One client with at least one captured exercise (so the ADJUST cell is bright/active for item 2 dim-state contrast).
  • One client with no exercises (or only rest periods) so ADJUST/PREVIEW/PUBLISH/SHARE all render dim for item 2.

1 · First-letter weight reads heavier

  • 1. Open any session in Studio. Inspect the CAPS toolbar. The first letter of each label — C in CAPTURE, A in ADJUST, P in PREVIEW, P in PUBLISH, S in SHARE — reads visibly heavier/bolder than the rest of the letters in the same label. The eye should pick up the CAPS acronym.

2 · Bolding survives in dim state

  • 2. Open a session with NO captured exercises. ADJUST / PREVIEW / PUBLISH / SHARE all render dim. The first-letter weight differentiation still works in the dim state — the leading letters are bolder than the rest, just at the dim grey colour. CAPTURE (always active) shows the same heavier-leading-letter treatment in its bright state.

3 · No clipping from the heavier weight

  • 3. Look closely at the longest labels (CAPTURE and PUBLISH). The heavier first letter does NOT push either label into clipping or ellipsis. All five labels still fit cleanly inside their 56-wide cells. The FittedBox(scaleDown) wrapper handles any unusual font-scaling case so labels never overflow.
Implementation: app/lib/widgets/studio_bottom_bar.dart — _CapsCell and _PublishCapsCell swap plain Text for Text.rich with a leading TextSpan(fontWeight: w900) + remainder TextSpan inheriting parent w600. FittedBox(scaleDown), fontSize 7, letter-spacing 0.4, colour resolution all preserved. Defensive empty-label fallback in _CapsCell to keep substring(1) safe. No schema, no SQLite bump, no SharedPrefs key.
