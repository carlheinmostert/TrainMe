B · Portal client avatar image (Wave 40 items 14 + 16 follow-up)

  • 5. B1 · /clients list shows the body-focus avatar JPG when present; falls back to initials when not.
    1. Setup. On the mobile app, pick a client that does NOT yet have an avatar. Go to their detail screen, tap the avatar slot, capture a body-focus still through the in-app camera. Wait for the upload to land (the avatar disc on the mobile screen swaps from initials to the blurred image).
    2. Hard-refresh manage.homefit.studio/clients in Safari (Cmd+Shift+R). Sign in if needed.
    3. The card for the client you just photographed shows the same body-focus blurred avatar as a 40×40 disc on the left of the card — not the initials chip you saw before. Hover-tooltip is the client name.
    4. A second client that has NEVER had an avatar captured continues to render initials (e.g. JS on coral). Confirm both states coexist on the same list.
    5. Right-click the avatar → "Open Image in New Tab". The URL contains /storage/v1/object/sign/raw-archive/… with a ?token= query string — the signed URL minted by the new RPC. Image loads.
    6. If a client has multiple sessions: the count + "shared X ago" copy below the disc is unchanged from Wave 40 (only the avatar idiom is new).
  • 6. B2 · /clients/[id] session-icon shows the same avatar idiom — matches mobile SessionCard.
    1. From /clients, click into the client you photographed in step 5. The page is /clients/{id}.
    2. Scroll to the Sessions section. Each row in the desktop table (or card on mobile) has a 40×40 session-icon disc on the left.
    3. The disc renders the body-focus blurred avatar — the same image you saw on the /clients list. The exercise-count badge (small coral pill, bottom-right of the disc) overlays cleanly on the image.
    4. Open the same client's detail screen on the mobile app. Side-by-side, the SessionCard avatar and the portal session-icon should look identical — same image, same crop, same body-focus blur. (R-10 parity.)
    5. For a different client whose avatar hasn't been captured: the session-icon shows initials, with the same exercise-count badge. Both states coexist correctly.
    6. Hover (or long-press on iPad) the badge: tooltip reads N exercises (or 1 exercise for singular). Hover the avatar disc itself: tooltip is the client name.

Out of scope for this wave

  • Re-blurring or re-processing the avatar on the portal — we render whatever bytes the mobile app uploaded. The privacy guarantee (face/body abstraction) is intrinsic to the JPG. Wave 30 owns the pipeline.
  • Caching the signed URL across renders — URLs expire after 1 hour. Each page load mints a fresh one (cheap; the RPC is one round-trip already underway).
  • Surfacing the avatar in the /clients/[id] header card — ClientDetailPanel layout is unchanged. Carl's QA flagged the list and the session-icon, both of which are now done.
  • Dashboard / audit / members / network avatar treatment — not asked for. list_practice_audit + the audit table use the practitioner's email + name, not the client's avatar.
