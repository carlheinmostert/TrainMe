2026-04-22 · Wave 18.6 — polish round 3

Device QA — Wave 18.6

Purpose: five items from live QA of Wave 18.5. (1) [+] horizontal padding 6pt → 12pt so the pill reads as a proper pill (~42pt wide), matching the visual weight of text chips. (2) Rest-row outer container flipped from surfaceRaised to surfaceBase so the unselected chips (which fill with surfaceRaised) sit against a contrasting background — same chip-on-surface contrast as PLAN chips. (3) Collapsed group headers tightened: inner vertical padding 10pt → 2pt, inter-group SizedBox(height: 16) spacers removed entirely. Groups flow directly against each other with no divider lines. Card stack is visibly much shorter when all four groups collapsed. (4) "Duration per rep" label stacks above its control (segmented or single pill) instead of sharing a horizontal row — fixes the iPhone overflow where the segmented control scrolled off-screen. (5) Font rebalance across accordion groups: section headers (coral w700 uppercase), inner labels (grey w600 uppercase) and inner values (mono w700 white) ALL at 13pt. Hierarchy carried by font family + weight + colour + case alone.

In scope: mobile-only. preset_chip_row.dart ([+] padding bump) + studio_mode_screen.dart (rest row background) + studio_exercise_card.dart (group header spacing, Duration per rep stacked layout, font rebalance).

Out of scope: anything already covered by Wave 18 / 18.1 / 18.2 / 18.3 / 18.4 / 18.5. Web player. Portal. Storage semantics of customDurationSeconds unchanged (still total = per-rep × reps). Chip long-press remove/undo, [+] long-press no-op, inline (not modal) input — all unchanged.
