2026-05-16 · html2canvas video-frame fix · export swap

Device QA — PDF active-row video swap to img

Bug. When the lobby's active row was a video, the PDF export captured a solid grey block in place of the hero image. Cause: html2canvas can't reliably rasterise a <video> element on iOS WKWebView — mid-play or auto-paused, the rasteriser falls back to a grey fill instead of reading the current frame.

Fix. In buildExportPageElement, after cloning each row, walk the clone's <video> elements and replace each with an <img> whose src is the already-cropped data-poster-src data URL. The className, style, and dataset are carried across so any downstream consumer sees the same shape. html2canvas rasterises <img> with no issues. Zero extra work — the poster URL is already cropped by hydrateHeroCrops.

R-10 parity. Both web-player/lobby.js and app/assets/web-player/lobby.js updated.
