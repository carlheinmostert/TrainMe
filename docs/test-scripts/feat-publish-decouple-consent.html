Device QA — Publish: decouple consent from upload

Purpose: verify the consent gate has moved out of _uploadRawArchives in app/lib/services/upload_service.dart. Before this PR: B&W / colour / segmented / mask files only uploaded when the client's video_consent jsonb allowed that treatment. After this PR: every variant file uploads on every publish, regardless of consent. Visibility stays gated on the player side via get_plan_full's NULL signed URLs.

Why this matters: today, if a client revokes B&W consent and later grants it again, the practitioner has to re-publish to push the missing files. After PR-A those files are already in raw-archive — flipping consent back on works without a republish.

Scope: diagnostic-only change in spirit. No user-visible behaviour change today (a fully-consented client still sees all three treatments; a fully-revoked client still sees only the line drawing — the gate moved, not the surface).

Out of scope (saved for PR-B & PR-C): default treatment swap to B&W, soft fallback to line drawing when preferred treatment lacks consent, atomic publish UI / progress sheet, "partial success" toast removal.

Prerequisites

  • Build from feat/publish-decouple-consent-from-upload installed on Carl's iPhone via ./install-device.sh.
  • At least one existing client + ability to flip their treatment consent toggles in the consent accordion (Studio → client card or portal /clients/[id]).
  • A short capture session ready to record: 1 video + 1 photo (enough to exercise both raw + segmented + colour-thumb + mask paths).

  • 2. Open the test client's clients-screen, create a fresh session, capture 1 video + 1 photo, let conversion settle to a green check on both. Publish. Expect: “Published ✓” toast (happy path). device

The previous behaviour would have silently skipped the colour-thumb + photo raw uploads for this revoked-consent client; the new behaviour uploads them anyway, so any failures here would be a real upload error, not a consent skip. If no toast appears at all — even better (means all variants pushed clean). device
