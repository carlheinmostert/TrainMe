Wave 41 · ghost-exercise fix

Studio: await delete, missing-media banner, better publish error

Diagnosed bug: Studio's in-memory list and SQLite drifted when storage.deleteExercise failed silently. UI showed the exercise gone; pre-flight saw the orphan and rejected publish with a "missing media" name the practitioner couldn't see in Studio. (1) Delete is now awaited, with rollback + red SnackBar on failure. (2) Studio cards render a red "Media missing" banner when the line-drawing/raw file is absent — mirrors the upload pre-flight check exactly. (3) Publish pre-flight failure now surfaces a coral-bordered SnackBar AND auto-scrolls the first broken card into view. 3 sections, 6 items.

A · await delete + rollback (1 item)

How to force a SQLite delete failure on the simulator (one approach): With the app installed, run xcrun simctl get_app_container booted studio.homefit.app data to find the data container path. Locate Documents/raidme.db. Set the file read-only with chmod 444. Re-open Studio and try to delete an exercise. After the test, restore with chmod 644. Alternative: close the app cold, lock the file with another process holding the SQLite WAL open, then re-launch. If neither is feasible, mark item 1 as skip in the note (via × + "skip-env" note); the unit test in app/test/studio_delete_rollback_test.dart covers the rollback contract.
  • A1. Forced delete failure: card pops back + red error SnackBar fires.
    1. In a session with at least 3 exercises, force the underlying SQLite file to fail writes (see tip above).
    2. Long-press an exercise thumbnail → tap Delete (or swipe-left to delete).
    3. Verify the card briefly disappears, then snaps back into the list at its original position.
    4. Verify a red SnackBar appears with copy "Couldn't delete '{name}' — try again." (4-second duration.)
    5. Verify position numbers on all other cards are unchanged from before the attempt.
    6. Restore write permission. Tap Delete on the same card again. This time it deletes cleanly with the standard "{name} deleted · Undo" SnackBar.

B · missing-media banner on Studio cards (3 items)

How to inject an orphan row for QA: Connect the iPhone via USB, navigate to the app's data container in Xcode → Window → Devices and Simulators → select device → Installed Apps → homefit.studio → Download Container. Open Documents/raidme.db in DBeaver or sqlite3. Find any session_id (SELECT id FROM sessions LIMIT 5). Insert a fake exercise row pointing at a non-existent raw file:
INSERT INTO exercises (id, session_id, position, raw_file_path, media_type, created_at, name) VALUES ('orphan-1', '<sessionId>', 99, 'raw/does-not-exist.mp4', 1, strftime('%s','now')*1000, NULL); Replace the container, re-open the app, navigate to the session.
  • B1. Orphan row renders with a red "Media missing" banner.
    1. Inject an orphan exercise row pointing at a non-existent raw file (see tip above).
    2. Open the affected session in Studio.
    3. Verify a red banner reads "Media missing — long-press to delete and recapture" above the card header.
    4. Verify the banner sits inside the card and does not push the layout in unexpected ways. Other cards in the same session look normal.
    5. Long-press the orphan card thumbnail → Delete. Verify the card disappears (real delete now succeeds because the file lookup is null-safe).
  • B2. Healthy exercises do NOT show the banner.
    1. Open any session with normally captured exercises.
    2. Confirm none of them show the red "Media missing" banner.
    3. Tap and play one to verify normal media playback still works.
  • B3. Rest periods never show the banner.
    1. In a session with a rest bar between two exercises, confirm the rest row never renders the "Media missing" banner regardless of any other broken neighbour.
