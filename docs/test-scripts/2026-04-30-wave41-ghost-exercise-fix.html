Studio: await delete, missing-media banner, better publish error

When storage.deleteExercise failed silently, the UI removed the row but SQLite kept it; publish pre-flight then failed on a ghost the practitioner could not see. (1) Delete is awaited with rollback + red SnackBar on failure. (2) Cards show a red “Media missing” banner when line-drawing/raw file is absent (same check as upload pre-flight). (3) Pre-flight publish failure shows a coral-bordered SnackBar and scrolls to the first broken card. 6 items, 3 sections.
