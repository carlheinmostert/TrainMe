2026-05-15 - Circuit observer loop (NINTH attempt)

Device QA - Circuit preview stays responsive

Ninth attempt at a recurring bug. PR #353 (eighth attempt) shipped a MutationObserver feedback loop that pegged the main thread on every circuit plan: preview goes black, share button non-responsive, lobby thumbnails starved. This PR adds an observer disconnect/reconnect around the SVG mutations so we no longer observe our own writes. Sensitive zone - Carl review before merge.

Why PR #353 broke things. PR #353 added MutationObserver on the circuit frame ({ childList: true, subtree: true }) so that genuine row swaps (treatment toggles, prep transitions) would re-trigger a paint. The trouble: inside paintLanesAndTracer we ourselves mutate the SVG - clear children via removeChild, append new lane paths, append the tracer. Each of those mutations re-enters the observer, which queues another rAF, which calls renderCircuitLanesFor, which calls paintLanesAndTracer, which fires the observer again. Infinite loop. The __laneRenderRaf guard does not break it because it is nulled INSIDE the rAF callback before paint runs.

The ninth-attempt fix:
- Grab the observer reference saved on circuitEl.__laneMutationObserver at the top of paintLanesAndTracer.
- disconnect() it before any SVG writes.
- In a finally block, observe() again with the same scope so genuine row changes still re-trigger a paint afterwards.
- ResizeObserver is left untouched - it fires on frame resize, not on our writes, so it is not part of the loop.
- The bounded rAF poll in tryMeasureAndApply only writes inline tracer style. A childList observer does not see inline style changes, so we do NOT need to keep the observer paused across those frames.

R-10 parity: web-player/lobby.js and app/assets/web-player/lobby.js are byte-identical (diff produces no output). web-player/sw.js cache marker comment bumped so Mobile Safari flushes the old loop-prone bundle.

In scope: circuit plans rendering preview without going black, share button stays responsive, lobby thumbnails keep painting, treatment toggle still re-fires a paint (the observer reconnect proves itself).

Out of scope: non-circuit plans (no observer there), tracer animation correctness (covered by attempt 7/8), filmstrip / PDF / popovers.
