Device QA — Wave 18.3 patch

Purpose: three items from live device feedback on Wave 18.2. Delta-only — this script validates ONLY the items that changed in 18.3. Items covered: (1) the rest bar chip row now renders horizontally-with-wrap (was stacking each chip on its own line, full-width); (2) the coral non-default dot has been relocated from the LEFT of the chevron to a RIGHT-aligned trailing position so chevrons line up vertically across all four groups; (3) expanded group headers keep the summary on the header row with a single-line ellipsis (was disappearing) and the label stays left-anchored (was drifting centre).

In scope: mobile-only (no R-10 mirror). Exercise card (studio_exercise_card.dart) + preset chip row (preset_chip_row.dart).

Out of scope: everything covered by Wave 18.1 / 18.2. Web player. Portal.

Prerequisites

  • Wave 18.2 merged to main. Wave 18.3 branch fix/wave-18.3-qa-patch installed via ./install-device.sh.
  • At least one published session with 3+ exercises. At least one exercise with a non-default PACING (e.g. 8s prep override or custom duration on a video) and one with a long NOTES entry (40+ chars).
  • At least one rest period in the session — add via the circuit control sheet or the inline "add rest" affordance.

1 · Rest chip row — horizontal with wrap

  • 1. Open a session with a rest period. On the rest bar, the chip row renders the 4 canonical values [15s][30s][60s][90s] + [Custom] tail on a single horizontal row. Chips flow left-to-right; NOT stacked one-per-line filling the full width of the row. The rest bar reads as a tidy inline control, same shape as the PLAN chip rows.
  • 2. Tap [Custom] on the rest row, enter a value (e.g. 20), Done. Repeat until you have ~6 custom values — e.g. 20, 25, 45, 75, 120, 180. When the chips stop fitting on one row, the row wraps onto a second line. All chips remain reachable — none clip past the right edge. No horizontal scrolling; the row simply grows taller.
  • 3. With a multi-row rest chip layout from item 2, swipe left on the rest row. The full row (all chip lines included) slides off as a single Dismissible — red background + Delete label + trash icon — and the rest period is removed on release. The Wrap widget does NOT eat the swipe gesture.
  • 4. Expand PLAN on any exercise card. The reps / sets / hold chip rows still scroll horizontally — they do NOT wrap to a second line. Only the rest bar's non-scrollable chip row wraps (Wave 18.2 behaviour preserved for PLAN rows).

2 · Coral dot — right-aligned trailing position

  • 5. Find (or create) an exercise with a non-default PACING value — e.g. 8s prep override, or flip "Use video length as 1 rep" on a video. Collapse all groups. The PACING row reads ▸ PACING · 8s prep [•]: the chevron is flush-left, the label + summary flow in the middle, and the 5pt coral dot is right-aligned at the trailing edge (12pt from the card border). NOT on the left of the chevron as Wave 18.2 rendered it.
  • 6. Look down the expanded card — all four group headers (PLAYBACK / PLAN / PACING / NOTES). Every chevron sits at the exact same horizontal x-position. A default row and a non-default row (with dot) have identical chevron positions — no ~11pt drift as Wave 18.2 had. Put a thumb on the screen and trace down; the chevrons form a perfectly vertical column.
  • 7. The group label stays at its normal font weight (w700) in BOTH default and non-default states. The coral dot is the sole "this group has content" signal — the previous Wave 18.2 w800 bold treatment is gone. Compare PLAN default header text vs PACING non-default header text; both labels read at the same weight. Only the trailing coral dot distinguishes them.

3 · Expanded header — label left, summary persists

  • 8. Collapse all groups on the PACING-non-default card. Confirm the PACING row reads label + summary, left-aligned, wrapping to 2 (or 3) lines as needed. Chevron + coral dot stay top-aligned with the first line.
  • 9. Tap PACING to expand. The label stays at the same left-anchored position it had when collapsed (does NOT drift to the centre). The summary does NOT disappear; it collapses to a single line with ellipsis if it would have wrapped. Header row shape is unchanged on expand — chevron left, label+summary middle, dot right.
  • 10. The expanded PACING body (PrepSecondsRow + Use-video-length toggle + custom-duration row) renders below the header row. The header row itself doesn't change height or shape — the body slides in beneath it.
  • 11. Add a long NOTES entry (40+ chars, e.g. "Keep back straight, slow on the way down, breathe out on exertion"). Collapse → NOTES header wraps to 2/3 lines of quoted text. Tap NOTES to expand → the label stays left, summary truncates to a single line with ellipsis, TextField renders below. Tap NOTES again to collapse → summary expands back to multi-line wrap. No jitter, no layout blow-out.
  • 12. Single-open accordion is preserved (Wave 18.2 behaviour). Tap PACING → expands. Tap NOTES → PACING collapses automatically, NOTES expands. Tap NOTES again → closes. All four groups can end up collapsed. Chevron rotation + selectionClick haptic on every tap.
