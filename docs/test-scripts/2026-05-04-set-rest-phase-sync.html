Device QA — Set/rest phase sync

Bug: the set/rest state machine in web-player/app.js was double-counting each set's trailing breather. Initialising setPhaseRemaining from calculatePerSetSeconds(...) (which BAKES IN the breather per Wave 41 contract) meant every non-final set spent physical + 2×breather seconds in its 'set' phase before falling into 'rest'. User-visible: “set 1 ends and it doesn't go into its rest period; the pill at the top still keeps counting.”

Fix: new calculatePhysicalSetSeconds helper (perSet minus baked-in breather). Every fresh 'set'-phase initialisation site now uses it: beginSetMachineForCurrent, both advanceSetPhase next-set transitions, the visibilitychange fast-forward's two next-set branches, and jumpToRep. The rep-stack painter's elapsed-in-physical math also flips from perSet - setPhaseRemaining to physicalSet - setPhaseRemaining directly. calculatePerSetSeconds stays untouched (load-bearing for calculateDuration, cumulative-before-set, matrix-pill fill). Rest-phase initialisations (= breatherForX) untouched. jumpToRest untouched.

R-10 parity: the public client web player at session.homefit.studio AND the mobile workflow Preview step (which embeds the same web-player/ bundle) both pick up the fix from one edit.

Surfaces touched: web-player/app.js (helper + 6 call-site swaps), web-player/sw.js (cache name bump to homefit-player-v31-set-rest-phase-sync). No schema, no RPCs, no Flutter, no portal.

Prerequisites

  • Branch built and installed via ./install-device.sh (this also reloads the web-player bundle on the iPhone via Vercel's auto-deploy — give it ~60s after merge before testing).
  • Hard-refresh session.homefit.studio in Safari (Cmd+Shift+R or pull-down-to-refresh) so the new SW cache name (v31-set-rest-phase-sync) wins over the previous v30.
  • Test plan: a session with at least one multi-set exercise — ideally 3 sets × 10 reps with 30–60s breather between sets. Plus, if convenient, a 1-set exercise (no breather) to prove the no-regression case.

1 · Core fix — rep stack flips into breather without a gap

  • 1. Set 1 → rest 1 transition is instant. Pick a 3×10 multi-set exercise on the public web player. Hit Start Workout, watch set 1. Expected: the moment the rep-stack's last rep block fills (rep 10 lands at 100%), the sage rest block lights up immediately. No “all reps done but pill still ticking” zone. Pre-fix this had a breather_seconds_after-long stall.
  • 2. Pill, breather overlay, and rep stack stay in sync during rest. While set 1's rest phase is playing, glance at the matrix pill at the top of the screen, the big sage breather countdown overlay over the video, and the rest-block fill in the rep stack. Expected: all three advance proportionally — when the breather digits show 30 (halfway through a 60s breather), the pill and rest block are also ~halfway. No more than ~1s drift between any pair.

2 · Multi-set walk-through

  • 3. Rest 1 plays its full duration, then set 2 starts cleanly. Don't skip rest. Expected: the breather ticks 60→0 (or whatever the set's breather_seconds_after is), and at 0 the rep stack flips to set-2 active — rep 1 of set 2 starts filling.
  • 4. Final set's trailing breather plays, then slide auto-advances. Walk all sets through to the last. Expected: the final set's reps fill, the trailing breather plays its full duration, and only then does the slide auto-advance to the next exercise. Total slide time (mental stopwatch from “Start Workout”) ≈ the duration shown in the matrix pill.
  • 5. End-of-exercise badge appears during the last set's rest. While the FINAL set's trailing breather is counting down, the End-of-exercise marker on the rep stack should light up (the existing setPhase === 'rest' && isLastSet gate). Expected: badge visible during the last breather, never earlier.

3 · Matrix peek + slide change

  • 6. Matrix peek → drop on a different slide resets cleanly. Long-press the matrix at the top to enter peek mode, then drop on a different exercise slide. Expected: the new slide starts at set 1, rep stack at zero, no leftover ‘rest’ state from the previous slide. Pill resets to the new slide's full duration.

4 · Rep-stack jumps

  • 7. Tap a rep block in the rep stack — jump lands cleanly. Mid-exercise, tap (e.g.) rep 5 of set 2. Expected: the active rep block becomes rep 5, the per-rep fill restarts cleanly from 0%, and when reps land at 10/10 the rest block lights immediately (no leftover gap). The matrix pill jumps to match.
  • 8. Tap a rest block in the rep stack — jumps straight into rest. Tap a rest block (the sage one between two sets). Expected: the player jumps straight into the breather phase, breather digits count down from full, and the next set begins cleanly when the breather ends.

5 · Background tab fast-forward

  • 9. Background tab for ~2 minutes mid-set, then resume. Mid-set in a multi-set exercise, swipe back to the home screen for ~2 min (or lock the phone), then return to Safari. Expected: the rep stack, matrix pill, and breather overlay all jump forward to where they should be by wall-clock; no doubled breather at the end of the slide; total slide time still matches the pill duration.

6 · No-regression edge cases

  • 10. Single-set exercise (or 0s breather) plays unchanged. Run a 1-set exercise (or any set with breather_seconds_after = 0). Expected: behaviour is identical to pre-fix — reps fill, slide ends, no orphan rest block, no extra wait. Total slide time matches the pill duration to the second.
