02

Drift decisions

Ten things were slightly different across the three codebases. Here's what wins, what loses, and why. One coral left-edge means the proposed canonical choice. Sign off on these before the first commit.

D-01"Secondary text on dark" — two different greys were calling themselves the same thingProposed
✓ Canonical
#9CA3AF   ink.dark.secondary

Flutter + web portal already use this. Contrast on #0F1117 is 5.84:1 — passes WCAG AA for body text. Web player's #6B7280 gets retired as a "secondary" and lives on as ink.dark.muted for placeholders and helpers — so nothing is lost.

Retired
#6B7280   → ink.dark.muted

Contrast 3.56:1 on #0F1117 — AA-large only. Fine as a third tier (tertiary / muted), but too dim to carry "secondary text" like timestamps and meta rows.

D-02Dark-surface naming — literal vs semanticProposed
✓ Canonical
surface.bg / base / raised / border

Semantic naming mirrors cleanly to light mode (just swap surface.dark.* → surface.light.*). The portal already uses .base/.raised; Flutter's literal darkSurfaceVariant stops making sense the moment a light theme exists.

Retired
darkBg / darkSurface / darkSurfaceVariant

Fine for a dark-only world. Becomes awkward the moment we ship the light mirror (D-08). Rename in AppColors is a mechanical find-replace, ~20 references.

D-03Semantic colour names — full words vs abbreviationsProposed
✓ Canonical
success / warning / error / rest

Matches Flutter + web player (2 of 3), matches Tailwind convention, matches every a11y reader you'll ever grep. Portal's ok/warn/err was cute but ambiguous — warn also means "console.warn", err also means a generic error object.

Retired
ok / warn / err

Portal-only abbreviation. Rename in tailwind.config.ts is one file.

D-04Motion tokens — do they exist at all?Proposed
✓ Canonical
150ms fast · 250ms normal · 400ms slow · 1.4s pulse

Web player already had them; now they exist as named tokens everywhere. Flutter gets Duration constants in app/lib/theme/motion.dart; portal gets Tailwind transitionDuration extensions. Easing is Material-standard + emphasized for entrances.

Previously
Ad-hoc / framework defaults

Three different feels for the same interaction (e.g. a card hover). Worst on Flutter, which falls through to Material's defaults — often snappier than the web surfaces.

D-05Spacing scaleProposed
✓ Canonical
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64

Matches the de-facto Flutter ladder you already use — zero refactors on the mobile side. Indices s-1 through s-16 (skipping 7/9/11/13/14/15) to stay numerically honest: the number is always the pixel count in quarter-px units.

Considered
4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 (strict 8pt)

Cleaner ladder, but drops 20/40 which Flutter uses ~40 times. Would force visible spacing changes on every existing screen. Not worth the churn for an MVP.

D-06Component state library — empty / loading / error / success / disabledProposed
✓ Canonical
Single structure, one spec each

See components.md. Empty = icon · title · body · CTA. Loading = skeleton (lists) / Pulse Mark (full page) / inline spinner (buttons), picked by expected duration. Error mirrors Empty with a semantic.error icon. Disabled bans opacity 0.5 in favour of explicit token swap.

Retired
Every screen inventing its own

The biggest existing gap. Eight different "Plan not found" designs across the web player alone.

D-07Shadow posture — brand element or noise?Proposed (per your pick)
✓ Canonical
Flat everywhere — borders carry separation

Kills the drift in one stroke: Flutter was already flat, web surfaces drop the --shadow-card tokens. Modern, a11y-friendly (shadows are invisible at high contrast), and one fewer thing for clients on mid-range Android to render. Only shadow that survives: shadow.focus-ring for keyboard focus.

Retired
Subtle shadows on web / flat on Flutter

The cross-platform inconsistency that kicked off this whole exercise. Resolved by picking flat, not by porting BoxShadows into Flutter.

D-08The dead Flutter light themeProposed (per your pick)
✓ Canonical
Formalize — define as mirror system, don't ship

Light tokens exist in tokens.json as surface.light.* / ink.light.*, mirrored 1:1 from dark. theme.dart's light theme stays alive but guarded behind a feature flag (kEnableLightTheme = false). You get future-proof without shipping a half-baked second theme that clients might accidentally see.

Rejected
Delete light theme entirely

Honest, but loses the investment. Healthcare practices sometimes request light mode for printed handouts — having the tokens ready is cheap.

D-09Typography scale on webProposed
✓ Canonical
Inherit Flutter's 15-step scale verbatim on web

Same named tokens across all three surfaces (display.lg through label.sm). Web player gets CSS custom properties; portal gets a Tailwind fontSize extension. Inline font sizes are banned in new code.

Retired
Inline sizes + Tailwind defaults

Today's state. Leads to client-visible typographic drift (e.g. "3 sets" in Inter 13px on web, 12px/600 on mobile).

D-10Tint / alpha-overlay tokensProposed
✓ Canonical
brand.tint-bg (12%) + brand.tint-border (30%)

Explicit tokens, not computed alpha. Flutter's withValues(alpha: 0.12) looks different from CSS rgba(…, 0.12) in edge cases (sRGB vs P3 on iOS 17+). Pinning them as named tokens removes that class of bug.

Retired
Computed alpha at the call site

Works 95% of the time. The 5% is designers noticing coral looks slightly more saturated on iPhones — because it is.

Pulse Mark

ECG trace meets rooftop. Single stroke, no fill — works at any size, any background. Same canonical path across all five lockups.

Mark (animated)
logos/mark.svg · 52×36 viewBox
Horizontal lockup
logos/lockup-horizontal.svg
Stacked lockup
logos/lockup-stacked.svg · avatar-friendly
Wordmark
logos/wordmark.svg · text contexts
Light-background variant
Uses brand.dark (#E85A24) for AA contrast on light.
Clear-space rule
Padding = height of the roof peak (≈ 22% of mark height).

Rules

• Minimum size: 24px tall on-screen, 8mm in print. Below that, stroke clarity drops.
• Clear space: padding equal to the roof peak height on all sides. Nothing crosses into that zone.
• Colour: coral (brand.default) on dark. brand.dark on light. Mono (currentColor) for single-ink print.
• Don't: recolour in gradient, fill the shape, add a frame, stretch non-uniformly, or rotate. Ever.
• Animation: heartbeat pulse at 1.4s (~43bpm, calm). Reserved for loading states — not for idle or nav.

06

Voice

Three knobs: Confident, never hedging. Warm, never cute. Precise, never clinical. South African English. Full guide in voice.md.

Do
"Couldn't load this plan. Check your connection and try again."
"Not enough credits. Top up to publish this plan."
"If something hurts, stop and message your bio."
Don't
"Oops! Something went wrong 😔"
"Hooray! Your plan is live 🎉"
"Contact your healthcare provider about your condition."
Vocabulary
exercise, not workout
plan, not program
credits, not tokens
bio / practitioner, not coach
colour / favourite, not -or
POPIA-safe
Describe the activity, never claim the outcome. "Do the exercise" is fine; "fix your back" is not. Line drawings mean we never show a real client — don't break that advantage with photos.
