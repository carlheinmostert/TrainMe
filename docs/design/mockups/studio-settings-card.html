Layout choices

  • One pane, vertical scroll. No tabs. Sections separated by a 1px divider + caps-mono label. Five sections fit comfortably at default detent (~80% screen height).
  • Sticky header with the plan title + a subtitle showing the timestamp + client name (so the practitioner can confirm they're editing the right plan). Close X top-right.
  • Section order is roughly "things you tweak often" → "things you set once" → "actions" → "info". Pacing is first because rest defaults change per-plan; info is last because it's read-only.

Control patterns

  • Toggle for booleans (web-viewer flags, audio default, body focus default).
  • Stepper for numerics with discrete bumps (rest seconds, auto-insert minutes).
  • Chip group for enums with ≤ 4 options (default treatment).
  • Action row for one-shots with a trailing arrow / glyph (save-all has ↓; copy has ⧉; nav has ›).
  • Danger rendered in --danger red, separated only by section break — no extra confirm dialog (R-01 governs; SnackBar undo is the safety net).

What lives elsewhere (deliberate)

  • Crossfade tuner stays on the MediaViewer chrome — tuned in-context while watching the playback. Don't move.
  • Per-exercise treatment stays on each Studio card row — already inline.
  • Per-exercise audio toggle stays in the editor sheet's settings tab.
  • Practice picker stays as the top-left chip on Home; immutable mid-session.

Storage

  • Web viewer flags → new web_viewer_config jsonb on plans; defaults all true so existing plans don't change behaviour. get_plan_full already returns the plan row, so no new RPC.
  • Display defaults + pacing → existing columns on plans (preferred_rest_interval_seconds exists; treatment / body-focus / audio defaults are new columns OR roll into the same jsonb).

Open questions

  • Does the practitioner ever want a per-plan language for the audio prep prompts? Could go in Display defaults if so.
  • Is "Allow client to skip rests" honoured by the existing rest skip gesture, or do we need a UI hide on the timer chip?
  • If the toolbar gear shows a changed state when any non-default value is set, the visual signal helps the practitioner notice "this plan deviates from defaults" — worth a small dot indicator?
