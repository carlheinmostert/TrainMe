Static multi-state mockup of the client web player at session.homefit.studio/p/{planId}. v2 of the wireframe, reflecting the workshop output with Carl on 2026-04-22. Six phone frames below show the same player across five in-flow states plus a fullscreen-ambient variant. Every structural region is outlined and labelled using the names that exist in the code (web-player/index.html, styles.css, app.js).

v2 at a glance: matrix-eta is promoted above the pill row; card-exercise-name sits below the pills and carries appended circuit context (Round N of M); btn-prev/btn-next, mute-toggle and the new fullscreen-toggle are video-layer overlays (not top-stack chrome); card-notes is a coral overlay on the video bottom edge (no box, no border); a new rest-countdown overlay mirrors prep-overlay on rest slides; treatment-seg is retired (Studio-owned); and a new ambient mode recedes the chrome to ~30% alpha inside fullscreen.

Outline families: coral = progress/time, blue = card/media, violet = metadata (name, grammar, notes), green = navigation (chevrons, mute, fullscreen), amber = state-specific overlays, slate = ambient / dimmed chrome.

Behavioural callouts (v2)

Runtime behaviour shifts that aren’t visible in a static mockup. Load-bearing rules for the build:

  1. 1
    Video holds its first frame during the 15s prep phase. Playback only begins at countdown 0. The card-media-stub.is-held state in state 1 illustrates this.
  2. 2
    Rest slides render a sage rest-card-fill across the video area and a coral rest-countdown-overlay (same font + weight as prep-overlay) — so a standing client can read the timer from across the room.
  3. 3
    card-notes is a coral overlay at the bottom of the video — no background box, no border. 3-line -webkit-line-clamp by default; tap to expand to full, tap again to collapse. Visible + tappable during prep so the client can pre-read.
  4. 4
    Fullscreen hides plan-bar (plan-title + header-client) entirely. The rest of the top stack (matrix-eta, progress-matrix, card-exercise-name, grammar) stays rendered but dims to ~32% alpha.
  5. 5
    All video-layer coral chrome (btn-prev, btn-next, mute-toggle, fullscreen-toggle, card-notes) fades to ~32% alpha in fullscreen. Tap anywhere on the video pops them to full coral for 3s, then fades back. prep-overlay, media-pause-overlay, and rest-countdown-overlay stay at 100% regardless — they’re state overlays, not chrome.
  6. 6
    treatment-seg is retired from the player. The practitioner’s per-exercise preferred treatment is set in Studio (preferred_treatment); the web player honours it silently and shows no switcher.
  7. 7
    Circuit context is doubled for redundancy: spatially via matrix-col.is-circuit column stacking + the appended Round N of M suffix on card-exercise-name. The legacy text-only circuit-indicator bar is gone.
  8. 8
    Swipe remains the primary mobile gesture. Mid-edge btn-prev/btn-next chevrons are the accessibility path + desktop browser affordance. nav-bar at the bottom is now dots only — it no longer carries chevrons.

Region legend (v2)

Every outline in the gallery above maps to a real DOM id / class in the web-player codebase (current or proposed-for-v2). Region names are quoted verbatim from web-player/index.html + styles.css + app.js so “the matrix-eta” means the same thing to design and engineering. Grouped by outline family.

progress + time (coral)

plan-bar [all except fullscreen] Row 1 of top-stack. Holds plan-title (session title) and header-client (client name), rendered inline as Title · Client. Hidden entirely in fullscreen (ambient mode).
matrix-eta (promoted) [all states] Row 2 of top-stack. 3-number time line — matrix-eta-current (coral, active-slide remaining) · matrix-eta-total (white, whole-plan remaining, with LEFT suffix) · matrix-eta-finish (muted, wall-clock finish). Promoted ABOVE the pill matrix in v2.
progress-matrix [all states] Row 3. Pill row, auto-centered on the active pill. Canonical source: docs/design/mockups/progress-pills.html. Unchanged position relative to v1.
pill [all states] Each exercise / rest slot. Empty idle, coral when active (pulse-glow) or completed, sage when rest. Holds pill-fill (live timer fill bar).
matrix-col.is-circuit [circuit] Vertical stack of pills — one row per cycle — sitting inside a coral tint band. Adjacent circuit columns seam together.

card + media (blue)

video-viewport [all states] Outer padded area that houses video-surface. Replaces v1’s card-viewport + exercise-card + card-inner compound — the player is now video-led with chrome around it, not a card inside a viewport.
video-surface [all states] Rounded rectangle holding the video element + all video-layer overlays (top-right pair, mid-edge chevrons, card-notes, state overlays). Hairline border, rounded 18px.
card-media-stub.is-held [prep] In the mockup, represents the video held on frame 0 during prep. In build, the real video element seeks to 0 and pauses until prep ends.

metadata (violet)

card-name-row [all states] Row 4 of top-stack. Two-line block: card-exercise-name on line 1 (with appended circuit context when applicable), decoded grammar on line 2 (3 sets · 10 reps · 5s hold).
card-exercise-name [exercise] Exercise title. Montserrat 15-16pt 700. Circuit mode appends · Round 2 of 3 inline (no separate legacy bar). On rest slides it shows NEXT UP: Single-leg bridge as a preview.
card-body (decoded grammar) [all states] Second line under the title — 3 sets · 10 reps · 5s hold (standalone), 10 reps · 5s hold (circuit; sets carried by matrix), 30s rest / 2 sets · 8 reps each side (rest next-up).
card-notes [exercise · video overlay] NEW placement. Coral free-text note from the practitioner, positioned absolute at the bottom of video-surface. No background box, no border — just coral text with a subtle shadow for legibility. 3-line ellipsis by default; tap to expand. Visible + tappable during prep so the client can pre-read.

navigation (green)

btn-prev / btn-next [all states · video overlay] Moved in v2. Coral chevron buttons sitting at the mid-height edges of video-surface. Low alpha by default, full coral on hover/tap. Swipe remains the primary mobile gesture; chevrons are accessibility + desktop.
mute-toggle [all states · top-right] Coral speaker icon in the top-right pair of video-surface. Only rendered when the exercise has audio; flips runtime isMuted.
fullscreen-toggle [all states · top-right] NEW. Coral icon in the top-right pair. Enter-fullscreen glyph when not in FS (⛶); minimise glyph when in FS. Calls the Fullscreen API on desktop browsers; on iOS Safari falls back to WebKit presentation mode on the video element.
nav-bar (dots only) [all states] Simplified in v2. Now hosts only nav-dots. Chevrons moved to the video edges.
nav-dots [all states] Slide indicator dots. Active dot is coral elongated. Hide automatically past 10 slides per current spec.
peek-panel [teaching, scrubbing] Centered dark-card showing peek-name + peek-meta. Auto-shows for 2s on preview start (teaching peek R-09); reappears during long-press matrix scrub.

state-specific overlays (amber)

prep-overlay [prep] Big coral countdown centered over the held first frame. Synchronised with matrix-eta-current + active pill is-prep-flashing. Stays at full coral even in ambient mode.
rest-countdown-overlay [rest] NEW. Mirrors prep-overlay typographically (Montserrat 800, 120px, letter-spacing -6px). Big coral number centered over the sage rest-card-fill. Readable from across the room. Stays at full coral in ambient.
media-pause-overlay [paused] Centered dark-disc play triangle. Toggled visible only on the active paused slide. Stays at full coral in ambient.
rest-card-fill [rest] Restructured. Sage-tinted gradient + idle rest-icon filling the entire video-surface. No longer a sibling card with its own body — the body grammar now lives in the top-stack card-name-row as a next-up preview.
workout-complete [final state] Full-screen overlay shown when the last slide completes. Holds circle-check icon + total time + Close button. Not drawn in the 6 frames above.
start-workout-btn [pre-start] Floating coral pill at bottom-center that kicks off the 15s prep countdown on slide 0. Not drawn above.

ambient / dimmed chrome (slate) — NEW

ambient-dim-chrome [fullscreen] Modifier applied to screen (or the player root) when Fullscreen API engages. Drops plan-bar entirely, dims top-stack and video-layer chrome to ~32% alpha. Tapping anywhere on the video toggles a 3-second “chrome reveal” class that pops everything to full coral, then fades back.
chrome-reveal-timer [fullscreen] Logic only — 3-second timeout that cancels itself on each tap and re-arms. Mirrors YouTube / Netflix chrome behaviour.
state-overlay exemption [fullscreen] prep-overlay, media-pause-overlay, rest-countdown-overlay are exempt from the dim — they’re load-bearing content, not chrome.
retired: treatment-seg [—] v1 segmented control “Line · B&W · Original” removed. Per-exercise preferred treatment is set in Studio now (exercises.preferred_treatment) and honoured silently by the player.
