Page layout

What the client sees when opening session.homefit.studio/p/{id}. Drag to re-arrange the vertical stack.

  • Plan bar top
    Plan title (left), client name (left, sub-line), progress indicator "3 / 10" (right). Brand anchor + location.
  • Progress pill matrix
    Horizontally scrolling row of small pills, one per exercise. Coloured by state: empty (upcoming), full coral (done), sage (rest), pulse on active. Auto-centres on active pill. ETA widget ("1:36 · 7:42 left · ~7:42 PM") lives inside this matrix's right end.
  • Card viewport swipeable
    The primary content area — swipeable carousel of exercise / rest cards. Tap to pause, swipe to skip. See the right-hand frame for its internal anatomy.
  • Nav bar bottom
    Prev button (left), dot indicators (centre, hidden past 10 slides), Next button (right). Secondary to swipe — visible primarily on desktop.
  • Footer brand bottom
    "powered by homefit.studio" wordmark + matrix-logo glyph. Subtle, low-contrast.
  • Peek panel floats
    Centered floating card. Shows exercise name + decoded grammar ("3 × 12 · 5s hold"). Auto-appears 2s on preview start (teaching peek), then surfaces only during long-press scrubbing.
  • Start Workout button floatscond.
    Coral pill, floating near bottom. Shown only before first exercise begins. Tapping kicks off 15s prep countdown.
  • Pause overlay floatscond.
    Centered play-triangle glyph shown only when the current video is paused. Tap media → pause / resume.
  • Prep countdown overlay floatscond.
    Big coral countdown numbers overlayed on the current card's media during the 15s pre-exercise window. Top-bar tokens flash in sync at 1 Hz.
  • Workout Complete floatsend
    Full-screen final state: circled check glyph, "Workout Complete!" title, total time, Close button. Only shown after the last exercise finishes.
  • Loading screen init
    Initial state before the plan fetches. Pulse-mark logo + "Loading your plan…". Replaced by app once data arrives.
  • Error screen error
    Shown when the plan ID is invalid / expired. "!" icon + "Plan not found" + contact-practitioner hint. Full-screen, replaces app.

Exercise card (inside viewport)

Anatomy of a single exercise card. This is what the Card viewport renders per slide. Rest slides are a sibling variant — see Rest card below.

  • Exercise name
    Bold title, e.g. "Goblet squat" or "Hip hinge". Optionally with a small index "3 of 10".
  • Treatment toggle
    Three-segment coral control: Line · B&W · Original. Locked segments (per client consent) show a lock glyph. Controls what video/image the media area renders.
  • Media area tap = pause
    The video or photo itself. Fills most of the card height. Auto-plays muted + looped on active slide. Tap = mode-aware pause/play/skip-prep.
  • Decoded grammar
    Bottom text line — "3 × 12 reps · 5s hold" or circuit context "Circuit · Round 2 of 3 · Exercise 1 of 3".
  • Notes if present
    Practitioner's free-text cue, e.g. "Keep back straight, slow on the way down". Only renders if populated.
  • Timer chip floats
    Bottom-right of the card. Three modes: prep (countdown ticking down 15 → 0), running (per-set timer), paused (play-triangle). Tap → skip prep / toggle pause.
  • Rest card variant rest-type
    Sibling card shape for rest slides: sage-tinted body, "Rest · 90s" centred, timer chip still bottom-right. No media, no treatment toggle, no notes.
