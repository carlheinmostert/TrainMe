Why this works (and what it kills)

No JS. Pure CSS keyframes on box-shadow / pseudo-elements. Animations run on the compositor thread, not the main thread.

No DOM mutation. CSS animations don't trigger MutationObservers. The runaway loop class of bugs goes away forever.

No measurement. No offsetHeight, no getTotalLength(), no rAF retry. Works the same in Safari and WKWebView because nothing depends on timing.

What we give up: the spiraling-tracer visual. The pulse is a different semantic — "this group cycles" instead of "energy travels along a path". Carl's call which semantic he prefers.
