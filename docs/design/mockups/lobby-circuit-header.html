Side-by-side mockup for hotfix round 2. Left: the broken state Carl saw on his iPhone after PR 234 merged — coral backdrops on the matrix circuit blocks AND the circuit container itself, a duplicate header on the first circuit (rest-inside-circuit closes the group then the next exercise re-opens it), redundant row numerals, and a verbose “× 3 ROUNDS” chip. No numerals. Cycles chip is just ×3. 

RAIL COMPOSITION (right panel)

   ┌─ circuit-header (32px)
   │     ::before    rail piece: centre → bottom            (3 px wide)

   │
   ├─ ex-row.in-circuit (first)
   │     ::before    rail piece: top → bottom               (continuous with header)

   │
   ├─ ex-row.in-circuit (middle)   …same as first…
   │
   └─ ex-row.in-circuit.last
         ::before    rail piece: top → 50% only             (terminates at centre → └)
         ::after     connector: left:8 → row content        (14 × 3, at row centre)

The container itself has NO rail. The rail is composed of header + per-row
pieces, so the height of the rail naturally tracks the rendered rows
without any JS measurement.

What changed

  • Container. .lobby-circuit-group goes from coral-tint background + coral-tint border to fully transparent. All visual grouping is conveyed by the rail.
  • Rail tokens. 3 px wide, left: 8px, colour rgba(255,107,53,0.85) (use --c-coral-rail).
  • Header rail piece. .circuit-header::before = centre → bottom of header (3 px). ::after = 14 × 3 horizontal stub at vertical centre, joining rail to label.
  • Per-row connectors. .ex-row.in-circuit::before = full-height vertical piece. ::after = 14 × 3 stub at row centre. Rows abut (no gap) so the rail concatenates seamlessly. padding-left: 26px clears the rail and connector.
  • Last-row L. .ex-row.in-circuit.last::before sets height: 50% instead of top:0; bottom:0 so the rail terminates at the row centre — combined with the connector at that centre line, it forms a clean └ corner.
  • Numerals. Removed entirely. Default-named exercises (“Exercise 1”, “Exercise 2”) already convey their position via Studio's auto-naming; renamed exercises trust the practitioner.
  • Cycles chip. “× 3 ROUNDS” → ×3. Just the multiplier.
  • Duplicate-header bug. Root cause: a rest slide that lives inside a circuit closed the open circuitGroup in the renderList loop; the very next circuit-id slide then opened a NEW group with the same id — rendering the same coral header twice. Fix: rest slides whose circuit_id matches the open group are appended to the group's rows instead of closing it.
