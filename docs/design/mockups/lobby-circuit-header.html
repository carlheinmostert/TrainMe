Lobby circuit header — tree-branch rail (round 5)

Side-by-side mockup brought up to round 5 reality. Left: the broken state Carl saw on his iPhone after PR 234 merged — coral backdrops on the matrix circuit blocks AND the circuit container itself, a duplicate header on the first circuit, redundant row numerals, and a verbose “× 3 ROUNDS” chip. Right: the corrected geometry as of round 5 — container transparent; group structure conveyed by a 3 px coral tree-branch rail; gutter widened to 56 px with the rail at left: 24px so the rail+connector cluster sits indented from the page edge; rail span gets a -5/+5 vertical bleed so adjacent pieces overlap the inter-row gap (no sub-pixel break); connector tip stops ~6 px before the card border (~16 px breathing with the card's 10 px padding); the last in-circuit row clamps the rail at height: calc(50% + 5px) to form a clean L corner. Default-named circuits render as Circuit A / Circuit B using the first-appearance letter map (mirrors Studio's _circuitLetter); custom names render verbatim.

KEY ARCHITECTURAL POINTS:
1. Rail pieces are REAL elements inside `.ex-row-gutter`, NOT pseudo-
   elements on `.ex-row`. Active-row coral border (intentional UX) sits
   on the CONTENT card and wraps just the card; the rail in the gutter
   column is unaffected.
2. Gutter widened 22 → 56 (round 5 Fix 3): rail at left:24, connector
   width 26, connector tip ~6px before card border → ~16px breathing
   with card padding. Cluster is indented from page edge with breathing
   on both sides; nothing touches the card.
3. Rail span has top:-5/bottom:-5 bleed (round 5 Fix 2): adjacent pieces
   overlap the inter-row gap so no sub-pixel break shows up at row
   boundaries. Last row uses calc(50% + 5px) so the upward bleed is
   preserved while the bottom edge still lands exactly at the row's
   actual vertical centre (the connector's height).
4. Default-named circuits resolve to "Circuit {Letter}" via a
   first-appearance letter map (round 5 Fix 1) mirroring Studio's
   _circuitLetter helper; custom names render verbatim.

What changed (round 5: name fallback + rail bleed + wider gutter)

  • Round 5 — circuit name fallback. When neither slide.circuitName nor plan.circuit_names[circuit_id] resolves, renderList in lobby.js substitutes Circuit {Letter} using a first-appearance letter map (A, B, C, … mod 26). Mirrors Studio’s _circuitLetter helper at app/lib/screens/studio_mode_screen.dart:1449. Custom names render verbatim. Right-panel mockup uses Circuit A to make the fix visible.
  • Round 5 — gutter widened. flex: 0 0 22px → flex: 0 0 56px on every gutter column (header + per-row). Rail moved from left: 11px to left: 24px within the gutter; connector width grew from 11 to 26 so its tip lands ~6 px before the card border. With the card’s 10 px padding-left that’s ~16 px of breathing between connector tip and the hero / info content. Cluster sits clearly indented from the page edge.
  • Round 5 — rail-piece bleed. .ex-row-gutter-rail gets top: -5px; bottom: -5px on regular in-circuit rows so adjacent rail pieces overlap into the inter-row gap and the line reads as one continuous coral. Header rail uses bottom: -5px for the same reason. Last row uses top: -5px; height: calc(50% + 5px) so the upward bleed is preserved while the bottom edge still lands exactly at the row’s actual vertical centre (where the connector sits) for the L corner.
  • Two-column rows (round 3 carry-over). Each row is a real flex container with a fixed-width gutter column on the left and a flex content column on the right. The rail pieces are real <span> children of the gutter, NOT absolutely-positioned pseudo-elements on the row. Result: the rail sits in its own layout slot OUTSIDE the card’s bounding box, so the active-row coral border (intentional UX) wraps around just the card and the rail is unaffected.
  • Container. .lobby-circuit-group stays transparent (no backdrop, no border). All grouping is conveyed by the rail.
  • Rail tokens. 3 px wide; colour rgba(255,107,53,0.85) via --c-coral-rail.
  • Per-row connectors. .ex-row-gutter-rail = -5 → +5 vertical piece (bleed both ends). .ex-row-gutter-connector = 26 × 3 stub at row centre, terminating ~6 px before the card border. Adjacent in-circuit cards collapse top borders (margin-top: -1px) so the rail reads as one continuous coral line.
  • Last-row L. .ex-row.in-circuit.last .ex-row-gutter-rail uses top: -5px; height: calc(50% + 5px) so the rail terminates at the row centre — combined with the connector at that centre line, it forms a clean └ corner with no boundary gap above it.
  • Numerals. Removed entirely (round 2). Default-named exercises have a fallback “Exercise N” label generated in lobby.js from a position counter (round 3), since Studio doesn’t persist the auto-name to the cloud row. Renamed exercises trust the practitioner.
  • Cycles chip. “× 3 ROUNDS” → ×3. Just the multiplier.
