Interaction model

  • Tap a handle → video pauses if playing, seeks to the handle's offset, handle gets the coral glow ring + time tooltip.
  • Drag a handle → moves it; the playhead follows live (the dragged handle is implicitly selected during the drag).
  • Tap the video (anywhere outside the trim panel) → resume playback. Selection clears (no handle is "holding" the playhead during play).
  • Playback flash — while the video is playing, all handles are white. When the playhead crosses an offset (begin / Hero / end) within ±100ms, the corresponding handle pulses coral for ~600ms and fades back. No tooltip, no persistent selection. The flash is the only "you just passed it" cue.
  • Hero offset is clamped to [startOffsetMs, endOffsetMs]. Drag begin past hero or end past hero → hero stays at the bound, doesn't slide.
  • Persistence: begin/end already write start_offset_ms / end_offset_ms. Hero writes focus_frame_offset_ms + triggers the existing regenerateHeroThumbnails regen. Both happen on drag-end / programmatic move.
  • Tab strip: Hero tab is removed. Plan · Notes · Preview · Settings (4 tabs).
