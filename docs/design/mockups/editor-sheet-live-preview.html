Spec

  • Preview: 168 × 110pt rectangle (slightly less than half the chrome's width on iPhone 14), border-radius: 12pt. Edge-anchored to the chrome's left padding (14pt). Object-fit cover.
  • Source: same line-drawing H.264 the web player serves (line_drawing_url). Muted, autoplay, loop. Already on disk via the converter.
  • Chevrons: 32 × 32pt circles centred vertically on the preview. Left chevron at left: 6pt, right chevron at right: 6pt. Coral icon stroke (--brand-default, 18pt, 2.4 stroke), translucent dark backdrop (rgba(15,17,23,0.55)) with a thin coral border. Disabled state: 35% alpha on coral, 18% on border.
  • The chevrons replace the existing in-chrome ‹ › IconButtons. Only ONE prev/next pair anywhere on the sheet — no duplication.
  • "Playing" pill bottom-right of preview — pulsing red dot + tiny label so the practitioner knows it's a live tile vs a still.
  • Title block (right of preview): inline-editable title with the existing dashed underline; meta line beneath (VIDEO · 8s · 3 reps captured or REST · 60s).
  • The preview itself is NOT a hit target — taps pass through to the title's existing interactions (rename) and the chevrons own their own hit zones. Per Carl: "I don't want the preview to be a hit target. It does not need it."
  • Rest variant: keep the sage glyph as before (no video). Tab strip shows ONLY the "Rest" tab. Chevrons still appear (still coral) so the practitioner can step out of the rest period to the next exercise.
  • Performance: pause the preview when the editor sheet's body shows the Preview tab (the tab embeds MediaViewerBody which would otherwise re-decode the same file). Pause off-screen previews when the chrome is scrolled out of view.
