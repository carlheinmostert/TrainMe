🔑 The Hybrid Approach

Why we don't use pure AI for equipment:

✓ AI Generates Body

Excellent at anatomically correct human poses. ControlNet ensures pose consistency with OpenPose skeletons.

⚙️ Deterministic Equipment

SVG assets anchored to skeleton joints. No hallucination, perfectly positioned dumbbells, bands, cables.

Result: Professional, accurate exercise animations without expensive licensed libraries.

Stage 1: Text-to-Motion (SayMotion API)

What happens:

Trainer types "Standing bicep curl with dumbbell" → SayMotion generates 3D human motion → Returns BVH file with 10+ frames of joint data.

Data format:

BVH (Biovision Hierarchy)
- 22 joints per frame
- 3D positions (X, Y, Z)
- 10-30 frames total

Timeline:

~2-5 minutes per exercise (depends on API load)

API Used:

POST /job/v1/process/text2motion
Authorization: Basic Auth
Response: { rid: "request_id" }

Cost:

~1 credit per 10 seconds of animation
Replicate pricing: $15/mo for 50 credits

Success criteria:

✓ Recognizable exercise movement
✓ Realistic joint angles

Stage 2: Pose Extraction & Skeleton Generation

What happens:

Parse BVH file → Select 4-6 keyframes (start, movement phases, end) → Project 3D joints to 2D → Render as skeleton stick figures.

Keyframe selection logic:

Frame 0: Starting position
Frame N: First movement
Frame M: Peak/midpoint
Frame K: Ending position

Output format:

6x PNG files (512x512)
OpenPose 18-joint format
White background, black skeleton

Processing:

Local (no API calls)
~5 seconds total

OpenPose joints:

0: Nose
1-4: Right arm
5-7: Left arm
8-10: Right leg
11-13: Left leg
14-17: Face

Why OpenPose?

Standard format for ControlNet conditioning. Ensures pose accuracy in next stage.

Stage 3: ControlNet + FLUX Rendering

What happens:

For each skeleton PNG: Submit to Replicate API with FLUX+ControlNet → Model generates illustrated figure matching pose → Download rendered image.

Prompt used:

"A fit person doing a standing bicep curl, illustrated style, anatomically correct, professional fitness illustration"

ControlNet parameters:

control_type: "openpose"
control_strength: 0.8
guidance_scale: 7.5
steps: 28

Timeline:

~10-30 seconds per frame
6 frames = 1-3 minutes total

Cost:

~$0.01-0.05 per image
6 images = $0.06-0.30

Output:

6x PNG files (512x512)
Illustrated human figures
Consistent character appearance

Why consistent?

IP-Adapter anchors character ID across frames. Style LoRA maintains illustration aesthetic.

Stage 4: Equipment SVG Compositing

What happens:

AI generated the figure, but NOT the dumbbell. We overlay a dumbbell SVG asset at the hand joint positions.

Why not AI?

AI struggles with equipment physics:
✗ Dumbbells bend
✗ Bands phase through skin
✗ Cables don't connect

Deterministic approach:
✓ Perfect positioning
✓ Realistic physics

Equipment assets:

dumbbell.svg
resistance_band.svg (future)
cable_handle.svg (future)
exercise_ball.svg (future)

Positioning logic:

Get hand joint positions from skeleton
Calculate center point
Determine rotation based on hand movement
Overlay dumbbell SVG
Composite onto figure

Processing:

Local (PIL/Pillow)
~1 second per frame

Output:

6x PNG files (512x512)
Figure + Equipment
Ready for animation

Stage 5: Animation Assembly

What happens:

Take 6 composited frame PNGs and assemble into viewable animation formats.

Output formats:

1. Lottie JSON
(web, mobile native)

2. Sprite sheet PNG
(quick preview)

3. HTML viewer
(step-through player)

Why Lottie?

✓ Tiny file size (~12KB)
✓ Scales to any screen
✓ Playback control (pause, step)
✓ Works offline when cached
✓ Works in web + mobile apps

Processing:

Local (python-lottie)
~5 seconds total

Timeline in portal:

Trainer sees: "Generating bicep curl..."
Waits 5-8 minutes
Animation appears in catalogue
Can drag into training plans

Total cost per exercise:

Motion generation: free (with SayMotion credits)
Rendering: $0.06-0.30
Equipment: $0 (SVG)
Total: ~$0.10-0.35
